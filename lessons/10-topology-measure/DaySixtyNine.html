Section 1

Deterministic Finite Automata (DFA)

A DFA is a machine with finite memory. It consists of a 5-tuple `(Q, \Sigma, \delta, q_0, F)`:

  • `Q`: Finite set of states.
  • `\Sigma`: Finite alphabet (e.g., `{0, 1}`).
  • `\delta`: Transition function `Q \times \Sigma \to Q`. (Takes state + input, gives new state).
  • `q_0`: Start state.
  • `F`: Set of Accept states.

It accepts a string `w` if the sequence of states ends in `F`. The set of all strings accepted by a DFA is a Regular Language.

Section 2

Nondeterministic Finite Automata (NFA)

An NFA can be in multiple states at once. It can assume multiple paths or "guess".

Theorem: Every NFA has an equivalent DFA (Subset Construction).

While NFAs are easier to design, they don't have more power than DFAs. However, the equivalent DFA might have exponentially more states (`2^{|Q|}`).

Section 3

Regular Expressions

Algebraic notation for regular languages. Built using union (`|`), concatenation, and Kleene star (`*`).

Kleene's Theorem: A language is Regular `\iff` it is recognized by a DFA `\iff` it is described by a Regex.

Note: Modern "Regex" in programming (with backreferences) is strictly more powerful than theoretical Regex (it can parse non-regular languages).

Section 4

The Pumping Lemma

How do we prove a language is NOT regular?

Since a DFA has finite states, if a string is long enough (`|w| \ge p`), it must loop (visit a state twice). This loop can be pumped.

w = xyz, \quad |xy| \le p, \quad |y| > 0, \quad xy^iz \in L \quad \forall i \ge 0

Example: `L = \{0^n 1^n\}` is not regular because you can't pump the 0s without matching the 1s.

Section 5

Closure Properties

Regular languages are closed under:

  • Union (Parallel NFAs)
  • Concatenation (Chain NFAs)
  • Star (Loop NFA)
  • Intersection (Product Construction of DFAs)
  • Complementation (Flip accept/reject states in DFA)
Section 6

Context-Free Grammars (CFG)

Regular languages cannot handle nested structures like `((()))`. CFGs can.

A CFG consists of variables, terminals, and rules like `S \to A | B`.

Example: `S \to 0S1 | \epsilon` generates `\{0^n 1^n\}`.

"Context-Free" means we can replace a variable `A` regardless of what surrounds it.

Section 7

Pushdown Automata (PDA)

Theorem: A language is Context-Free `\iff` it is recognized by a Non-deterministic PDA.

A PDA is an NFA with a Stack (LIFO memory).

Important: Deterministic PDAs (DPDA) are strictly weaker than Non-deterministic PDAs. This is unlike Finite Automata where DFA=NFA. Programming languages usually target DPDAs for efficient parsing.

Section 8

Ambiguity

A grammar is ambiguous if a string has multiple distinct parse trees.

Example: `E \to E+E | E*E | a`. The string `a+a*a` can be parsed as `(a+a)*a` or `a+(a*a)`.

We fix this in compilers using operator precedence. Some languages are inherently ambiguous.

Section 9

Chomsky Normal Form

Any CFG can be simplified to rules of the form:

  • `A \to BC` (Two variables)
  • `A \to a` (One terminal)

This canonical form enables the CYK Algorithm, which checks if a string is in the grammar in `O(n^3)` time.

Python: FSM & Parser

Python DFA & Parser 
# 1. Finite Automata (DFA)
class DFA:
    def __init__(self, transitions, accept_states):
        self.transitions = transitions
        self.current_state = 'q0'
        self.accept_states = accept_states

    def process_symbol(self, symbol):
        self.current_state = self.transitions.get((self.current_state, symbol), 'reject')

    def process_string(self, string):
        self.current_state = 'q0'
        for char in string:
            self.process_symbol(char)
        return self.current_state in self.accept_states

# Accept strings ending in '11'
transitions = {
    ('q0', '0'): 'q0', ('q0', '1'): 'q1',
    ('q1', '0'): 'q0', ('q1', '1'): 'q2',
    ('q2', '0'): 'q0', ('q2', '1'): 'q2'
}
dfa = DFA(transitions, {'q2'})
print(f"DFA '0011': {dfa.process_string('0011')}")

# 2. Recursive Descent Parser
# S -> 'a' S 'b' | epsilon (Language: a^n b^n)
class SimpleParser:
    def __init__(self, token_stream):
        self.tokens = list(token_stream)
        self.pos = 0

    def consume(self, expected):
        if self.pos < len(self.tokens) and self.tokens[self.pos] == expected:
            self.pos += 1
            return True
        return False

    def parse_S(self):
        # S -> 'a' S 'b'
        if self.consume('a'):
            if self.parse_S():
                return self.consume('b')
            return False
        # S -> epsilon
        return True

    def parse(self):
        return self.parse_S() and self.pos == len(self.tokens)

print(f"Parser 'aabb': {SimpleParser('aabb').parse()}")
print(f"Parser 'aabbb': {SimpleParser('aabbb').parse()}")
