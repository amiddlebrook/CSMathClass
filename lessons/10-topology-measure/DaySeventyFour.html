Section 1

PSPACE

SPACE(f(n)): Maximum number of tape cells used by a TM on input size `n`.

PSPACE: Problems solvable using polynomial space.

We know `P \subseteq NP \subseteq PSPACE`.

Surprisingly, `PSPACE = NPSPACE` (Savitch's Theorem).

Section 2

Savitch's Theorem

Theorem: `NSPACE(f(n)) \subseteq SPACE(f(n)^2)`.

This means nondeterminism doesn't help much with space. We can simulate a nondeterministic path using recursion (Depth First Search).

With time, we have to explore all branches simultaneously (Parallel/BFS), which takes exponential time. With space, we can reuse the same memory for each branch (Serial/DFS).

Section 3

Logarithmic Space (L vs NL)

L: Deterministic Log Space. (e.g. searching a list with pointers).

NL: Nondeterministic Log Space. (e.g. Graph connectivity).

It is known that `L \subseteq NL \subseteq P`.

Surprise: `NL = coNL`. (Immerman–Szelepcsényi theorem).

Section 4

The Hierarchy

L \subseteq NL \subseteq P \subseteq NP \subseteq PSPACE \subseteq EXP

We know `L \neq PSPACE` (Space Hierarchy Theorem) and `P \neq EXP` (Time Hierarchy Theorem).

But we don't know if `P=NP` or `NP=PSPACE`. Most are believed to be strict subsets.

Section 5

Games & Space

PSPACE problems often look like 2-player games (e.g., Generalized Chess, Go).

"Is there a winning strategy?" requires searching a game tree. This tree has polynomial depth (turns) but exponential size. DFS can solve it in Poly Space.

Section 6

True Quantified Boolean Formulas

SAT asks `\exists x_1 \dots \exists x_n \phi(x)`. This is NP.

TQBF allows universal quantifiers: `\exists x_1 \forall x_2 \exists x_3 \dots \phi`. This is PSPACE-Complete.

It represents a game: "Does Player 1 (Existential) have a move such that for all Player 2 (Universal) moves..."

Section 7

Generalized Geography

A game played on a directed graph. Players take turns moving a token along an edge. You cannot reuse nodes. First player unable to move loses.

This is PSPACE-Complete. Most "generalized" board games (Go, Hex, Reversi) are PSPACE-hard.

Section 8

Planning Problems

Sokoban (pushing boxes) is PSPACE-Complete.

Robot Motion Planning with movable obstacles is often PSPACE-Complete.

Why? Because to solve the puzzle, you might need to undo moves, cycle back, and explore a state space exponentially larger than the board.

# Space Complexity ~ Recursion Depth (Stack Height)

def depth_first_search(node, target, visited):
    # O(D) space where D is max depth
    if node == target:
        return True
    visited.add(node)
    
    for neighbor in get_neighbors(node):
        if neighbor not in visited:
            if depth_first_search(neighbor, target, visited):
                return True
                
    visited.remove(node) # Backtrack: Save space!
    return False

Python QBF Evaluator 
def evaluate_qbf(formula_tree):
    """
    Evaluates a Quantified Boolean Formula.
    Tree Structure: ('forall', 'x', sub_expr) or ('exists', 'x', sub_expr) ...
    """
    op = formula_tree[0]
    
    if op == 'const': return formula_tree[1]
    
    if op == 'exists':
        # Try True, Try False. If ANY is true, return True.
        var, sub = formula_tree[1], formula_tree[2]
        return evaluate_qbf(substitute(sub, var, True)) or \
               evaluate_qbf(substitute(sub, var, False))
               
    if op == 'forall':
        # Try True, Try False. If BOTH are true, return True.
        var, sub = formula_tree[1], formula_tree[2]
        return evaluate_qbf(substitute(sub, var, True)) and \
               evaluate_qbf(substitute(sub, var, False))

    # This recursion depth is proportional to the number of variables (space)
    # But execution time is exponential.
    return False
