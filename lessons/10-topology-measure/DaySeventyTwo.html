Section 1

Time Complexity

We measure efficiency by counting steps `f(n)` relative to input size `n`.

Polynomial Time: `O(n^k)` for some constant `k`. This is "efficient" in theory.

Exponential Time: `O(2^n)`. This is "intractable".

Section 2

Class P

P is the class of decision problems solvable by a deterministic Turing Machine in polynomial time.

Examples: Sorting, Path Finding, Primality Testing, GCD.

P is robust: Changing from one model of computation to another (e.g., 1-tape to k-tape) only changes complexity by a polynomial factor.

Section 3

Class NP

NP stands for Nondeterministic Polynomial time. It is NOT "Non-Polynomial".

A problem is in NP if a Nondeterministic TM can solve it in polynomial time. (It guesses the solution).

Alternative Definition: A problem is in NP if a solution can be Verified in polynomial time by a deterministic TM.

Section 4

Verifier Definition

For a problem `A` (e.g., Hamiltonian Path), a verifier `V` works as follows:

w \in A \iff \exists c \text{ (certificate) such that } V(w, c) \text{ accepts in poly time.}

If you give me a path (certificate), I can quickly check if it visits every city once. Finding that path might be hard, but checking it is easy. That is NP.

Section 5

P vs NP Question

We know `P \subseteq NP`. (If you can find it, you can verify it).

The Million Dollar Question: Is `P = NP`?

Interpretation: "Can every problem whose solution can be quickly verified also be quickly solved?"

Most scientists believe `P \neq NP`. If `P=NP`, cryptography would break, and creativity could be automated.

Section 6

Python: Verification

Python Subset Sum Verifier 
def verify_subset_sum(numbers, target, certificate):
    """
    Problem: Given a list of numbers, is there a subset that sums to target?
    This is in NP. Finding the subset is hard (O(2^n)).
    Verifying a given subset is easy (O(n)).
    """
    # 1. Check if certificate is valid subset
    # (Certificate is a list of indices)
    current_sum = 0
    used_indices = set()
    
    for idx in certificate:
        if idx < 0 or idx >= len(numbers):
            return False # Invalid Index
        if idx in used_indices:
            return False # Duplicate Index
        used_indices.add(idx)
        current_sum += numbers[idx]
        
    # 2. Check if sum matches target
    return current_sum == target

# Example
nums = [3, 7, 1, 9, 2]
t = 10
cert = [0, 1] # Indices for 3 and 7

print(f"Verified: {verify_subset_sum(nums, t, cert)}")
