Section 1

Breadth-First Search (BFS)

BFS: Explore all neighbors at current depth before moving to next depth. Uses queue.
BFS ALGORITHM: 1. Start at source, add to queue 2. While queue not empty: a. Dequeue vertex v b. For each unvisited neighbor u: - Mark u visited - Enqueue u Time: O(V + E) Space: O(V)

Use cases

  • Shortest path: In unweighted graphs
  • Level-order traversal: Process by distance
  • Connected components: Find all reachable nodes
  • Bipartite check: 2-colorable?
Section 2

Depth-First Search (DFS)

DFS: Go as deep as possible before backtracking. Uses stack (or recursion).
DFS ALGORITHM: 1. Start at source, mark visited 2. For each unvisited neighbor: - Recursively DFS - Backtrack when stuck Time: O(V + E) Space: O(V)

Use cases

  • Cycle detection: Back edge ‚Üí cycle
  • Topological sort: Reverse post-order
  • Strongly connected components: Kosaraju/Tarjan
  • Path finding: Any path (not shortest)
BFS vs DFS: BFS for shortest path, DFS for exhaustive exploration. BFS uses more memory (full level), DFS uses less (just path).

Dijkstra's algorithm

Greedy: always expand the unvisited vertex with smallest distance.

PythonDijkstra's Algorithm
def dijkstra(graph, start):
    dist = {v: float('inf') for v in graph}
    dist[start] = 0
    pq = [(0, start)]  # (distance, vertex)
    
    while pq:
        d, u = heapq.heappop(pq)
        if d > dist[u]: continue
        for v, w in graph[u]:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                heapq.heappush(pq, (dist[v], v))
    return dist
Dijkstra fails with negative edges! Use Bellman-Ford for graphs with negative weights.

Kruskal's algorithm

  • Sort all edges by weight
  • For each edge (u, v, w) in sorted order:
  • If u and v in different components, add edge
  • Stop when V-1 edges added
