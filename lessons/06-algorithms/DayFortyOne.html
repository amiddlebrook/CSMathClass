Section 1

Tree Terminology

A tree is a non-linear data structure. It has a single Root node and branches out.

  • Root: The top node (no parent).
  • Child: A node directly connected below another.
  • Leaf: A node with no children.
  • Height: The length of the longest path to a leaf.
Section 2

Binary Search Trees (BST)

In a BST, every node follows this rule:

$$ \text{Left Child} < \text{Parent} < \text{Right Child} $$

Efficiency: Finding a value takes $O(\log n)$ (assuming the tree is balanced). It's like a sorted array but with fast insertions.

Section 3

The Heap (Priority Queue)

A Heap is a Complete Binary Tree that satisfies the Heap Property. It is NOT sorted perfectly.

Max-Heap

Parent $\ge$ Children.

Root is the Largest element.

Min-Heap

Parent $\le$ Children.

Root is the Smallest element.

Heaps are used to implement Priority Queues (e.g., task scheduler, Dijkstra's algorithm).

Section 4

Heap Operations

Heaps are typically implemented using an Array, not pointers!

Child indices for node $i$: $2i+1$ (left), $2i+2$ (right).

  • Peek (Get Min/Max): $O(1)$. Just look at `arr[0]`.
  • Push (Insert): Add to end, then Sift Up (swap with parent). $O(\log n)$.
  • Pop (Remove Root): Swap root with last item, remove last, then Sift Down (swap with children). $O(\log n)$.
Section 5

Python: Min-Heap

Python has a built-in `heapq` module (Min-Heap by default).

Python Using heapq 
import heapq

# Create a heap
min_heap = []

# Push items (O(log n))
heapq.heappush(min_heap, 10)
heapq.heappush(min_heap, 5)
heapq.heappush(min_heap, 20)

print(min_heap) 
# Output might be [5, 10, 20] (heap order, not sorted!)

# Peek (O(1))
print(f"Smallest: {min_heap[0]}") # 5

# Pop (O(log n))
smallest = heapq.heappop(min_heap)
print(f"Removed: {smallest}") # 5
print(min_heap) # [10, 20]
