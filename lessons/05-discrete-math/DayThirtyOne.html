Section 1

The Anatomy of a Tree

A Tree is a graph that is **Connected** and **Acyclic** with a single **Root** node.

Key Terms

Root: The single top node.

Child: A node directly below another.

Leaf: A node with NO children.

Metrics

Depth: Distance from Root (Root is 0).

Height: Distance from deepest Leaf (Leaf is 0).

Section 3

Binary Search Trees (BST)

A binary tree where every node follows a strict ordering rule.

The BST Property:
For any node $X$:
All values in **Left Subtree** < $X$.
All values in **Right Subtree** > $X$.

Search Complexity

Balanced: $O(\log N)$ (Halving search space).

Skewed: $O(N)$ (Basically a Linked List).

Self-Balancing

Real databases use AVL or Red-Black trees to stay balanced automatically.

Section 4

Tree Traversal

How do you visit every node? There are 3 main Depth-First strategies.

Pre-Order

(Root, Left, Right)

Use: Copying a tree.

In-Order

(Left, Root, Right)

Use: Getting sorted data from a BST.

Post-Order

(Left, Right, Root)

Use: Deleting a tree (Children first).

Section 5

Advanced Trees

Min-Heaps

Root is always the Minimum.

Used for Priority Queues.

Tries (Prefix Trees)

Stores strings by character.

Used for Autocomplete.

The B-Tree:
Used by SQL Databases/Filesystems.
Nodes have thousands of children to match Disk Block sizes.
Minimizes slow disk reads.
Section 6

Python: Recursion

Trees are recursive structures. A folder contains folders. A JSON object contains objects.

Python 1. Recursive Directory Size 
# A mock file system (JSON)
fs = {
    "name": "root",
    "type": "folder",
    "children": [
        {"name": "file1.txt", "type": "file", "size": 10},
        {"name": "documents", "type": "folder", "children": [
            {"name": "resume.pdf", "type": "file", "size": 20},
            {"name": "notes.txt", "type": "file", "size": 5}
        ]}
    ]
}

def get_total_size(node):
    # Base Case: It's a file
    if node["type"] == "file":
        return node.get("size", 0)
    
    # Recursive Step: Sum directly + children
    total = node.get("size", 0) # Some folders have metadata size
    for child in node.get("children", []):
        total += get_total_size(child)
    
    return total

print(f"Total Size: {get_total_size(fs)} MB")
# 10 + 20 + 5 = 35
Python 2. Parsing JSON (The Web Tree) 
import json

data = """
{
    "id": 1,
    "name": "Alice",
    "posts": [
        {"id": 101, "title": "Hello World"},
        {"id": 102, "title": "Trees are cool"}
    ]
}
"""

root = json.loads(data)

def print_keys(node, depth=0):
    indent = "  " * depth
    if isinstance(node, dict):
        for key, value in node.items():
            print(f"{indent}{key}")
            print_keys(value, depth + 1)
    elif isinstance(node, list):
        for item in node:
            print_keys(item, depth + 1)

print("JSON Keys Structure:")
print_keys(root)
