Section 1

Core Concept

Two key properties: Optimal Substructure + Overlapping Subproblems
OPTIMAL SUBSTRUCTURE: Optimal solution contains optimal solutions to subproblems OVERLAPPING SUBPROBLEMS: Same subproblems solved multiple times → Cache/memoize results to avoid recomputation
DP vs Greedy: Greedy makes one choice per step. DP explores all choices and picks optimal.
Section 2

Top-Down vs Bottom-Up

Top-Down (Memoization)

• Recursive approach • Cache results as computed • Natural to write • May have stack overhead

Bottom-Up (Tabulation)

• Iterative approach • Build table from base cases • Often more efficient • Explicit iteration order
Python Fibonacci: Both approaches 
# Top-down (memoization)
def fib_memo(n, memo={}):
    if n <= 1: return n
    if n not in memo:
        memo[n] = fib_memo(n-1) + fib_memo(n-2)
    return memo[n]

# Bottom-up (tabulation)
def fib_tab(n):
    if n <= 1: return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]
Section 3

Classic Examples

CLASSIC DP PROBLEMS: • Fibonacci: dp[i] = dp[i-1] + dp[i-2] • Climbing Stairs: same as Fib • Coin Change: dp[i] = min(dp[i-coin]) + 1 • Longest Common Subsequence (LCS) • Longest Increasing Subsequence (LIS) • 0/1 Knapsack: dp[i][w] = max(take, skip) • Edit Distance: min insertions/deletions/subs
Python Coin Change 
def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for i in range(1, amount + 1):
        for coin in coins:
            if coin <= i:
                dp[i] = min(dp[i], dp[i - coin] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1
Section 4

Common Patterns

DP PATTERNS: 1. Linear DP: dp[i] depends on dp[i-1], dp[i-2], etc. 2. Grid DP: dp[i][j] for 2D problems 3. Interval DP: dp[i][j] for ranges 4. Tree DP: DP on tree structures 5. Bitmask DP: State as bitmask for subsets SPACE OPTIMIZATION: If dp[i] only needs dp[i-1], use just 2 variables!
