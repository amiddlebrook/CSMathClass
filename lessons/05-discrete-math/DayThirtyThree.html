Section 1

Propositions

A Proposition is a sentence that is definitely True or definitely False.

Valid Propositions

• "The sky is blue." (True)

• "2 + 2 = 5." (False)

Not Propositions

• "Hello." (No truth value)

• "x + 2 = 5." (Values depends on x)

Section 2

Logical Operators (Gates)

Every computer chip is built from these atomic operations. They are the "Lego bricks" of computation.

Name Symbol Logic Meaning
AND $p \land q$ True only if BOTH are True. "Series Circuit"
OR $p \lor q$ True if AT LEAST ONE is True. "Parallel Circuit"
NOT $\neg p$ Flips the value (T $\to$ F). "Inverter"
XOR $p \oplus q$ True if inputs are DIFFERENT. "Exclusive OR"
Section 3

Implications (Conditionals)

The core of mathematical proofs. The statement $P \implies Q$ ("If P, then Q").

The Promise Analogy:
"If it rains ($P$), I will bring an umbrella ($Q$)."
The promise is broken (False) ONLY if it rains ($P=\top$) and I have NO umbrella ($Q=\bot$).
If it doesn't rain ($P=\bot$), the promise stands (Vacuously True).

Derived Forms

Converse: $Q \implies P$ (Not equivalent)

Inverse: $\neg P \implies \neg Q$ (Not equivalent)

Contrapositive: $\neg Q \implies \neg P$ (Equivalent!)

Biconditional ($P \iff Q$)

"P if and only if Q".

True when P and Q have the SAME truth value.

Section 4

Logical Equivalence & De Morgan

Two statements are Logically Equivalent ($\equiv$) if they have identical Truth Tables.

De Morgan's Laws (Breaking Loops)

How do you negate a group?

1. $\neg(A \land B) \equiv (\neg A) \lor (\neg B)$

2. $\neg(A \lor B) \equiv (\neg A) \land (\neg B)$

Code Tip: `!(fast && cheap)` == `!fast || !cheap`.

Section 5

Predicates & Quantifiers

Standard logic ($P \lor Q$) is static. Predicate Logic introduces variables.

Universal Quantifier ($\forall$)

"For All" / "Every".

$\forall x, x^2 \ge 0$ (For every real number x, x squared is non-negative).

Existential Quantifier ($\exists$)

"There Exists" / "For Some".

$\exists x, x^2 = 25$ (There exists a number x such that x squared is 25).

Negating Quantifiers:
$\neg \forall x, P(x) \equiv \exists x, \neg P(x)$ (To disprove "All", find "One Counterexample").
$\neg \exists x, P(x) \equiv \forall x, \neg P(x)$ (To disprove "Exists", show "None").
Section 6

Circuits & Applications

Logic maps directly to hardware.

Hardware The Half Adder 
  Inputs: A, B  ->  Outputs: Sum (S), Carry (C)

  1. Sum   = A XOR B  (1+0=1, 1+1=0)
  2. Carry = A AND B  (1+1 produces Carry)
Section 7

Python: Bitwise & Logic

Integers are just 32 booleans in a trenchcoat. We can use "Bit Masks" to store efficient settings.

Python 1. Bitwise Permissions System 
# Define Flags (Powers of 2)
READ    = 0b001 # 1
WRITE   = 0b010 # 2
EXECUTE = 0b100 # 4

# User Permissions (Read + Execute)
user_perms = READ | EXECUTE  # 001 | 100 = 101 (5)

print(f"Permissions Value: {user_perms}")

# Check Permission (Masking)
# We use AND to isolate the bit we care about
if user_perms & WRITE:
    print("User can write.")
else:
    print("User CANNOT write.")

# Toggle Permission (XOR)
user_perms = user_perms ^ WRITE # flip the write bit
print(f"New Permissions: {bin(user_perms)}")
Python 2. Short-Circuit Evaluation 
def expensive_check():
    print("Checking database...")
    return True

# CASE 1: OR
# If first is True, second is skipped (Short Circuit)
if True or expensive_check():
    print("Done (Case 1)")

# CASE 2: AND
# If first is False, second is skipped
if False and expensive_check():
    print("Done (Case 2)")
