Section 1

Sets: The Atomic Unit

In formal mathematics, everything is a Set. Numbers are constructed from sets. Functions are sets of pairs. A Set is a collection of unique objects, order does not matter.

A = \{1, 2, 3\}
B = \{a, b, c\}
Empty Set: \emptyset = \{\}
Analogy: The Bag
Think of a set as a bag of marbles.
• Swaking the bag (reordering) doesn't change the set.
• Putting the same marble in twice doesn't change the set (Uniqueness).

Subsets & Power Sets

$A \subseteq B$ means every element of A is also in B.

The Power Set $P(A)$ is the set of all subsets of A.
If $A = \{1, 2\}$, then $P(A) = \{\emptyset, \{1\}, \{2\}, \{1, 2\}\}$.

The Exponential Explosion
The size of the power set is $|P(A)| = 2^{|A|}$.
If a set has 30 elements, its power set has 1 billion elements.
This is why brute-force algorithms (checking every subset) are usually $O(2^n)$.
Section 2

Set Operations

These operations form the basis of Boolean Logic and SQL Queries.

Union ($A \cup B$)

Everything in A OR B.

A = {1, 2}
B = {2, 3}
A ∪ B = {1, 2, 3}

Intersection ($A \cap B$)

Everything in A AND B.

A = {1, 2}
B = {2, 3}
A ∩ B = {2}

Difference ($A - B$)

In A but NOT in B.

A - B = {1}

Complement ($A^c$)

Everything NOT in A (within Universe U).

Section 3

The Cartesian Product

Sets have no order. But often we need coordinates (x, y) where order matters. We build these using the Cartesian Product.

A \times B = \{(a, b) \mid a \in A, b \in B\}

The Dimensions

If $A = \{1, 2\}$ and $B = \{x, y, z\}$:
$A \times B = \{(1,x), (1,y), (1,z), (2,x), (2,y), (2,z)\}$

Size = $|A| \cdot |B| = 2 \cdot 3 = 6$.

The Space

$\mathbb{R} \times \mathbb{R} = \mathbb{R}^2$ (The 2D Plane).
$\mathbb{R} \times \mathbb{R} \times \mathbb{R} = \mathbb{R}^3$ (3D Space).

SQL Connection: Cross Join
Selecting from two tables without a WHERE clause creates a Cartesian Product.
If Table 1 has 1,000 rows and Table 2 has 1,000 rows, the result has 1,000,000 rows.
This is a common way to crash a database.
Section 4

Relations & Database Tables

A Relation is simply a subset of a Cartesian Product.
In math we say: $R \subseteq A \times B$.
In Databases we say: "A Table".

Example:
Let $U$ be a set of Users. Let $P$ be a set of Products.
The "Purchased" relation is a list of pairs $(u, p)$ where user $u$ bought product $p$.
This creates a table: Purchases(User, Product).

Properties of Relations

Relations on a set $A$ (subsets of $A \times A$) can have special properties:

Reflexive

Everyone is related to themselves.

$(a, a) \in R$ for all $a$.

Ex: "Is equal to", "Is as tall as".

Symmetric

Relationships go both ways.

If $(a, b) \in R$, then $(b, a) \in R$.

Ex: "Is sibling of", "Is connected to".

Transitive

The chain rule.

If $(a, b) \in R$ and $(b, c) \in R$, then $(a, c) \in R$.

Ex: "Is taller than", "Is ancestor of".

Equivalence Relations:
If a relation is Reflexive, Symmetric, AND Transitive, it defines "Equality".
It partitions the set into "Equivalence Classes".
Ex: "Has the same birthday as".
Section 5

Functions as Relations

A Function $f: A \to B$ is just a Relation with a special rule:
Vertical Line Test: Every input $a$ maps to EXACTLY ONE output $b$.

Injective (1-to-1)

No two inputs go to the same output.

"Horizontal Line Test passes".

Ex: $f(x) = e^x$.

Surjective (Onto)

Every possible output gets hit at least once.

"Range = Co-domain".

Ex: $f(x) = x^3$.

Bijective

Both Injective and Surjective.

Perfect pairing. Implies $|A| = |B|$.

Ex: $f(x) = x$.

Section 6

Python: Similarity & Joins

Python's `set` is heavily optimized (Hash Table). Use it for $O(1)$ lookups.

Python Jaccard Similarity 
def jaccard_similarity(set_a, set_b):
    """
    Measures how similar two sets are (0 to 1).
    J(A,B) = |A ∩ B| / |A ∪ B|
    """
    intersection = len(set_a.intersection(set_b))
    union = len(set_a.union(set_b))
    return intersection / union

# Example: Document Similarity
doc1_words = {"data", "science", "math", "code"}
doc2_words = {"data", "science", "business", "money"}

score = jaccard_similarity(doc1_words, doc2_words)
print(f"Similarity: {score:.2f}") 
# Intersection: {data, science} (2)
# Union: {data, science, math, code, business, money} (6)
# Score: 2/6 = 0.33
Python Implementing an Inner Join 
# Users Data (ID -> Name)
users = {
    1: "Alice",
    2: "Bob",
    3: "Charlie"
}

# Orders Data (List of dicts)
orders = [
    {"user_id": 1, "item": "Laptop"},
    {"user_id": 1, "item": "Mouse"},
    {"user_id": 2, "item": "Monitor"}
    # Charlie ordered nothing
]

# INNER JOIN (Intersection of Keys)
joined_data = []
for order in orders:
    uid = order["user_id"]
    if uid in users:  # O(1) Lookup
        # Combine the data (Tuple)
        row = (users[uid], order["item"])
        joined_data.append(row)

print("Joined Data:")
for row in joined_data:
    print(row)
# Expect:
# ('Alice', 'Laptop')
# ('Alice', 'Mouse')
# ('Bob', 'Monitor')
