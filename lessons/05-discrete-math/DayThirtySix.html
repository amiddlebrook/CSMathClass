Section 1

Anatomy of an FSM

A Deterministic Finite Automaton (DFA) is formally a 5-tuple: $M = (Q, \Sigma, \delta, q_0, F)$

The Components

  • $Q$: Finite set of states.
  • $\Sigma$: Finite set of input symbols (Alphabet).
  • $\delta$: Transition function ($Q \times \Sigma \to Q$).
  • $q_0$: Start state ($q_0 \in Q$).
  • $F$: Set of accept states ($F \subseteq Q$).

State Transition Table

State Input 0 Input 1
$q_0$ (Start) $q_0$ $q_1$
$q_1$ (Accept) $q_1$ $q_0$

Example: Binary string with odd number of 1s.

Section 2

DFA vs NFA

What if the machine could "guess"?

DFA (Deterministic)

One path per input.

Easy to implement in hardware.

Analogy: Following a recipe step-by-step.

NFA (Non-Deterministic)

Multiple possible next states (or none).

Accepts if AT LEAST ONE path works.

Analogy: A maze where you can clone yourself at every fork.

Power Set Construction: Any NFA can be converted into an equivalent DFA. However, the DFA may have up to $2^n$ states (Exponential blowup).
Section 3

Regex & Lexical Analysis

FSMs are the "engine" behind string pattern matching.

Regex: /a+b/

1. Start State.

2. Must see 'a'. Transition to State 1.

3. If see 'a' again, stay in State 1 (The '+' logic).

4. If see 'b', move to Accept State.

Compiler Lexers:
Compilers use FSMs to group characters into "Tokens".
Example: `int x = 5;` -> `[KEYWORD_INT, IDENTIFIER_X, OPERATOR_ASSIGN, CONSTANT_5, SEMICOLON]`.
Section 4

Real-World Latches

FSMs model hardware and sequential logic.

Turnstile

• States: Locked, Unlocked.

• Inputs: Coin, Push.

• Logic: Locked + Coin -> Unlocked. Unlocked + Push -> Locked.

Vending Machine

• States: Idle, Collecting, ItemSelected.

• Inputs: InsertCoin, PressButton.

• Logic: Transitions depend on the current balance ($\delta$).

The Limit of FSMs: FSMs have "Finite Memory". They CANNOT recognize languages that require counting to an unknown number, like Balanced Parentheses `(())`. (For that, you need a Pushdown Automaton with a stack).
Section 5

Python: DFA Simulator

Building a generic DFA engine in Python.

Python Generic DFA Engine 
class DFA:
    def __init__(self, states, alphabet, transitions, start_state, accept_states):
        self.states = states
        self.alphabet = alphabet
        self.transitions = transitions # Dict: { (state, input): next_state }
        self.current_state = start_state
        self.accept_states = accept_states

    def reset(self):
        self.current_state = "START" # Assuming start_state name

    def process(self, input_string):
        self.current_state = self.current_state # Simplified for reuse
        for char in input_string:
            if char not in self.alphabet:
                return False # Invalid input
            self.current_state = self.transitions.get((self.current_state, char))
            if self.current_state is None:
                return False # Crash/Dead state
        return self.current_state in self.accept_states

# Example: Accept strings with EVEN number of 1s
# Alphabet = {0, 1}
# Q = {S0, S1}
# F = {S0}
parity_checker = DFA(
    states={"S0", "S1"},
    alphabet={"0", "1"},
    transitions={
        ("S0", "0"): "S0", ("S0", "1"): "S1",
        ("S1", "0"): "S1", ("S1", "1"): "S0"
    },
    start_state="S0",
    accept_states={"S0"}
)

test_strings = ["1010", "111", "000", "10101"]
for s in test_strings:
    result = parity_checker.process(s)
    # Reset current state for next test
    parity_checker.current_state = "S0" 
    print(f"String '{s}' accepted? {result}")
