Section 0

Why Relations?

Relations are everywhere. Equality, divisibility, subsets, inheritance—all are relations with special properties.

Applications

  • Databases: Keys, dependencies
  • Type systems: Subtyping is a partial order
  • Sorting: Topological sort uses partial orders
  • Quotient structures: Equivalence classes
Section 1

Equivalence Relations

Definition

Equivalence Relation ∼
Reflexive + Symmetric + Transitive
1. Reflexive: a ∼ a for all a 2. Symmetric: a ∼ b → b ∼ a 3. Transitive: a ∼ b and b ∼ c → a ∼ c

Equivalence classes

[a] = {x : x ∼ a} Fundamental theorem: Equivalence relation ↔ Partition of set The equivalence classes partition the set into disjoint subsets that cover everything.

Examples

• Equality (=): Each element is its own class • Mod n: a ∼ b iff n | (a-b) Classes: [0], [1], ..., [n-1] • Same parity: {evens}, {odds}
Section 2

Partial Orders

Definition

Partial Order ≤
Reflexive + Antisymmetric + Transitive
1. Reflexive: a ≤ a for all a 2. Antisymmetric: a ≤ b and b ≤ a → a = b 3. Transitive: a ≤ b and b ≤ c → a ≤ c

Key concepts

• Comparable: a ≤ b or b ≤ a • Incomparable: neither a ≤ b nor b ≤ a • Total order: all pairs comparable (e.g., ≤ on ℝ) • Hasse diagram: Visual representation

Examples

• Divisibility: a | b on positive integers 1 | everything, not all pairs comparable • Subset: A ⊆ B on power set • Dependency: Task A before Task B
Section 3

Applications

Topological sort

Given partial order (dependencies), find total order. Used in: build systems, course prerequisites, package managers. Valid iff no cycles (DAG).

Quotient structures

ℤ/nℤ = integers mod n (equivalence classes) Cosets in group theory Quotient spaces in topology
