Why Trig Matters in CS

In high school, Trigonometry feels like a collection of triangle identities. In Computer Science, it is the Mathematics of Periodicity. If something repeats, spins, or oscillates, it uses Trig.

Graphics & Robotics

Calculating the "LookAt" vector for a camera, rotating a 3D model, or calculating the joint angles of a robotic arm (Inverse Kinematics).

Data Compression

JPEG images and MP3 audio don't store raw pixels or sounds. They store Trig coefficients (Discrete Cosine Transform) to save space.

GPS & Navigation

Global Positioning Systems use spherical trigonometry to triangulate your exact position on a curved earth using satellite signal timing.

Machine Learning

Cosine Similarity is the standard way to measure how similar two text documents or vectors are in a Recommendation System.

Industrial Fact: The Fast Fourier Transform (FFT) is estimated to be running on billions of devices every second, facilitating everything from your 5G connection to your noise-canceling headphones.
Section 1

The Unit Circle (Coordinates)

Forget SOH CAH TOA for a second. Think coordinates. In programming, we rarely have a physical triangle; we have a position and an angle.

The Definition:
Draw a circle of Radius 1 at origin (0,0).
Walk angle $\theta$ along the edge counter-clockwise.
‚Ä¢ The x-coordinate is $\cos(\theta)$.
‚Ä¢ The y-coordinate is $\sin(\theta)$.
üîÑ Analogy: The Radar Dish
Imagine a radar dish rotating.
‚Ä¢ At 0¬∞, it points East $(1, 0)$. Cosine is 1, Sine is 0.
‚Ä¢ At 90¬∞, it points North $(0, 1)$. Cosine is 0, Sine is 1.
‚Ä¢ As it spins, the $(x, y)$ values oscillate between -1 and 1.
The Pythagorean Identity:
Since any point on the circle forms a right triangle with hypotenuse 1: $$ \cos^2\theta + \sin^2\theta = 1 $$ In game engines, this is used to Normalize vectors. If you move diagonally, you don't want to move faster ($\sqrt{1^2+1^2} = 1.41$) than moving straight (1.0). Trig keeps your speed constant.
Section 2

Radians: The Natural Unit

Degrees (360¬∞) are an arbitrary human choice based on an ancient calendar. Radians are based on the circle itself.

Definition: One radian is the angle where the arc length equals the radius.
Since a full circle has circumference $2\pi r$, a full circle is $2\pi$ radians.
‚Ä¢ $180^\circ = \pi$ radians.
‚Ä¢ $90^\circ = \pi/2$ radians.
Programmer Trap: Almost every standard library (Python `math`, JS `Math`, C++ `cmath`) expects Radians.
Passing degrees into sin() will produce "voodoo" numbers that look right but are subtly wrong.

Python: Safe Conversion

PythonDegree to Radian
import math

deg = 45
rad = math.radians(deg) # Or deg * (math.pi / 180)

print(f"Sin(45 deg): {math.sin(rad):.4f}") # 0.7071
print(f"Wrong Sin: {math.sin(deg):.4f}")    # 0.8509 (interprets 45 as radians!)
Section 3

Waves (Amplitude & Frequency)

If you unroll the Unit Circle over time, you get waves.

Anatomy of a Sine Wave (The Code View)

$$ y(t) = A \cdot \sin(B(x - h)) + k $$

$A$: Amplitude (Scale)

In audio, this is Volume. In graphics, it determines how high/low an object bobs.

sprite.y = 100 * sin(t);

$B$: Frequency (Speed)

Related to $2\pi f$. Controls how fast the wave repeats.

sprite.x = cos(5 * t);

$h$: Phase (Offset)

Shifts the wave left or right. Used to sync animations.

val = sin(t + 1.57); // Shifted by 90¬∞

$k$: Vertical Shift (Center)

Moves the wave up or down on the screen.

y = sin(t) + screenCenter;

Note for Programmers: Math libraries usually take Radians, not Degrees.
Math.sin(90) interprets 90 as radians (~14 full rotations).
Math.sin(Math.PI / 2) returns 1.0 (Correct).
Section 4

Inverse Trig (The LookAt Function)

If you have the angle, you use sin() to get the position. If you have the position, you use Inverse Trig to get the angle.

ArcSine ($\sin^{-1}$)

Input: Ratio $(-1 \text{ to } 1)$

Output: Angle

ArcTangent ($\tan^{-1}$)

Input: Slope ($y/x$)

Output: Angle

The Programmer's Secret: atan2(y, x)
Standard $\tan^{-1}(y/x)$ has two failures:
1. Division by zero (if $x=0$).
2. Quadrant ambiguity (it can't tell the difference between $-y/-x$ and $y/x$).
Solution: Always use atan2. It handles the signs and the zeros for you.

Python: Making an Enemy Face the Player

PythonDirection Steering
import math

player_pos = (10, 5)
enemy_pos = (2, 2)

# Calculate relative vector
dx = player_pos[0] - enemy_pos[0] # 8
dy = player_pos[1] - enemy_pos[1] # 3

# Use atan2 to get the angle in radians
angle_rad = math.atan2(dy, dx)
angle_deg = math.degrees(angle_rad)

print(f"Enemy must rotate to: {angle_deg:.2f}¬∞")
Section 5

Identities & Physics Engines

Trigonometric identities aren't just for tests; they are Performance Optimizations for real-time systems.

Symmetry

$\sin(-x) = -\sin(x)$ (Odd)

$\cos(-x) = \cos(x)$ (Even)

Shift

$\sin(x + \pi/2) = \cos(x)$

Sine is just Cosine delayed!

Double Angle Identities (Graphics Hacks)

Calculating $\sin(2x)$ directly is slow in a shader. If you already have $\sin(x)$ and $\cos(x)$, you can use multiplication instead.

$$ \sin(2\theta) = 2 \sin(\theta) \cos(\theta) $$ $$ \cos(2\theta) = \cos^{2}(\theta) - \sin^{2}(\theta) $$
üèéÔ∏è Speed Tip: Multiplication is several orders of magnitude faster than a library sin() call. High-performance game engines use these identities to avoid redundant math across millions of pixels per frame.
Section 6

Fourier & Signal Compression

The Fourier Transform is a mathematical bridge. It allows us to view the world either as Time (what happened when) or Frequency (how fast it repeats).

The Big Idea:
ANY periodic signal (square wave, heartbeat, voice) can be built by adding up enough Sine waves of different frequencies.
üéº Analogy: The Smoothie (Revisited)
‚Ä¢ Time Domain: The Smoothie. You taste it all once.
‚Ä¢ Frequency Domain: The Ingredient List. You see exactly what went in.
If a sound has a high-pitched hiss, you can find that specific frequency in the "Recipe" and delete it to clean the audio.

JPEG & DCT (The Cosine Power)

JPEG images use a cousin of Fourier called the Discrete Cosine Transform (DCT). Instead of saving colors for every pixel, the computer saves the frequencies of color changes.

Compression Rule: Human eyes are bad at seeing high-frequency details. We save the low frequencies (general shapes) and throw away the high ones.
Section 7

Polar Coordinates (Radar)

Standard $(x, y)$ coordinates are "Grid-based". Polar coordinates $(r, \theta)$ are "Rotation-based".

Cartesian $(x, y)$

Good for: Square grids, UI layouts, Spreadsheet math.

Polar $(r, \theta)$

Good for: Radar, Particle explosions, Spinning wheels.

Conversion Formula

To convert Polar to Cartesian: $$ x = r \cdot \cos(\theta) $$ $$ y = r \cdot \sin(\theta) $$
Section 8

The Dot Product Connection

Why do we care about Cosine in Data Science? Similarity. How well do two vectors "line up"?

$$ \vec{A} \cdot \vec{B} = |\vec{A}| |\vec{B}| \cos(\theta) $$ If normalized (length 1): $$ \vec{A} \cdot \vec{B} = \cos(\theta) $$
Cosine Similarity:
‚Ä¢ If $\theta = 0$ (Parallel), Dot = 1. (Identical).
‚Ä¢ If $\theta = 90$ (Perpendicular), Dot = 0. (Zero overlap).
‚Ä¢ If $\theta = 180$ (Opposite), Dot = -1. (Polar opposites).

Python: Recommendation Logic

PythonVector Similarity
import numpy as np

user_a = np.array([1, 0, 1]) # Likes Sci-Fi and Action
user_b = np.array([1, 0, 0.9]) # Likes Sci-Fi mostly

# Calculate similarity
dot = np.dot(user_a, user_b)
norm = np.linalg.norm(user_a) * np.linalg.norm(user_b)
similarity = dot / norm

print(f"Similarity Score: {similarity:.4f}") # 0.999 (High match!)

Section 8.5: Field of View (FOV) Detection

In game development, how do you know if an enemy can "see" the player? You use the Dot Product to calculate the angle between the enemy's Forward Vector and the Vector to Player.

The FOV Logic:
1. Calculate vector $\vec{V} = \text{PlayerPos} - \text{EnemyPos}$.
2. Normalize $\vec{V}$ and $\vec{Forward}$.
3. $\text{dot} = \vec{V} \cdot \vec{Forward}$.
4. If $\text{dot} > \cos(\text{FOV\_Angle}/2)$, the player is visible!
PythonFOV Check
import math

def is_in_fov(enemy_fwd, enemy_pos, player_pos, fov_deg):
    # Vector from enemy to player
    v = (player_pos[0] - enemy_pos[0], player_pos[1] - enemy_pos[1])
    dist = math.sqrt(v[0]**2 + v[1]**2)
    
    if dist == 0: return True
    v_norm = (v[0]/dist, v[1]/dist) # Normalize
    
    # Dot Product
    dot = v_norm[0]*enemy_fwd[0] + v_norm[1]*enemy_fwd[1]
    
    # Threshold (half the total FOV arc)
    threshold = math.cos(math.radians(fov_deg / 2))
    
    return dot > threshold

# Enemy looking right (1, 0), player slightly above and right
print(is_in_fov((1, 0), (0, 0), (5, 2), 90)) # True (Inside 45¬∞ half-angle)
Section 9

The Law of Cosines & GPS

The Pythagorean Theorem only works for right triangles. For everything else, we use the Law of Cosines.

$$ c^2 = a^2 + b^2 - 2ab \cos(\theta) $$
üìç GPS Application: The Haversine Formula
Because the Earth is a sphere, the "Straight Line" distance between two points (Latitude/Longitude) isn't a straight line‚Äîit's a Great Circle arc. The Haversine formula is essentially the Law of Cosines applied to a sphere.

Interpolation (LERP) vs SLERP

When animating a camera rotation, a simple Linear Interpolation (LERP) makes the movement look "robotic" and inconsistent. Spherical Linear Interpolation (SLERP) uses trigonometry to ensure the constant angular velocity required for premium cinematography in 3D engines.

Section 10

Performance & Approximations

Calculating sin(x) is expensive for a CPU. It's not a single operation like addition. How do computers actually do it?

Taylor Series Approximation

Computers approximate trig functions using infinite polynomials. $$\sin(x) \approx x - \frac{x^3}{3!} + \frac{x^5}{5!}$$ The more terms you calculate, the more accurate the "fit" becomes for your hardware.

Lookup Tables (LUT)

In legacy game consoles (NES, GameBoy), the CPU was too slow for high-precision math. Devs pre-calculated 256 or 360 values of Sine and saved them in a fast Read-Only Array. val = sineTable[angle];

Python Audio Synthesis

Let's make some noise (literally).

Python Generating a Tone 
import numpy as np
import simpleaudio as sa # Requires: pip install simpleaudio

# Parameters
fs = 44100  # 44100 samples per second
seconds = 3 

# Generate time array
t = np.linspace(0, seconds, seconds * fs, False)

# Generate Sine Wave (440Hz - Musical Note 'A')
note = np.sin(440 * 2 * np.pi * t)

# Normalize to 16-bit audio range
audio = note * (2**15 - 1) / np.max(np.abs(note))
audio = audio.astype(np.int16)

# Play
play_obj = sa.play_buffer(audio, 1, 2, fs)
play_obj.wait_done()

Trig Gotchas for Programmers

‚ö†Ô∏è Common Pitfalls:
‚Ä¢ Angle Wrapping: If you add 10 degrees to 355, you get 365. Many systems expect 5. Always use angle % 360 (or $2\pi$) for safety.
‚Ä¢ The atan vs atan2 Trap: Never use atan(y/x). It will crash if $x=0$ and lose the sign information (quadrant). Always use atan2(y, x).
‚Ä¢ Float Precision: sin(pi) rarely returns exactly 0.0 in code; it returns 1.22e-16. Always use a small epsilon (e.g., abs(val) < 0.0001) when checking for zero.
