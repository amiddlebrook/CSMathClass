Section 0

Why Recurrences Matter

The pattern: Recursive code leads to recurrence relations. To find the Big-O of recursive algorithms, you must solve these recurrences.

From code to recurrence

PythonMerge sort
def merge_sort(arr):
    if len(arr) <= 1:
        return arr           # O(1) base case
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])   # T(n/2)
    right = merge_sort(arr[mid:])  # T(n/2)
    return merge(left, right)      # O(n)

# Recurrence: T(n) = 2T(n/2) + O(n)

The goal

Convert T(n) = 2T(n/2) + n into T(n) = O(n log n). That's what this day teaches!

Section 1

Recurrence Basics

Anatomy of a recurrence

T(n) = a·T(n/b) + f(n) Where: • T(n) = time to solve problem of size n • a = number of subproblems • n/b = size of each subproblem • f(n) = work done outside recursive calls (divide + combine)

Common patterns

Algorithm Recurrence Solution
Binary search T(n) = T(n/2) + O(1) O(log n)
Merge sort T(n) = 2T(n/2) + O(n) O(n log n)
Naive multiply T(n) = 4T(n/2) + O(n) O(n²)
Karatsuba T(n) = 3T(n/2) + O(n) O(n^1.58)
Linear scan T(n) = T(n-1) + O(1) O(n)
Section 2

The Unrolling Method

The technique

Unrolling: Repeatedly substitute the recurrence into itself until you see a pattern, then sum up all the work.

Example: T(n) = 2T(n/2) + n

Unroll step by step: T(n) = 2T(n/2) + n = 2[2T(n/4) + n/2] + n = 4T(n/4) + n + n = 4T(n/4) + 2n = 4[2T(n/8) + n/4] + 2n = 8T(n/8) + n + 2n = 8T(n/8) + 3n Pattern: After k steps: = 2ᵏ T(n/2ᵏ) + kn Stop when n/2ᵏ = 1, so k = log₂ n: = n·T(1) + n log n = O(n log n) ✓

Example: T(n) = T(n/2) + 1 (Binary search)

T(n) = T(n/2) + 1 = T(n/4) + 1 + 1 = T(n/8) + 1 + 1 + 1 ... = T(n/2ᵏ) + k When n/2ᵏ = 1, k = log n: = T(1) + log n = O(log n) ✓
Section 3

Guess and Check (Substitution)

The technique

Guess and Check:
1. Guess the answer (based on intuition or unrolling)
2. Prove by induction that your guess works
3. Find the constant that makes it valid

Example: T(n) = 2T(n/2) + n

Guess: T(n) ≤ cn log n for some constant c. Prove by induction: Assume T(k) ≤ ck log k for all k < n. T(n)=2T(n/2) + n ≤ 2 · c(n/2)log(n/2) + n [by I.H.]=cn log(n/2) + n=cn log n - cn log 2 + n=cn log n - cn + n=cn log n + n(1 - c) For this ≤ cn log n, we need: n(1 - c) ≤ 0 c ≥ 1 So T(n) ≤ cn log n for c ≥ 1. ✓ Therefore T(n)=O(n log n).
Common mistake: Guessing T(n) = O(n) for T(n) = 2T(n/2) + n doesn't work because the substitution doesn't go through. Always verify with induction!
Section 4

The Master Theorem

The theorem

Master Theorem
T(n) = aT(n/b) + f(n)

Compare f(n) with n^(log_b a):

Case 1: f(n) = O(n^(log_b a - ε)) for some ε > 0 → T(n) = Θ(n^(log_b a)) "Recursion dominates" Case 2: f(n) = Θ(n^(log_b a)) → T(n) = Θ(n^(log_b a) · log n) "Balanced work" Case 3: f(n) = Ω(n^(log_b a + ε)) for some ε > 0 AND a·f(n/b) ≤ c·f(n) for c < 1 → T(n)=Θ(f(n)) "Non-recursive work dominates"

Quick reference

Recurrence a b f(n) n^(log_b a) Case Result
T(n) = 2T(n/2) + n 2 2 n n 2 O(n log n)
T(n) = 2T(n/2) + 1 2 2 1 n 1 O(n)
T(n) = 2T(n/2) + n² 2 2 n² n 3 O(n²)
T(n) = T(n/2) + 1 1 2 1 1 2 O(log n)
T(n) = 4T(n/2) + n 4 2 n n² 1 O(n²)
T(n) = 3T(n/2) + n 3 2 n n^1.58 1 O(n^1.58)

Step-by-step Master Theorem

Given T(n) = 4T(n/2) + n: 1. Identify: a = 4, b = 2, f(n) = n 2. Compute n^(log_b a) = n^(log_2 4) = n^2 3. Compare f(n) = n vs n^2: n = O(n^(2-ε)) for ε = 1 ✓ 4. Case 1 applies: T(n) = Θ(n²)
Section 5

Common Algorithm Examples

Binary Search: T(n) = T(n/2) + O(1)

a = 1, b = 2, f(n) = 1 n^(log_2 1) = n^0 = 1 f(n) = Θ(1) = Θ(n^0) → Case 2 T(n) = Θ(1 · log n) = Θ(log n)

Merge Sort: T(n) = 2T(n/2) + O(n)

a = 2, b = 2, f(n) = n n^(log_2 2) = n^1 = n f(n) = Θ(n) → Case 2 T(n) = Θ(n log n)

Strassen's Matrix Multiply: T(n) = 7T(n/2) + O(n²)

a = 7, b = 2, f(n) = n² n^(log_2 7) ≈ n^2.81 f(n) = n² = O(n^(2.81-ε)) for ε ≈ 0.81 → Case 1 T(n) = Θ(n^2.81) ≈ Θ(n^2.81)

Fibonacci (naive): T(n) = T(n-1) + T(n-2) + O(1)

Master Theorem doesn't apply! The problem size doesn't divide by a constant. Use unrolling or tree method:

T(n) ≈ 2^n (exponential) because the recursion tree has 2^n nodes.
