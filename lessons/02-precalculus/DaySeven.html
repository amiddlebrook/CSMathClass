Section 0

Why Growth Rates Matter

The key insight: The difference between polynomial and exponential is the difference between "slow but tractable" and "impossible at scale."

A concrete example

Suppose each operation takes 1 microsecond (10⁻⁶ seconds):

n n² (polynomial) 2ⁿ (exponential)
10 100 μs 1 ms
20 400 μs 1 second
30 900 μs 18 minutes
40 1.6 ms 12.7 days
50 2.5 ms 35.7 years
100 10 ms 4 × 10²² years

Where you'll see this

  • Algorithm Design: Choosing O(n log n) sort over O(n²)
  • Cryptography: Security relies on exponential hardness
  • NP-Complete Problems: No known polynomial solutions
  • System Scaling: Will your app survive 10× growth?
  • Machine Learning: Training time vs dataset size
Section 1

Polynomial Growth

Definition

A function f(n) has polynomial growth if: f(n) = O(nᵏ) for some constant k Examples: n → O(n) linear n² → O(n²) quadratic n³ → O(n³) cubic 5n² + 3n + 7 → O(n²) (highest power dominates)

Key property: Doubling input

When input doubles (n → 2n), polynomial functions grow by a constant factor:

• Linear (n): time doubles (×2)
• Quadratic (n²): time quadruples (×4)
• Cubic (n³): time octuples (×8)

Common polynomial algorithms

Examples
O(1)      - Array access, hash table lookup
O(log n)  - Binary search
O(n)      - Linear search, single loop
O(n log n)- Merge sort, heap sort
O(n²)     - Bubble sort, nested loops
O(n³)     - Matrix multiplication (naive)
Section 2

Exponential Growth

Definition

A function f(n) has exponential growth if: f(n) = O(cⁿ) for some constant c > 1 Examples: 2ⁿ - Doubles with each increment 3ⁿ - Triples with each increment 1.5ⁿ - Still exponential!

Key property: Adding to input

Critical difference: When input increases by 1 (n → n+1), exponential functions multiply by a constant!

• 2ⁿ⁺¹ = 2 × 2ⁿ → time doubles for each +1
• 3ⁿ⁺¹ = 3 × 3ⁿ → time triples for each +1

Why exponential is intractable

For 2ⁿ with n = 300: 2³⁰⁰ ≈ 2 × 10⁹⁰ The universe has ~10⁸⁰ atoms. No computer can ever compute this many operations.

Common exponential problems

Examples
O(2ⁿ)  - Naive recursive Fibonacci
       - Subset enumeration (power set)
       - Brute-force password cracking

O(n!)  - Permutation enumeration
       - Traveling Salesman (brute force)
       - Even worse than 2ⁿ!
Section 3

Polynomial vs Exponential

The fundamental theorem

For any k and c > 1:

nᵏ = o(cⁿ)

Every polynomial is eventually dominated by every exponential. No matter how large k is, 2ⁿ will eventually surpass nᵏ.

Crossover points

When does 2ⁿ overtake n¹⁰? For small n, n¹⁰ > 2ⁿ: n=10: 10¹⁰ = 10 billion, 2¹⁰ = 1,024 But eventually 2ⁿ wins: n=59: 2⁵⁹ ≈ 5.8 × 10¹⁷ 59¹⁰ ≈ 5.1 × 10¹⁷ After n=59, exponential dominates forever.

P vs NP connection

P = Problems solvable in Polynomial time
NP = Problems verifiable in Polynomial time

The P vs NP question asks: Is verification as hard as solving?
If P ≠ NP, some problems have no polynomial algorithm.
Section 4

Logarithms: The Inverse of Exponentials

Definition

logₐ(x) = y means aʸ = x "log base a of x equals y" Examples: log₂(8) = 3 because 2³ = 8 log₂(1024) = 10 because 2¹⁰ = 1024 log₁₀(1000) = 3 because 10³ = 1000

Key logarithm properties

log(ab) = log(a) + log(b) Product → Sum log(a/b) = log(a) - log(b) Quotient → Difference log(aᵏ) = k × log(a) Power → Multiply logₐ(a) = 1 log of base = 1 logₐ(1) = 0 log of 1 = 0

Why O(log n) is amazing

log₂(n) grows incredibly slowly:

log₂(1,000) ≈ 10
log₂(1,000,000) ≈ 20
log₂(1,000,000,000) ≈ 30

A billion elements? Only 30 steps with binary search!

Change of base

logₐ(x) = logᵦ(x) / logᵦ(a) In Big-O, base doesn't matter: log₂(n) = log₁₀(n) / log₁₀(2) ≈ 3.32 × log₁₀(n) All logarithms differ by a constant factor, so: O(log₂ n) = O(log₁₀ n) = O(ln n) = O(log n)
Section 5

Log Plots: Seeing the Pattern

Why use log scale?

Problem: On a linear plot, exponential growth makes everything else look flat.

Solution: Log scale compresses large values, revealing patterns.

Log-log plots

If y = xᵏ (polynomial), then: log(y) = k × log(x) On a log-log plot, polynomials become straight lines! The slope tells you the exponent k. slope = 1 → linear (n) slope = 2 → quadratic (n²) slope = 3 → cubic (n³)

Semi-log plots

If y = cⁿ (exponential), then: log(y) = n × log(c) On a semi-log plot (log y, linear x): Exponentials become straight lines! The slope reveals the base c.

Python example

PythonIdentifying growth with log plots
import numpy as np
import matplotlib.pyplot as plt

n = np.arange(1, 100)

# Different growth rates
linear = n
quadratic = n**2
exponential = 2**n

# Log-log plot reveals polynomial degree
plt.figure()
plt.loglog(n, linear, label='n (slope=1)')
plt.loglog(n, quadratic, label='n² (slope=2)')
plt.legend()
plt.title('Log-Log: Polynomials are lines')

# Semi-log plot reveals exponential
plt.figure()
plt.semilogy(n[:30], exponential[:30], label='2ⁿ')
plt.legend()
plt.title('Semi-Log: Exponential is a line')
