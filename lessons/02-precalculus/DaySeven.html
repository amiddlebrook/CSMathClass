Why Graphs Matter in CS

You rarely solve specific equations in CS. Instead, you analyze "Trends". Is the memory usage linear ($y=x$) or exponential ($y=e^x$)? Does the database latency blow up at a certain load (Asymptotes)? Graphs give you the intuition for system behavior.

üìà Analogy: The Heart Monitor
Looking at raw numbers (x, y coordinates) is like listening to a heartbeat once a minute.
Looking at the GRAPH is like seeing the EKG trace. You spot the irregularities immediately.
Section 1

Function Families & APIs

In CS, a function is a Map. It takes one set of values (Domain/Input) and maps them to another set (Range/Output). Think of these as typed API parameters.

Domain & Input Sanitization

In math, you can't divide by zero. In CS, you can't process a negative index. The Domain of your function is the set of "Safe Inputs."

Validation Rule: Before executing $f(x)$, any robust system must verify that $x \in \text{Domain}(f)$. This prevents runtime exceptions like NullPointer or ArithmeticException.

Linear ($y=mx+b$)

Look: Constant slope.

CS: Single-pass loops. Processing an Array. 1:1 mapping.

Quadratic ($y=x^2$)

Look: Curving acceleration.

CS: Nested loops. Comparison sorts (Bubble/Selection).

Exponential ($y=a^x$)

Look: Vertical takeoff.

CS: Brute-forcing passwords. Tree depth grows. $O(2^n)$.

Logarithmic ($y=\log x$)

Look: Rapid start, infinite slowdown.

CS: Binary search. Quicksort partitions. $O(\log n)$.

Python: Benchmarking Growth

PythonGrowth Comparison
import math

def compare_growth(n):
    return {
        "linear": n,
        "quadratic": n**2,
        "logarithmic": math.log2(n) if n > 0 else 0,
        "exponential": 2**n if n < 100 else "INF"
    }

print(f"{'N':<5} | {'Lin':<10} | {'Quad':<10} | {'Log':<10}")
for n in [10, 20, 30]:
    g = compare_growth(n)
    print(f"{n:<5} | {g['linear']:<10} | {g['quadratic']:<10} | {g['logarithmic']:<10.2f}")

Glossary of Growth Trends

Use this table to translate between mathematical growth and real-world software performance.

Complexity Logic Name Visual Behavior Scaling Verdict
$O(1)$ Constant Flat line. No change for $N$. Perfect
$O(\log n)$ Logarithmic Slow growth. Near horizontal. High Scalability
$O(n)$ Linear Steady diagonal (45 degrees). Industry Standard
$O(n^2)$ Quadratic Curving upward. Acceleration. Danger Zone
$O(2^n)$ Exponential Vertical takeoff. Impossible N. Catastrophic
Section 2

Composite Functions (Pipelines)

Composition is the act of passing the output of one function as the input to another: $(f \circ g)(x) = f(g(x))$.

üöÄ The Pipeline Analogy:
In Unix, this is the | pipe. cat file | grep "error" is a composite function.
$f(g(x)) = \text{grep}( \text{cat}(x) )$

JavaScript: Functional Decorators

JavaScriptPipe Helper
const add1 = (x) => x + 1;
const square = (x) => x * x;

// Manual composition
const f_of_g = (x) => square(add1(x));

// High-order pipe
const pipe = (...fns) => (x) => fns.reduce((v, f) => f(v), x);

const optimized = pipe(add1, square);
console.log(optimized(5)); // (5+1)^2 = 36
Section 3

Visual Transformations

Mathematical transformations are the basis for all 2D/3D graphics. Every time you scale a window or pan a map, you are applying transformations to a function $f(x)$.

The Transformation Sheet:
‚Ä¢ $f(x) + k$: Vertical Shift (Lift/Drop).
‚Ä¢ $f(x - h)$: Horizontal Shift (Lag/Lead).
‚Ä¢ $a \cdot f(x)$: Vertical Stretch (Amplitude).
‚Ä¢ $f(b \cdot x)$: Horizontal Squeeze (Frequency).
üéöÔ∏è Analogy: The Soundboard
Think of a sine wave (sound).
‚Ä¢ Shift Up = DC Offset.
‚Ä¢ Shift Left = Phase Delay.
‚Ä¢ Vertical Stretch = Volume (Amplitude).
‚Ä¢ Horizontal Squeeze = Pitch (Frequency).

Python: Transform Visualizer

PythonShifting x^2
import numpy as np
import matplotlib.pyplot as plt

x = np.linspace(-5, 5, 100)
f_x = x**2

# Transformed: g(x) = 2(x-2)^2 + 3
g_x = 2 * (x - 2)**2 + 3

plt.figure(figsize=(8, 4))
plt.plot(x, f_x, '--', label="Original f(x)=x^2")
plt.plot(x, g_x, label="Transformed g(x)=2x(x-2)^2+3")
plt.grid(True)
plt.legend()
plt.title("Visualizing Transformations")
plt.show()
Section 4

Inverse Functions (The Undo Button)

An inverse function $f^{-1}(x)$ undoes the action of $f(x)$. If $f$ maps $x \to y$, then $f^{-1}$ maps $y \to x$.

üîë Security & Cryptography:
Encryption is a function $f(\text{plain}) = \text{cipher}$.
Decryption is exactly the Inverse function $f^{-1}(\text{cipher}) = \text{plain}$.
In CS, we look for "One-Way Functions": easy to compute, but impossible to find the inverse.

Python: Symbolic Inverses

PythonSymPy Solver
from sympy import symbols, solve

x, y = symbols('x y')
# Define f(x) = 2x + 5
f_expr = 2*x + 5

# Set y = f(x) and solve for x to find inverse function g(y)
inv = solve(y - f_expr, x)[0]
print(f"Inverse function is: {inv}") # (y - 5)/2
Section 5

Rational Functions & Asymptotes

A Rational Function is a ratio of polynomials: $f(x) = \frac{P(x)}{Q(x)}$. Interesting things happen when $Q(x) = 0$.

Vertical Asymptote (The Wall):
When the denominator hits zero, the value shoots to $\infty$.
CS Context: This is a crash. Division by zero. Infinite loop. Memory exhaustion.
Horizontal Asymptote (The Limit):
As $x \to \infty$, what does the function do?
If degree(P) < degree(Q), it goes to 0.
If degree(P) == degree(Q), it goes to ratio of coefficients.
CS Context: Steady state. Max throughput of a server.

Python: Handling Discontinuities

PythonMasking Asymptotes
import numpy as np
import matplotlib.pyplot as plt

x = np.linspace(-5, 5, 400)
y = 1 / (x - 2)

# Mask the large values near the asymptote for a cleaner plot
y[np.abs(y) > 10] = np.nan

plt.plot(x, y)
plt.axvline(x=2, color='r', linestyle='--', label="Vertical Asymptote")
plt.axhline(y=0, color='g', linestyle='--', label="Horizontal Asymptote")
plt.title("Rational Function: 1 / (x-2)")
plt.legend()
plt.show()

Piecewise Functions (Switch Cases)

A Piecewise function uses different formulas for different parts of its domain. This is the mathematical version of an if/else or switch/case block.

The Absolute Value Function:
$|x| = \begin{cases} x & \text{if } x \geq 0 \\ -x & \text{if } x < 0 \end{cases}$
üéÆ Game Dev Context:
Collision detection or AI state machines often use piecewise functions. If distance < 10, "Attack" ; else if distance < 50, "Follow" ; else "Idle" .

Python: Plotting Piecewise Logic

Pythonnp.select
x = np.linspace(-10, 10, 100)

# Functional logic: f(x) = x^2 if x < 0, else 2x
conditions = [x < 0, x >= 0]
functions = [lambda x: x**2, lambda x: 2*x]
y = np.piecewise(x, conditions, functions)

plt.plot(x, y)
plt.title("Piecewise Function Visualization")
plt.show()
Section 7

Symmetry (Even vs. Odd)

Recognizing symmetry allows engineers to cut computation time in half. If a function is symmetric, you only need to calculate half the values!

Even Functions

Rule: $f(x) = f(-x)$

Look: Reflection across Y-axis.

Example: $x^2$, $\cos(x)$.

Odd Functions

Rule: $f(-x) = -f(x)$

Look: 180¬∞ Rotation about Origin.

Example: $x^3$, $\sin(x)$.

‚ö° Optimization:
In Signal Processing (DSP), knowing the symmetry of a signal helps optimize the **Fourier Transform**. Calculating only half the points reduces CPU cycles by 50%.

Python: Symmetry Checker

PythonAutomated Check
def check_symmetry(func, range_x=10):
    for x in range(1, range_x):
        if func(x) != func(-x):
            break
    else:
        return "Even"
        
    for x in range(1, range_x):
        if func(-x) != -func(x):
            break
    else:
        return "Odd"
        
    return "Neither"

# Test it
print(f"x^2 is: {check_symmetry(lambda x: x**2)}")
print(f"x^3 is: {check_symmetry(lambda x: x**3)}")
Section 8

Log-Log Plots

When data spans orders of magnitude (like $10$ requests vs $1,000,000$ requests), linear plots are useless. Everything looks like zero except the last point.

The Power of Log-Log:
On a log-log plot (taking log of both axes), Power Laws become Straight Lines.
If $y = x^2$, then $\log y = \log(x^2) = 2 \log x$.
This is a line with slope 2!
üïµÔ∏è Big O Detective:
Run your algorithm for N=10, 100, 1000.
Plot Time vs N on Log-Log paper.
Step 3: Measure the slope.
‚Ä¢ Slope 1 ‚Üí O(n)
‚Ä¢ Slope 2 ‚Üí O(n¬≤)
‚Ä¢ Slope 0 ‚Üí O(1)

Visualizing in Python (The Lab)

Using `matplotlib` to see the invisible.

Python Log-Log Plotting 
import matplotlib.pyplot as plt
import numpy as np

# Simulate input sizes N
N = np.array([10, 100, 1000, 10000, 100000])

# Algorithms
t_linear = N * 0.001           # O(n)
t_quad = (N ** 2) * 0.000001   # O(n^2)

# Create Log-Log plot
plt.figure(figsize=(10, 6))
plt.loglog(N, t_linear, label='Linear O(n)', marker='o')
plt.loglog(N, t_quad, label='Quadratic O(n^2)', marker='x')

plt.grid(True, which="both", ls="-")
plt.xlabel('Input Size (N)')
plt.ylabel('Time (seconds)')
plt.title('Algorithm Complexity on Log-Log Scale')
plt.legend()
plt.show()

# Observe: The "curve" of n^2 becomes a straight line with steeper slope!
Section 10

Optimization Gaps

In high-performance computing, we often care about how functions behave in the gaps‚Äîwhere data is sparse or precision is critical.

Floating Point Precision:
Graphs look smooth, but computers are discrete. For extremely large or small values, functions like $e^x$ can suffer from "Underflow" or "Overflow".
Solution: Use Log-space calculations to keep numbers manageable.
The "Hot Spot" Rule:
If a function has a steep slope (high derivative), your sampling density must increase in that region to avoid aliasing in graphics or numerical errors in physics engines.
