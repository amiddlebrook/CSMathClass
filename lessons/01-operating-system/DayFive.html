Section 0

Why Proof Writing Matters in CS

Writing proofs is like writing documentation for your logical reasoning. Just as code needs structure (functions, modules, classes), proofs need structure (definitions, claims, steps).

Key insight: A well-structured proof is readable, verifiable, and maintainable—just like good code.

The parallel to code

  • Definitions = Type declarations and interfaces
  • Claims/Theorems = Function signatures (what you promise to deliver)
  • Proof = Implementation (how you deliver it)
  • QED = Return statement / tests passing
ComparisonProof structure ≈ Code structure
// CODE
interface Number { ... }     // Definition
function isEven(n): bool    // Claim (signature)
{ return n % 2 == 0; }      // Proof (implementation)

// PROOF
Definition: n is even if n = 2k for some integer k.
Claim: The sum of two even numbers is even.
Proof: Let a = 2m and b = 2n. Then a + b = 2(m+n). ∎
Section 1

The Standard Proof Structure

The four parts

1. DEFINITIONS — State precisely what terms mean 2. CLAIM/THEOREM — State exactly what you will prove 3. PROOF — The logical argument 4. QED — Signal that the proof is complete (∎ or □ or QED)

Complete example

Definition
An integer n is even if n = 2k for some integer k.
An integer n is odd if n = 2k + 1 for some integer k.
Theorem
The sum of two even integers is even.
Proof
Let a and b be even integers.
By definition, a = 2m for some integer m, and b = 2n for some integer n.
Then a + b = 2m + 2n = 2(m + n).
Since m + n is an integer, a + b has the form 2k where k = m + n.
Therefore, by definition, a + b is even.
∎
Section 2

Writing Good Definitions

What makes a good definition

A good definition is:
• Precise — no ambiguity
• Complete — covers all cases
• Minimal — no redundant conditions
• Usable — easy to apply in proofs

Definition patterns

Existential: "X is Y if there exists Z such that..." Example: n is even if there exists an integer k such that n = 2k. Universal: "X is Y if for all Z, ..." Example: f is continuous if for all ε > 0, there exists δ > 0... Recursive: Base case + recursive case Example: 0! = 1, and n! = n × (n-1)! for n > 0.

CS parallel: Type definitions

TypeScriptDefinitions in code
// Existential definition
type Even = number & { _brand: 'even' };
const isEven = (n: number): n is Even => n % 2 === 0;

// Recursive definition
type Tree = 
  | { kind: 'leaf', value: T }
  | { kind: 'node', left: Tree, right: Tree };
Section 3

Stating Claims and Theorems

The hierarchy

Theorem: A major result, often the main goal Lemma: A helper result used to prove theorems Corollary: An easy consequence of a theorem Proposition: A result of medium importance Claim: An informal assertion (often inside a proof)

Writing precise claims

Bad: "Even numbers plus even numbers are even."
Good: "For all integers a and b, if a is even and b is even, then a + b is even."

A precise claim should:

  • Quantify all variables (for all, there exists)
  • Specify the domain (integers, reals, etc.)
  • Make the logical structure explicit (if...then)

Common claim patterns

Universal: For all x in S, P(x). "Every prime > 2 is odd." Existential: There exists x in S such that P(x). "There exists a prime between n and 2n." Conditional: If P, then Q. "If n² is even, then n is even." Biconditional: P if and only if Q. "n is even iff n² is even."
Section 4

Writing the Proof Body

Start with a roadmap

Opening line should state your approach:
"We proceed by induction on n."
"We prove the contrapositive."
"Suppose for contradiction that..."
"Let a and b be arbitrary even integers."

Justify every step

Each step should have a reason: - "by definition" - "by assumption" - "by the inductive hypothesis" - "by Theorem 2.3" - "since a = 2k" - "by algebra"

Proof template

TemplateStandard proof structure
Proof:
  // 1. State your setup
  Let [variables] be [what they are].
  
  // 2. Apply definitions
  By definition, [variable] = [form].
  
  // 3. Do the work
  Then [calculation or logical step].
  [More steps as needed, each justified.]
  
  // 4. Connect to goal
  Therefore, [conclusion].
  
  // 5. End marker
  ∎

Complete worked example

Theorem
For all integers n, if n² is odd, then n is odd.
Proof (by contrapositive)
We prove the contrapositive: if n is even, then n² is even.

Let n be an even integer.
By definition, n = 2k for some integer k.
Then n² = (2k)² = 4k² = 2(2k²).
Since 2k² is an integer, n² has the form 2m where m = 2k².
Therefore n² is even by definition.

Since the contrapositive is true, the original statement holds.
∎
Section 5

Style Guide for Proofs

Do's

  • Write in complete sentences. "Let n be an integer." not "n = integer"
  • Define variables before using them. "Let ε > 0." before using ε
  • Be explicit about assumptions. "Assume P(k) holds for some k ≥ 1."
  • Use transitional words. "Then", "Therefore", "Hence", "Thus", "Since"
  • End with a clear conclusion. "Therefore, the claim holds."

Don'ts

Don't use undefined variables. Always introduce before use.

Don't skip steps. What's obvious to you may not be to readers.

Don't prove by example. Showing P(5) doesn't prove "for all n".

Don't assume what you're proving. Circular reasoning is invalid.

Don't forget edge cases. Handle n = 0, empty sets, etc.

Mathematical notation

∀ — "for all" ∃ — "there exists" ∈ — "is an element of" ⊆ — "is a subset of" ⟹ or → — "implies" ⟺ or ↔ — "if and only if" ∎ or □ or QED — "end of proof"
