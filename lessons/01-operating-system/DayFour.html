Section 0

Why Proofs Matter in CS

Every algorithm correctness argument is a proof. When you claim your sorting function works, you're implicitly proving it. Understanding proof techniques makes you better at reasoning about code.

Key insight: Debugging is just proof-reading. You're checking if your implicit proof of correctness actually holds.

Where you'll use this

  • Algorithm Analysis: Proving correctness, termination, complexity bounds
  • Loop Invariants: Showing loops do what they claim
  • Recursive Functions: Induction proves recursion is correct
  • Security: Cryptographic security proofs
  • Type Systems: Type safety proofs guarantee no runtime type errors
PythonEvery algorithm has an implicit proof
def binary_search(arr, target):
    """
    Implicit proof claims:
    1. If target exists, we find it (correctness)
    2. Loop terminates (lo eventually meets hi)
    3. O(log n) time (halving search space)
    
    Each claim needs a proof technique!
    """
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if arr[mid] == target: return mid
        elif arr[mid] < target: lo = mid + 1
        else: hi = mid - 1
    return -1
Section 1

Direct Proof

The method

To prove P → Q: Assume P is true, then show Q must follow through logical steps.
Structure: 1. Assume P (the hypothesis) 2. Apply definitions and known facts 3. Derive Q (the conclusion) 4. Therefore P → Q. ∎

Example: Sum of two even numbers

Claim: The sum of two even numbers is even. Proof: 1. Let a and b be even numbers. 2. By definition, a = 2m and b = 2n for some integers m, n. 3. Then a + b = 2m + 2n = 2(m + n). 4. Since m + n is an integer, a + b = 2k where k = m + n. 5. Therefore a + b is even by definition. ∎

CS Example: Array access is O(1)

Claim: Accessing arr[i] takes constant time. Proof: 1. An array stores elements contiguously in memory. 2. The address of arr[i] = base_address + i × element_size. 3. This calculation uses only addition and multiplication. 4. Both operations take constant time. 5. Therefore array access is O(1). ∎
Section 2

Proof by Contrapositive

The method

Key fact: P → Q is logically equivalent to ¬Q → ¬P.
Instead of proving "if P then Q", prove "if not Q then not P".
Original: If n² is even, then n is even. Contrapositive: If n is odd, then n² is odd. These are logically equivalent! Sometimes the contrapositive is easier to prove.

Example: If n² is even, then n is even

Prove contrapositive: If n is odd, then n² is odd. Proof: 1. Assume n is odd. 2. Then n = 2k + 1 for some integer k. 3. n² = (2k + 1)² = 4k² + 4k + 1 = 2(2k² + 2k) + 1. 4. This is of the form 2m + 1, so n² is odd. 5. Therefore, by contrapositive, if n² is even then n is even. ∎

When to use it

Use contrapositive when:

  • The conclusion Q involves negation or "not"
  • ¬Q gives you something concrete to work with
  • The direct path seems blocked
Section 3

Proof by Contradiction

The method

Strategy: Assume the opposite of what you want to prove. Show this leads to a logical impossibility. Therefore, your original claim must be true.
Structure: 1. Assume ¬P (the negation of your claim) 2. Derive a contradiction (something impossible) 3. Therefore P must be true. ∎

Classic: √2 is irrational

Claim: √2 is irrational. Proof: 1. Assume √2 is rational. 2. Then √2 = a/b where a,b are integers with no common factors. 3. Squaring: 2 = a²/b², so a² = 2b². 4. Thus a² is even, so a is even (from contrapositive proof). 5. Let a = 2k. Then (2k)² = 2b², so 4k² = 2b², so b² = 2k². 6. Thus b² is even, so b is even. 7. But if a and b are both even, they share factor 2. Contradiction! 8. Therefore √2 is irrational. ∎

CS Example: Halting problem is undecidable

Claim: No program can decide if any program halts. Proof sketch (contradiction): 1. Assume a Halts(P, I) function exists that returns true/false. 2. Construct Paradox(P): if Halts(P, P) then loop forever else halt. 3. Does Paradox(Paradox) halt? - If yes → Halts returns true → it loops forever. Contradiction! - If no → Halts returns false → it halts. Contradiction! 4. Therefore Halts cannot exist. ∎
Section 4

Mathematical Induction

The method

To prove P(n) for all n ≥ base:
1. Base case: Prove P(base) is true
2. Inductive step: Prove P(k) → P(k+1)
Think of it like dominoes: - Base case: First domino falls - Inductive step: Each domino knocks down the next - Conclusion: ALL dominoes fall

Example: Sum formula

Claim: 1 + 2 + ... + n = n(n+1)/2 for all n ≥ 1. Base case (n = 1): Left: 1 Right: 1(2)/2 = 1 ✓ Inductive step: Assume true for k: 1 + 2 + ... + k = k(k+1)/2 Show true for k+1: 1 + 2 + ... + k + (k+1) = k(k+1)/2 + (k+1) = (k+1)(k/2 + 1) = (k+1)(k+2)/2 ✓ By induction, the formula holds for all n ≥ 1. ∎

CS Example: Proving recursion correct

PythonInduction proves recursion works
def factorial(n):
    if n == 0: return 1        # Base case
    return n * factorial(n-1)   # Inductive step

# Proof that factorial(n) = n!
# Base: factorial(0) = 1 = 0! ✓
# Inductive: Assume factorial(k) = k!
#   factorial(k+1) = (k+1) × factorial(k)
#                  = (k+1) × k!  (by assumption)
#                  = (k+1)! ✓

Strong induction

Strong induction: Assume P(1), P(2), ..., P(k) all true to prove P(k+1). Useful when P(k+1) depends on multiple previous cases, not just P(k). Example: Every integer > 1 has a prime factorization. - Need strong induction because n might factor as a × b where both a, b < n (not just n-1).
Section 5

Loop Invariants

What is a loop invariant?

Definition: A condition that is true before and after each iteration of a loop.

To prove correctness:
1. Initialization: Invariant holds before loop starts
2. Maintenance: If invariant holds before iteration, it holds after
3. Termination: When loop ends, invariant + exit condition → correctness

Example: Linear search

PythonLoop invariant proof
def find(arr, target):
    # Invariant: target is not in arr[0..i-1]
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

# Initialization: Before i=0, arr[0..-1] is empty, so invariant trivially holds.
# Maintenance: If arr[i] ≠ target, invariant extends to arr[0..i].
# Termination: Loop ends when found (correct!) or i = len(arr),
#              meaning target not in arr[0..n-1], so -1 is correct.

Example: Binary search invariant

Invariant: If target exists in arr, it is in arr[lo..hi]. Maintenance: - If arr[mid] < target, target must be in arr[mid+1..hi]. Setting lo=mid+1 maintains invariant. - If arr[mid]> target, target must be in arr[lo..mid-1]. Day 4 · Operating Systemid-1 maintains invariant. Termination: - Found at mid: correct! - lo > hi: search space empty, target doesn't exist. ✓
