Section 1

The Dream of a Universal Language

Before computers were metal boxes on our desks, they were a mathematical question: **Can every problem be solved by a mechanical process?**

In 1936, Alan Turing answered this by defining the simplest possible machine that could still perform any calculation that a human could. We call this the **Turing Machine**.

The Analogy

Think of a Turing Machine as a person with an infinite roll of graph paper and a very strict set of rules. They can only see one square at a time, erase what's there, write a new symbol, and move one square left or right. It sounds primitive, but this is the "DNA" of every CPU in existence.

Section 2

The 7-Tuple Definition

A Turing Machine $M$ is formally defined as a 7-tuple: $M = (Q, \Sigma, \Gamma, \delta, q_0, B, F)$

  • $Q$: A finite set of **states**.
  • $\Sigma$: The **input alphabet**.
  • $\Gamma$: The **tape alphabet** (all symbols allowed on the tape).
  • $\delta$: The **transition function**: $Q \times \Gamma \to Q \times \Gamma \times \{L, R\}$.
  • $q_0$: The **initial state**.
  • $B$: The **blank symbol**.
  • $F$: The set of **final/accepting states**.

The "Transition Function" is the logic. It says: "If I am in state $q$ and I see symbol $s$, then go to state $q'$, write $s'$, and move my head Left or Right."

Section 3

The Universal Turing Machine (UTM)

A normal Turing Machine is built for one specific task (e.g., adding numbers). But Turing realized you could build a machine that takes **the description of another machine** as its input tape.

This is the **Universal Turing Machine**. It is the mathematical proof that a single piece of hardware can run any software. Your laptop is a physical implementation of a UTM.

Section 4

The Halting Problem

Turing's most famous discovery was something a machine **cannot** do. He proved that no program can ever exist that determines if any arbitrary program will eventually stop or run forever.

$$ HALT(M, w) = \text{Undecidable} $$

The Paradox

If you have a function `stops(program)` and you pass it a program that does the opposite of whatever `stops` says, you create a logical paradox. Therefore, `stops` cannot exist. Computation has limits.

Section 5

The Church-Turing Thesis

The thesis states that **anything** that can be calculated by a human with a pen and paper can be calculated by a Turing Machine. There is no "super-computation" that escapes this model. If a Turing Machine can't do it, it's not computable.

Section 6

Python: A Simple Turing Machine

Let's simulate a machine that simply increments a binary number on a tape.

Python Turing Simulator 
class TuringMachine:
    def __init__(self, tape, initial_state, transition_function):
        self.tape = list(tape)
        self.head = 0
        self.state = initial_state
        self.delta = transition_function

    def step(self):
        char = self.tape[self.head]
        if (self.state, char) not in self.delta:
            return False # Halt
        
        new_state, new_char, direction = self.delta[(self.state, char)]
        self.tape[self.head] = new_char
        self.state = new_state
        self.head += 1 if direction == 'R' else -1
        return True

    def run(self):
        while self.step():
            print(f"State: {self.state}, Tape: {''.join(self.tape)}")

# Rules for incrementing binary (simple version)
# (State, Sym) -> (NewState, NewSym, Dir)
rules = {
    ('START', '1'): ('START', '1', 'R'),
    ('START', '0'): ('START', '0', 'R'),
    ('START', ' '): ('INC', ' ', 'L'),
    ('INC', '0'): ('HALT', '1', 'R'),
    ('INC', '1'): ('INC', '0', 'L'),
}

tm = TuringMachine("110 ", "START", rules)
tm.run()
