Why Limits Matter in CS

In Algorithm Analysis, we ask: "What happens as N approaches Infinity?" (Big O). In Numerical Computing, we ask: "What happens as our step size approaches Zero?" (Integration). Limits are the formal way to answer "What happens at the boundary?"

Algorithm Analysis

Big O: $\lim_{n \to \infty} \frac{f(n)}{g(n)}$

"How does f(n) compare to g(n) as input grows?"

Numerical Methods

Derivatives: $\lim_{h \to 0} \frac{f(x+h) - f(x)}{h}$

"What is the instantaneous slope?"

üéØ Analogy: The Photographer
Imagine zooming in on a photo. A limit asks: "What would this pixel BE if we had infinite resolution?"
Sometimes the answer is clear (continuous image). Sometimes there's a hard edge (discontinuity).
Section 1

The "Pixels" Argument

Imagine a curve on a screen. If you zoom in, it looks jagged (pixels). A Limit asks: "If we had infinite pixels, where would this point BE?"

$$ \lim_{x \to c} f(x) = L $$ Translation: "As x gets closer and closer to c, f(x) gets closer and closer to L." Note: It doesn't matter what happens AT c. The function could be undefined there.

One-Sided Limits

Sometimes the left and right don't agree. We use separate notation:

Left-Hand Limit

$\lim_{x \to c^-} f(x)$

Approaching c from values less than c.

Right-Hand Limit

$\lim_{x \to c^+} f(x)$

Approaching c from values greater than c.

Key Rule: A limit exists at c if and only if both one-sided limits exist AND are equal.
üö¶ Analogy: The Traffic Light
One-sided limits are like approaching an intersection from different directions.
If cars from the left see green and cars from the right see red, there's no agreement‚Äîno limit.
Python One-Sided Limit Check 
def check_one_sided(f, c, delta=0.0001):
    """Check left and right limits at c."""
    left_limit = f(c - delta)
    right_limit = f(c + delta)
    
    print(f"Left limit:  f({c - delta}) = {left_limit:.6f}")
    print(f"Right limit: f({c + delta}) = {right_limit:.6f}")
    
    if abs(left_limit - right_limit) < 1e-6:
        print(f"Limits agree! L ‚âà {(left_limit + right_limit)/2:.6f}")
    else:
        print("Limits do NOT agree. No limit exists at this point.")

# Example: f(x) = 1/x at x=0
import math
check_one_sided(lambda x: 1/x, 0.0)  # Left: -inf, Right: +inf - no limit!
Section 2

Epsilon-Delta (The Ultimate Unit Test)

This is the scariest definition in Calculus, but to a programmer, it's just a test case.

The Definition:
For every $\epsilon > 0$ (Error Tolerance), there exists a $\delta > 0$ (Input Precision) such that...
If $|x - c| < \delta$, then $|f(x) - L| < \epsilon$.
üß™ Programmer's Translation:
Boss: "I need the output to be within 0.001 of the target." ($\epsilon$)
You: "Okay, I need to make sure the input is within 0.0005 of the target." ($\delta$)
If you can ALWAYS satisfy the Boss, no matter how small $\epsilon$ gets, the Limit exists.

Finding Delta

For the function $f(x) = 2x$, with limit $L = 6$ at $c = 3$:

We need $|f(x) - 6| < \epsilon$, which means $|2x - 6| < \epsilon$, so $|x - 3| < \epsilon/2$.

Therefore, $\delta = \epsilon/2$ works!

Python Epsilon-Delta Verification 
def verify_epsilon_delta(f, c, L, epsilon, delta):
    """
    Verify that delta works for the given epsilon.
    Sample many points within delta of c and check if f(x) is within epsilon of L.
    """
    import numpy as np
    
    # Sample 1000 points in (c-delta, c+delta), excluding c itself
    x_values = np.linspace(c - delta + 1e-10, c + delta - 1e-10, 1000)
    
    for x in x_values:
        error = abs(f(x) - L)
        if error >= epsilon:
            print(f"FAILED at x={x:.6f}: |f(x)-L| = {error:.6f} >= {epsilon}")
            return False
    
    print(f"PASSED! For Œµ={epsilon}, Œ¥={delta} works.")
    return True

# Example: lim(x->3) 2x = 6
f = lambda x: 2*x
verify_epsilon_delta(f, c=3, L=6, epsilon=0.01, delta=0.005)  # PASSED!
Section 3

Continuity

A function is continuous if you can draw it without lifting your pen.

Formal Definition

1. $f(c)$ is defined.

2. Limit exists at c.

3. Limit == $f(c)$.

Discontinuities

Jump: Teleporting (Heaviside step).

Removable: A hole (0/0).

Infinite: Asymptote (1/x).

üöó Analogy: The GPS Navigation
Continuity is like a road without potholes. You can drive smoothly from A to B.
A discontinuity is a pothole, cliff, or missing bridge‚Äîyou can't smoothly pass through it.
Section 4

Intermediate Value Theorem (IVT)

If a function is continuous on $[a, b]$, and you want to find where $f(x) = k$ for some value $k$ between $f(a)$ and $f(b)$, then such an $x$ MUST exist.

The Theorem:
If $f$ is continuous on $[a, b]$ and $f(a) < k < f(b)$ (or vice versa),
then there exists $c \in (a, b)$ such that $f(c) = k$.

CS Application: Binary Search

We use IVT intuitively whenever we do binary search for a root.

If $f(a) < 0$ and $f(b)> 0$, and $f$ is continuous, there MUST be a root between them.

Root Finding

The Bisection Method exploits IVT:

1. Check sign of $f(mid)$

2. Narrow interval to where sign changes

3. Repeat until precision is met

Python Bisection Method (Root Finding) 
def bisection(f, a, b, tol=1e-9, max_iter=100):
    """Find root of f in [a, b] using IVT."""
    if f(a) * f(b) >= 0:
        raise ValueError("f(a) and f(b) must have opposite signs")
    
    for i in range(max_iter):
        mid = (a + b) / 2
        if abs(f(mid)) < tol:
            return mid
        if f(a) * f(mid) < 0:
            b = mid
        else:
            a = mid
    return (a + b) / 2

# Example: Find sqrt(2) by solving x^2 - 2 = 0
import math
f = lambda x: x**2 - 2
root = bisection(f, 1, 2)
print(f"sqrt(2) ‚âà {root}")  # 1.4142135...
Section 5

The Squeeze Theorem

Sometimes you can't evaluate a limit directly, but you can trap it between two other limits.

The Theorem:
If $g(x) \leq f(x) \leq h(x)$ near $c$, and
$\lim_{x \to c} g(x) = \lim_{x \to c} h(x) = L$,
then $\lim_{x \to c} f(x) = L$.
ü•™ Analogy: The Sandwich
If the top and bottom slices of bread both go to the same place, the filling (middle function) MUST go there too.

Classic Example: $\lim_{x \to 0} x \sin(1/x)$

We know $-1 \leq \sin(1/x) \leq 1$.

So $-|x| \leq x \sin(1/x) \leq |x|$.

As $x \to 0$, both $-|x|$ and $|x|$ go to 0.

Therefore, by the Squeeze Theorem, $\lim_{x \to 0} x \sin(1/x) = 0$.

Section 6

L'H√¥pital's Rule

When direct substitution gives an indeterminate form like $0/0$ or $\infty/\infty$, you can take derivatives of the top and bottom.

The Rule:
If $\lim_{x \to c} \frac{f(x)}{g(x)}$ is $\frac{0}{0}$ or $\frac{\infty}{\infty}$,
then $\lim_{x \to c} \frac{f(x)}{g(x)} = \lim_{x \to c} \frac{f'(x)}{g'(x)}$ (if the right side exists).

Example 1: $\lim_{x \to 0} \frac{\sin x}{x}$

Direct: $\frac{0}{0}$ (indeterminate)

Apply L'H√¥pital: $\frac{\cos x}{1} = \cos(0) = 1$

Example 2: $\lim_{x \to \infty} \frac{x^2}{e^x}$

Direct: $\frac{\infty}{\infty}$ (indeterminate)

Apply twice: $\frac{2x}{e^x} \to \frac{2}{e^x} = 0$

Common Mistake:
L'H√¥pital's Rule only works for $0/0$ or $\infty/\infty$ forms.
Do NOT use it for forms like $1/0$, $0 \cdot \infty$, or $\infty - \infty$ directly.
Python SymPy Limit Calculation 
from sympy import symbols, sin, limit, oo, exp

x = symbols('x')

# Example 1: sin(x)/x as x -> 0
result1 = limit(sin(x)/x, x, 0)
print(f"lim sin(x)/x as x‚Üí0: {result1}")  # 1

# Example 2: x^2 / e^x as x -> infinity
result2 = limit(x**2 / exp(x), x, oo)
print(f"lim x¬≤/eÀ£ as x‚Üí‚àû: {result2}")  # 0

# Example 3: (1 + 1/x)^x as x -> infinity (classic e definition)
result3 = limit((1 + 1/x)**x, x, oo)
print(f"lim (1+1/x)À£ as x‚Üí‚àû: {result3}")  # E
Section 7

CS Application: Anti-Aliasing

Computer screens are discrete grids. They cannot perfectly represent continuous lines. This causes "Aliasing" (jaggies).

The Problem

A diagonal line on a screen must be approximated by pixels.

Without smoothing, the edge looks like a staircase.

This is the discrete-continuous gap in action.

The Solution

MSAA: Sample multiple points per pixel.

SSAA: Render at higher resolution, then downsample.

Both approximate the "limit" of infinite samples.

üéÆ Analogy: The Blurry Photo
Anti-aliasing is like taking a photo that's slightly out of focus.
The edges are softer, but they look more natural than hard pixels.

How MSAA Works

Instead of asking "Is this pixel on the line or not?" (binary),

We ask "What percentage of this pixel is covered by the line?" and blend accordingly.

$$ \text{Color} = \alpha \cdot \text{Object} + (1-\alpha) \cdot \text{Background} $$ Where $\alpha$ is the coverage ratio (0 to 1).
Python Simple Alpha Blending 
def blend(fg_color, bg_color, alpha):
    """Blend foreground and background based on coverage (alpha)."""
    r = alpha * fg_color[0] + (1 - alpha) * bg_color[0]
    g = alpha * fg_color[1] + (1 - alpha) * bg_color[1]
    b = alpha * fg_color[2] + (1 - alpha) * bg_color[2]
    return (int(r), int(g), int(b))

# 50% white line on black background
white = (255, 255, 255)
black = (0, 0, 0)
edge_pixel = blend(white, black, 0.5)
print(f"Edge pixel: {edge_pixel}")  # (127, 127, 127) - gray!
Section 8

The Reality: Float Precision

Computers don't have real numbers. They have Floating Point numbers. $\lim_{x \to 0} 1/x$ is Infinity, but in Python it's ZeroDivisionError or inf.

IEEE 754 Double

64 bits: 1 sign, 11 exponent, 52 mantissa

Range: $10^{-308}$ to $10^{308}$

Precision: ~15-16 decimal digits

Machine Epsilon

The smallest $\epsilon$ such that $1 + \epsilon \neq 1$

For double: $\approx 2.22 \times 10^{-16}$

Below this, additions are "invisible"

Why This Matters for Limits:
If you try to numerically evaluate a limit with steps smaller than machine epsilon,
you get garbage results. The computer can't represent the numbers you need.
Python Discovering Machine Epsilon 
import sys
import numpy as np

# Python's built-in
print(f"sys.float_info.epsilon: {sys.float_info.epsilon}")
# 2.220446049250313e-16

# NumPy's version
print(f"np.finfo(float).eps: {np.finfo(float).eps}")

# Manual discovery
eps = 1.0
while 1.0 + eps != 1.0:
    eps /= 2.0
print(f"Manually found: {eps * 2}")  # Multiply by 2 since we went too far

# DANGER ZONE: What happens below epsilon?
x = 1.0
y = 1.0 + 1e-17
print(f"1.0 == 1.0 + 1e-17? {x == y}")  # True! The difference is invisible
‚ö†Ô∏è Analogy: The Blind Spot
Machine epsilon is like a blind spot in your vision. Numbers too close together
look identical to the computer, even though they're mathematically different.

Glossary of Limit Concepts

Core Terms:
‚Ä¢ Limit: The value f(x) approaches as x ‚Üí c
‚Ä¢ Epsilon (Œµ): Error tolerance on output
‚Ä¢ Delta (Œ¥): Precision required on input
‚Ä¢ Continuous: No breaks, holes, or jumps
Theorems:
‚Ä¢ IVT: Continuous functions hit all intermediate values
‚Ä¢ Squeeze: Trap a limit between two known limits
‚Ä¢ L'H√¥pital: Differentiate 0/0 or ‚àû/‚àû forms

Common Gotchas

Watch Out For:
‚Ä¢ 0/0 ‚â† 1: It's indeterminate, not defined. Use other methods.
‚Ä¢ Left ‚â† Right: One-sided limits must match for a limit to exist.
‚Ä¢ Machine Epsilon: Computers can't represent infinitely small steps.
‚Ä¢ L'H√¥pital Abuse: Only use for 0/0 or ‚àû/‚àû forms!
