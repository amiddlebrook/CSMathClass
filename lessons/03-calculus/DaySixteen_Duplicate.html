The Art of Probability Arguments

Goal: Walk through complete probability arguments, understanding the justification for each step. This is how you learn to construct and critique probabilistic proofs.

Common justification types

  • Definition: "By definition of conditional probability..."
  • Independence: "Since events are independent..."
  • Linearity: "By linearity of expectation..."
  • Complement: "P(A) = 1 - P(not A)..."
  • Union bound: "P(A or B) ≤ P(A) + P(B)..."
  • Indicator trick: "Let Xᵢ = 1 if event i occurs..."
Example 1

Birthday Problem: P(shared birthday)

Problem: In a room of n people, what is the probability that at least two people share a birthday?

Complete Annotated Solution
P(at least one match) = 1 - P(all different)
Complement rule: It's easier to compute P(no match) and subtract from 1.
P(all different) = P(person 1 unique) × P(person 2 unique | person 1) × ...
Chain rule: P(A₁ and A₂ and ... Aₙ) = P(A₁) × P(A₂|A₁) × P(A₃|A₁,A₂) × ...
= (365/365) × (364/365) × (363/365) × ... × ((365-n+1)/365)
Counting: Person 1 has 365 options. Person 2 must avoid 1 date (364 options). Person k must avoid k-1 dates.
= 365! / ((365-n)! × 365ⁿ)
Simplification: Product of consecutive integers = falling factorial = 365!/(365-n)!
For n = 23: P(all different) ≈ 0.493
P(at least one match) ≈ 0.507
Numerical evaluation: Plugging in n=23 and computing. The 50% threshold is surprisingly low!
Key insight: The complement approach turned a complex "at least one" problem into a simpler "all different" problem. This is a powerful technique!
Example 2

Coupon Collector: Expected Time

Problem: There are n different coupons. Each draw gives a uniformly random coupon. How many draws to collect all n?

Complete Annotated Solution
Define phases: Phase i = waiting for the iᵗʰ new coupon (after having i-1 unique)
Problem decomposition: Break into phases to analyze separately.
In phase i: P(new coupon) = (n-i+1)/n
Simple counting: We have i-1 coupons, so n-(i-1) = n-i+1 are new.
Let Xᵢ = draws in phase i. Xᵢ ~ Geometric(pᵢ) where pᵢ = (n-i+1)/n
Geometric distribution: Waiting time for first success with success probability p.
E[Xᵢ] = 1/pᵢ = n/(n-i+1)
Known formula: For geometric distribution, E[X] = 1/p.
E[total] = E[X₁ + X₂ + ... + Xₙ] = Σᵢ E[Xᵢ]
Linearity of expectation: E[sum] = sum of E[each].
= n/n + n/(n-1) + n/(n-2) + ... + n/1
= n × (1/n + 1/(n-1) + ... + 1/1)
= n × Hₙ ≈ n ln n
Harmonic series: Hₙ = Σᵢ₌₁ⁿ 1/i ≈ ln n + γ where γ ≈ 0.577.
Example 3

QuickSort: Expected Comparisons

Problem: Prove QuickSort with random pivot makes O(n log n) expected comparisons.

Complete Annotated Solution
Let elements be z₁ < z₂ < ... < zₙ in sorted order.
Setup: Name elements by their rank for analysis.
Define Xᵢⱼ = 1 if zᵢ and zⱼ are compared, 0 otherwise.
Indicator random variables: Count comparisons using 0/1 variables.
Total comparisons X = Σᵢ<ⱼ Xᵢⱼ
Counting: Sum over all pairs.
E[X] = Σᵢ<ⱼ E[Xᵢⱼ]=Σᵢ<ⱼ P(zᵢ, zⱼ compared)
Linearity + indicator property: E[indicator] = P(event).
zᵢ and zⱼ compared ⟺ one of them is first pivot among {zᵢ, zᵢ₊₁, ..., zⱼ}
Key insight: Two elements compare only if one is picked as pivot before any element between them separates them.
P(zᵢ, zⱼ compared) = 2/(j-i+1)
Uniform probability: j-i+1 elements in range, 2 cause comparison.
E[X] = Σᵢ₌₁ⁿ⁻¹ Σⱼ₌ᵢ₊₁ⁿ 2/(j-i+1)
= Σᵢ₌₁ⁿ⁻¹ Σₖ₌₂ⁿ⁻ⁱ⁺¹ 2/k
≤ 2n × Hₙ = O(n log n)
Summation: Change variables k = j-i+1, bound by harmonic series.
Example 4

Bayes: Spam Classification

Problem: An email contains the word "FREE". Given: 80% of spam contains "FREE", 5% of ham (non-spam) contains "FREE", 20% of all emails are spam. What's P(spam | contains "FREE")?

Complete Annotated Solution
Define: S = spam, F = contains "FREE"
Given: P(F|S) = 0.80, P(F|¬S) = 0.05, P(S) = 0.20
Define notation: Clearly state what each probability represents.
Want: P(S|F)
State the goal: Probability of spam given the word appears.
P(S|F) = P(F|S) × P(S) / P(F)
Bayes' Theorem: Flip the conditional.
P(F) = P(F|S)×P(S) + P(F|¬S)×P(¬S)
= 0.80×0.20 + 0.05×0.80
= 0.16 + 0.04 = 0.20
Law of total probability: Sum over all cases (spam and ham).
P(S|F) = (0.80 × 0.20) / 0.20
= 0.16 / 0.20 = 0.80
Substitute and compute: 80% probability this email is spam!
Common error: Don't confuse P(F|S) = 0.80 with P(S|F). The word "FREE" appears in 80% of spam, but that doesn't mean 80% of emails with "FREE" are spam—until you compute it with Bayes!
