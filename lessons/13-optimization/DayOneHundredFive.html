Section 1

Direct vs Iterative

Gaussian Elimination (Direct) is $O(N^3)$. Too slow for N=100,000.

Iterative Methods approximate `x` by computing `Ax` repeatedly ($O(N^2)$ or faster if sparse).

Section 2

Krylov Subspaces

We search for solutions in the space spanned by `{b, Ab, A^2b, ...}`.

The idea is to find the "best" update based on how the matrix transforms vectors, without explicitly inverting it.
Section 3

Conjugate Gradient (CG)

For Symmetric Positive Definite matrices.

CG picks search directions that are "conjugate" (orthogonal under inner product defined by A). It guarantees convergence in N steps, but often gets very close in $\sqrt{κ}$ steps.

Section 4

Preconditioning

If κ(A) is huge, CG is slow.

Solve `M^{-1}Ax = M^{-1}b` where `M ≈ A` is easy to invert.

Good preconditioners (like Incomplete Cholesky) drastically reduce condition number.

Section 5

Python: Sparse Linear Algebra

Python Conjugate Gradient 
import numpy as np
from scipy.sparse import csc_matrix
from scipy.sparse.linalg import cg

# Large sparse matrix
A = csc_matrix([[3, 2, 0], [2, 3, 2], [0, 2, 3]])
b = np.array([1, 2, 1])

x, exit_code = cg(A, b)
print("Solution:", x)
# Much faster than dense solver for huge systems
