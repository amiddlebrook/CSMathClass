Section 1

The Two-Pass Model

Learning strictly linearly (Page 1 to 500) is inefficient. You get stuck on details before understanding the context.

Algorithm: 1. Pass 1 (The Scout): Survey the terrain. Gain vocabulary. Build a mental map. Ignore hard proofs. (Fast) 2. Pass 2 (The Engineer): Build the road. Go deep. Derive proofs. Solve hard problems. (Slow)
Section 2

Pass 1: The Scout

Your goal is familiarity, not mastery.

  • Read the Table of Contents.
  • Read Introductions and Conclusions.
  • Scan for bolded terms.
  • Look at the diagrams.

If you see a wall of math, skip it. Ask: "What is this TRYING to do?"

Section 3

Identifying Keywords

In Pass 1, collect "Black Boxes".

Example: You read "The kernel trick allows SVMs to classify non-linear data."

Do not stop to learn RKHS. Just Note: "Kernel Trick = magic for non-linear."

Section 4

Building the Map

At the end of Pass 1, draw a dependency graph.

"A depends on B." "C is a type of D."

This contextual knowledge makes Pass 2 10x faster because you know WHY you are learning each piece.

Section 5

Python: TF-IDF for Keywords

Python Basic Extractor 
from sklearn.feature_extraction.text import TfidfVectorizer

documents = [
    "Machine learning uses the kernel trick for non-linear classification.",
    "The kernel trick maps data to high dimensional space.",
    "Linear algebra is the foundation of machine learning."
]

# TF-IDF highlights unique important words
vectorizer = TfidfVectorizer(stop_words='english')
X = vectorizer.fit_transform(documents)

feature_names = vectorizer.get_feature_names_out()
dense = X.todense()

print("Top keywords per document:")
for i in range(len(documents)):
    # Get indices of top 3 words
    row = dense[i].tolist()[0]
    top_indices = sorted(range(len(row)), key=lambda i: row[i], reverse=True)[:3]
    print(f"Doc {i}: {[feature_names[j] for j in top_indices]}")
