Compound Interest for Skills

Knowledge compounds. Learning 1% a day means you are 37x better in a year.

1.01^{365} \approx 37.8

However, 0.99^365 is 0.03. Atrophy is real. You must touch the material daily.

The Compound Interest Analogy: Learning is like putting money in a high-interest savings account. The $1 you put in today isn't worth much tomorrow. But if you keep adding just $1 every single day, after a few years, the **interest alone** is paying for your entire lifestyle. Knowledge works exactly the same wayâ€”the foundations you learn today make everything you learn next year 10x easier to digest.
Section 2

Don't Break the Chain

Jerry Seinfeld's advice to a young comic: Write a joke every day. Mark an X on the calendar. After a few days, you have a chain. Your only job is: Don't break the chain.

ðŸŽ¡
The Flywheel Analogy: Getting started with a math routine is like trying to spin a 10-ton granite flywheel. The first 100 pushes are exhausting and it barely moves. But once it starts spinning, its own **momentum** keeps it going. Suddenly, it's harder to stop it than it is to keep it spinning.
Rule: Never miss two days in a row. Life happens, but missing twice starts a new habit of not doing it.
Section 3

Time Blocking

Don't "find" time. Make time.

Schedule a 2-hour Deep Work block. Protect it with your life. Turn off phone. Turn off WiFi if needed.

Maker vs Manager Schedule: You need long, uninterrupted blocks (Maker) to learn complex math.

Section 4

Environment Design

Willpower is finite. Design your environment so you don't need it.

  • Physical: Clean desk. Books ready. Notebook open.
  • Digital: Block distraction sites. Use a "Focus" mode.
  • Social: Tell people you are busy during your block.
ðŸªž
The Mirror Analogy: Your environment is a mirror of your intentions. If your desk is covered in distractions, your environment is "telling" you to be distracted. If your desk has only one book and a notebook open to the current page, your environment is **pre-loading** your next decision toward success.

Why This Matters in CS

Technical Debt vs Wealth: Just like habits, technical decisions compound. Small, clean code fixes made daily are "Technical Wealth" that pay dividends in speed later. Skipping tests or "cramming" bad code today is "Technical Debt" that compounds with interest until the whole project grinds to a halt. Consistent refactoring is the "Seinfeld Strategy" of software engineering.

Python: Habit Tracker

Python Simple Streak Logic 
import datetime

class HabitTracker:
    def __init__(self, name):
        self.name = name
        self.history = [] # List of dates
        
    def check_in(self):
        today = datetime.date.today()
        if not self.history or self.history[-1] != today:
            self.history.append(today)
            print(f"Checked in for {self.name}!")
        else:
            print("Already checked in today.")
            
    def current_streak(self):
        if not self.history: return 0
        
        streak = 0
        today = datetime.date.today()
        check_date = today
        
        # Check backward from today
        idx = len(self.history) - 1
        while idx >= 0:
            if self.history[idx] == check_date:
                streak += 1
                check_date -= datetime.timedelta(days=1)
                idx -= 1
            elif self.history[idx] == check_date - datetime.timedelta(days=1):
                 # Allow checking in yesterday if missed today (logic varies)
                 pass
            else:
                break
        return streak

math_habit = HabitTracker("Math Study")
math_habit.check_in()
print(f"Current Streak: {math_habit.current_streak()} days")
