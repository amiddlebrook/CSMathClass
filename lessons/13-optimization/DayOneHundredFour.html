Section 1

Floating Point

Computers cannot store real numbers exactly. They store `sign * mantissa * 2^exp` (IEEE 754).

Catastrophic Cancellation: Subtracting two nearly identical numbers leads to massive loss of precision. `1.000001 - 1.000000 = 0.000001` (most significant digits lost).
Section 2

Condition Number κ

How much does the output change given a small change in input?

κ(A) = ||A|| * ||A^{-1}|| = σ_max / σ_min

If `κ` is large (e.g. $10^{16}$), the matrix is ill-conditioned. Solving `Ax=b` becomes unreliable.

Section 3

Numerical Stability

An algorithm is stable if small errors in input (or roundoff) do not blow up.

Example: Gaussian Elimination needs pivoting (swapping rows) to be stable. Without pivoting, dividing by small numbers causes explosion.
Section 4

SVD Connection

Singular Value Decomposition (SVD) reveals the condition of a matrix.

Small singular values mean the matrix is "almost" singular (close to non-invertible).

Section 5

Python: Condition Number

Python Calculate κ 
import numpy as np

# Identical rows = Singular = Infinite Cond
A = np.array([[1, 1], [1, 1.00000001]])
cond = np.linalg.cond(A)

print(f"Condition Number: {cond}")
# Will be huge (approx 10^8)
# This means we lose ~8 digits of precision solving Ax=b
