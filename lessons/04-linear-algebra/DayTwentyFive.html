Section 1

What is a Relation?

A binary relation $R$ from set $A$ to set $B$ is a subset of the Cartesian product $A \times B$. If $(a, b) \in R$, we say "a is related to b" ($aRb$).

Example: "Less than" on {1, 2, 3} R = {(1,2), (1,3), (2,3)}

Functions are just special types of relations where every $a \in A$ appears exactly once as a first element.

Section 2

Key Properties

For a relation $R$ on a set $A$ (i.e., $R \subseteq A \times A$):

Reflexive: $\forall a \in A, (a,a) \in R$.
Example: $\leq$ (since $x \leq x$).
Symmetric: If $(a,b) \in R$, then $(b,a) \in R$.
Example: "=" or "is sibling of".
Antisymmetric: If $(a,b) \in R$ and $(b,a) \in R$, then $a=b$.
Example: $\leq$ (If $x \le y$ and $y \le x$, then x=y).
Transitive: If $(a,b) \in R$ and $(b,c) \in R$, then $(a,c) \in R$.
Example: "Ancestor of".
Section 3

Equivalence Relations

An Equivalence Relation is Reflexive, Symmetric, and Transitive.

It generalizes the concept of equality. It partitions the set $A$ into disjoint Equivalence Classes.

Example: Modulo 3 Integers related if x % 3 == y % 3. Classes: [0] = {..., -3, 0, 3, 6, ...} [1] = {..., -2, 1, 4, 7, ...} [2] = {..., -1, 2, 5, 8, ...}
Section 4

Partial Orders

A Partial Order is Reflexive, Antisymmetric, and Transitive.

We call the set a Poset. It captures the idea of "precedence" or "hierarchy".

Hasse Diagrams

A visual way to represent finite posets. We draw $b$ above $a$ if $a < b$, and connect them avoiding redundant transitive edges.

Total Order: A partial order where every pair is comparable (e.g., $\leq$ on Integers).
Lattice: A poset where every pair has a Least Upper Bound (Join) and Greatest Lower Bound (Meet).
Section 5

Relations in Python

We can model a relation as a set of tuples or a boolean matrix.

Python Analyzing Relation Properties 
def is_reflexive(R, A):
    for a in A:
        if (a, a) not in R:
            return False
    return True

def is_symmetric(R):
    for (a, b) in R:
        if (b, a) not in R:
            return False
    return True

def is_transitive(R):
    for (a, b) in R:
        for (c, d) in R:
            # If we have (a,b) and (b,c), we need (a,c)
            if b == c and (a, d) not in R:
                return False
    return True

# Define Set A and Relation R
A = {1, 2, 3}
R = {(1,1), (2,2), (3,3), (1,2), (2,1)}

print(f"Reflexive? {is_reflexive(R, A)}")  # True
print(f"Symmetric? {is_symmetric(R)}")    # True
print(f"Transitive? {is_transitive(R)}")  # False! (1,2) and (2,1) exist, but (1,1) is needed (it is there)
# Wait, (1,2) and (2,1) -> needs (1,1) and (2,2). Both are there.
# Let's verify manually: 1->2, 2->1. Transitive needs 1->1. YES.
# Actually, transitivity holds for this R.
