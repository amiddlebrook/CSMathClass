Section 1

Regular Languages & DFAs

A Regular Language is the simplest class of computational languages. It can be recognized by a Deterministic Finite Automaton (DFA).

M = (Q, Σ, δ, q₀, F) • Q: Finite set of states • Σ: Alphabet • δ: Transition function Q × Σ → Q • q₀: Start state • F: Accept states

NFA (Nondeterministic): Can have multiple paths. Surprisingly, NFA = DFA in power (subset construction).

Section 2

The Pumping Lemma

How do we prove a language is NOT regular?

Pumping Lemma: If L is regular, there exists a length `p` such that any string `s` with `|s| ≥ p` can be split into `xyz` where: 1. `|xy| ≤ p` 2. `y` is not empty 3. `xyⁱz ∈ L` for all `i ≥ 0`.
Classic Example: `L = {0ⁿ1ⁿ | n ≥ 0}` is NOT regular. (You can't "count" to arbitrary `n` with finite states).
Section 3

Context-Free Grammars (CFG)

To capture nested structures (like `0ⁿ1ⁿ` or programming language syntax), we need a Stack.

Pushdown Automaton (PDA) = NFA + Stack

CFG Rules: `A → α` where A is a variable and α is a string of vars/terminals.

Section 4

Python: Simple FSM

Implementing a Finite State Machine to recognize even numbers of 1s.

Python Even Parity DFA 
class DFA:
    def __init__(self):
        self.state = 'EVEN'

    def process(self, string):
        self.state = 'EVEN'
        for char in string:
            if self.state == 'EVEN':
                if char == '1': self.state = 'ODD'
            else: # ODD
                if char == '1': self.state = 'EVEN'
        return self.state == 'EVEN'

dfa = DFA()
print(dfa.process("101"))   # True (two 1s)
print(dfa.process("111"))   # False (three 1s)
