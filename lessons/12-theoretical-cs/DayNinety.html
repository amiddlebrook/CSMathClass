Section 1

Probabilistic Turing Machines

A PTM can access a stream of random bits. It outputs correct answers with high probability.

Note: The definition of "Efficient" changes. We accept small error probabilities (like `2^{-100}`) because they are negligible in practice.
Section 2

BPP (Bounded-Error Probabilistic P)

BPP: Solvable in poly-time with error probability ≤ 1/3 (two-sided error).

By running the algorithm `k` times and taking the majority vote, error drops exponentially (Chernoff bound).
Section 3

RP and ZPP

RP (Randomized Polynomial): • If Answer is NO: Always output NO. • If Answer is YES: Output YES with prob ≥ 1/2.

ZPP (Zero-Error Probabilistic P): Always correct, but runtime is random (expected poly-time). ZPP = RP ∩ coRP.

Section 4

Derandomization

Major Open Question: Does **P = BPP**?

Many computer scientists believe Yes. Strong pseudo-random number generators could simulate any randomized algorithm deterministically.

Primes is in P (AKS test), but was believed to be in BPP for decades (Miller-Rabin).
