Section 1

The PCP Theorem

Probabilistically Checkable Proofs.

Theorem: NP = PCP(log n, 1).

This means any mathematical proof (of length `L`) can be rewritten in a format where a verifier can check it with 99% confidence by looking at only a constant number of bits!

Section 2

Hardness of Approximation

The PCP theorem revolutionized optimization. It implies that for many NP-Hard problems (like Max-Clique), finding even an approximate solution is NP-Hard.

It creates a gap: If a formula is satisfiable, we can satisfy 100% of clauses. If not, we can satisfy at most 90%. Detecting the difference is easy (constant queries), so finding the opt is hard.

Section 3

Interactive Proofs

What if the prover and verifier can talk? `IP` is the class of problems solvable this way.

IP = PSPACE

This means interactive proofs are incredibly powerful—able to prove things like "this Chess position is a win" (in principle).

Section 4

Zero Knowledge Proofs

A proof that conveys zero information other than the validity of the statement.

Example: Proving you know a graph coloring without revealing the colors. Crucial for crypto (Z-Cash, Authentication).

Section 5

Python: 3-SAT Verifier

A simple verifier for a 3-SAT assignment.

Python SAT Checker 
def is_satisfied(clauses, assignment):
    """
    clauses: list of tuples (e.g., [(1, 2, -3), ...])
    assignment: dict {1: True, 2: False, ...}
    """
    for clause in clauses:
        clause_sat = False
        for lit in clause:
            var = abs(lit)
            val = assignment.get(var, False)
            if lit < 0: val = not val
            
            if val:
                clause_sat = True
                break
        
        if not clause_sat:
            return False # Failed constraint
    return True

# Example (x1 OR x2 OR NOT x3)
clauses = [(1, 2, -3)]
sol = {1: False, 2: False, 3: True} # 0 0 0 -> Fail
print("Satisfied?", is_satisfied(clauses, sol))
