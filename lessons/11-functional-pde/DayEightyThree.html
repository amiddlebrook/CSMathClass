Section 1

Floating Point Arithmetic

Computers do not do real arithmetic. They do floating point arithmetic (IEEE 754).

fl(x) = x(1 + \epsilon), \quad |\epsilon| \le \epsilon_{mach}

For double precision (64-bit), machine epsilon `\epsilon_{mach} \approx 10^{-16}`. This small error can explode.

Section 2

Condition Number

The Condition Number `\kappa(f)` measures how sensitive a function is to input perturbations.

\text{Relative Output Error} \approx \kappa(f) \times \text{Relative Input Error}

Matrices

For solving `Ax=b`:

\kappa(A) = \|A\| \cdot \|A^{-1}\| = \frac{\sigma_{max}}{\sigma_{min}}

If `\kappa(A) = 10^k`, you lose `k` digits of precision in the solution.

Section 3

Algorithm Stability

Conditioning is a property of the problem. Stability is a property of the algorithm.

  • Forward Stable: The computed answer is close to the true answer.
  • Backward Stable: The computed answer is the exact answer to a slightly wrong problem.
  • Unstable: Errors magnify uncontrollably (e.g., Catastrophic Cancellation).
Section 4

The Hilbert Matrix

The canonical example of an ill-conditioned matrix. `H_{ij} = \frac{1}{i+j-1}`.

Even for `n=10`, `\kappa(H) \approx 10^{13}`. Solving `Hx=b` is nearly impossible with standard precision.

Section 5

Python: Catastrophic Cancellation

Solving a Hilbert matrix system shows how precision vanishes.

Python Ill-Conditioned System 
import numpy as np
from scipy.linalg import hilbert, solve
import matplotlib.pyplot as plt

n = 10
H = hilbert(n)
print(f"Condition Number (n={n}): {np.linalg.cond(H):.2e}")

# Create a known solution x_true = [1, 1, ..., 1]
x_true = np.ones(n)
b = H @ x_true

# Solve Hx = b
x_comp = solve(H, b)

# Check error
error = np.linalg.norm(x_comp - x_true)
print(f"Error: {error:.2e}")

# Visualize solution
plt.plot(x_true, 'o-', label='True')
plt.plot(x_comp, 'x--', label='Computed')
plt.title(f'Solving Hilbert(10) - Correct? Almost.')
plt.ylim(0, 2)
plt.legend()
plt.show()
