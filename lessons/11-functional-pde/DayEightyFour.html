Section 1

Direct vs Iterative

  • Direct (Gaussian Elimination): Exact step count (`O(n^3)`). Modifies `A` (fill-in). Bad for massive sparse matrices.
  • Iterative: Approximate solution. Matrix-vector products only (`O(nnz)`). Good for massive sparse matrices.
Section 2

Krylov Subspaces

Modern iterative solvers search for the solution in the Krylov Subspace:

\mathcal{K}_k(A, b) = \text{span}\{b, Ab, A^2b, \dots, A^{k-1}b\}

Why? Because `A^k b` is easy to compute if `A` is sparse!

Section 3

Conjugate Gradient (CG)

For Symmetric Positive Definite (SPD) matrices. CG finds the optimal solution in `\mathcal{K}_k` that minimizes the A-norm error.

Magic Property

Unlike standard Gradient Descent which can zig-zag, CG takes "conjugate" steps. It guarantees convergence in `n` steps (in exact arithmetic), but we usually stop much earlier.

Section 4

GMRES

Generalized Minimal Residual. Works for non-symmetric matrices.

It minimizes the residual `\|Ax_k - b\|_2`. However, it explicitly builds an orthonormal basis (Arnoldi iteration), so memory usage grows with `k`. We often "restart" it (GMRES(k)).

Section 5

Preconditioning

Convergence depends on `\kappa(A)`. If `A` is ill-conditioned, CG is slow.

Idea: Solve `M^{-1}Ax = M^{-1}b` where `M \approx A` but easy to invert. `M` is the Preconditioner.

A good preconditioner makes `\kappa(M^{-1}A) \ll \kappa(A)`.
Section 6

Python: Solving Sparse Systems

Python Sparse CG (SciPy) 
import numpy as np
import scipy.sparse as sp
import scipy.sparse.linalg as spla
import time

# 1. Create a large sparse SPD matrix (Poisson 1D Laplacian)
n = 10000
diagonals = [np.ones(n) * 2, -np.ones(n-1), -np.ones(n-1)]
A = sp.diags(diagonals, [0, -1, 1], format='csr')
b = np.ones(n)

print(f"Matrix shape: {A.shape}, Non-zeros: {A.nnz}")

# 2. Solve with Conjugate Gradient
start = time.time()
x, exit_code = spla.cg(A, b, tol=1e-5)
end = time.time()

print(f"Solved in {end-start:.4f}s")
print(f"Exit code (0=success): {exit_code}")
print(f"Residual norm: {np.linalg.norm(b - A@x):.2e}")
