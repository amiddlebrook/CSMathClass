Section 1

The Language of Math

In CS and advanced math, ambiguity is a bug. We use specific symbols to be precise. These symbols are the "shorthand" of logic, letting us communicate complex ideas with minimal characters.

Quantifiers (Universal & Existential)

∀ (For All)
Universal quantifier. Think of it as All.
"∀x ∈ S" means "for every x in the set S".
∃ (There Exists)
Existential quantifier. Think of it as Exists.
"∃x ∈ S" means "there is at least one x in S".
∀x ∈ ℝ, x² ≥ 0 "For all real numbers x, x squared is greater than or equal to 0." ∃x ∈ ℤ, x² = 25 "There exists an integer x such that x squared is 25."

Implication (The If-Then Arrow)

The arrow ⇒ is shorthand for "implies". P ⇒ Q means "If P is true, then Q must also be true."

Analogy: "It is raining" ⇒ "The ground is wet". The rain causes the wet ground, but wet ground doesn't always mean it rained (someone could have spilled a bucket!).
x > 10 ⇒ x > 5 "If x is greater than 10, it implies x is greater than 5."
Section 2

Set Theory Basics

A set is a collection of distinct objects. It is the fundamental data structure of mathematics, much like a list or a dictionary in programming, but with unique rules.

Notation & Membership

The Math Bag Analogy: A "set" is just a container or a bag. x ∈ A means "x is inside bag A".
  • x ∈ A: x is an element of set A.
  • A ⊆ B: A is a subset of B (every element of A is also in B).
  • ∅: The empty set (an empty bag).

Operations (Union, Intersection)

Union (∪)
Combine everything into one big bag. (A OR B).
Intersection (∩)
Only take the items that are in both bags. (A AND B).
PythonSet operations map directly to code
A = {1, 2, 3}
B = {3, 4, 5}

# Union (A ∪ B)
print(A | B)  # {1, 2, 3, 4, 5}

# Intersection (A ∩ B)
print(A & B)  # {3}
Section 3

Number Systems

Not all numbers are created equal. We classify them into sets with specific properties. Think of them like nesting dolls—some sets are entirely contained within others.

The Hierarchy of Numbers

Symbol Name Description
ℕ Natural Numbers {0, 1, 2, ...} (Counting numbers)
ℤ Integers {..., -1, 0, 1, ...} (Whole numbers + Negatives)
ℚ Rationals p/q where p,q ∈ ℤ, q≠0 (Fractions)
ℝ Reals All points on the number line
ℕ ⊂ ℤ ⊂ ℚ ⊂ ℝ
Remember: Every Natural number is an Integer, and every Integer is a Rational (e.g., 5 = 5/1).
Section 4

Rigorous Arithmetic

Divisibility (Perfect Fits)

We say a | b ("a divides b") if b / a is a whole number with no remainder.

3 | 12 because 12 = 3 × 4 3 ∤ 13 because no integer k satisfies 13 = 3k

Modulo (The Leftovers)

The Mod operator (%) gives you the remainder after division.

Clock Analogy: On a 12-hour clock, 13:00 is just 13 \pmod{12} = 1.
PythonModulo in action
def is_even(n):
    return n % 2 == 0

index = 12 % 5   # Result: 2
Section 5

Algebraic Structures

Algebra isn't just about solving for \(x\); it's about finding rules for how types of numbers behave. We group these rules into Structures.

Fields vs. Rings

Field (ℝ, ℚ, ℂ): A structure where you can do all "Big Four" operations: \(+\), \(-\), \(*\), and \(/\) (except by 0). Think Fractions.
Ring (ℤ): A structure where you can \(+\), \(-\), and \(*\), but division might fail to stay in the set. Think Integers (\(5/2\) isn't an integer).
Why it matters: This is why in programming, 5 / 2 might return 2.5 (Field logic) or 2 (Ring/Integer logic). Knowing your structure prevents bugs!
