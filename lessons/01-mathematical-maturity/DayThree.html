Section 1

Function Anatomy

A function $f: A \to B$ assigns exactly one element of $B$ to each element of $A$.

Components

  • Domain ($A$): The set of valid inputs.
  • Codomain ($B$): The set of possible outputs.
  • Range (Image): The set of actual outputs produced. Rules: Range $\subseteq$ Codomain.
PythonType hints reveal function structure
# f: int -> str
def int_to_binary(x: int) -> str:
    return bin(x)

# Domain: Set of all integers (int)
# Codomain: Set of all strings (str)
# Range: Set of strings starting with "0b" followed by 0s and 1s
Section 2

Types of Functions

1. Injective (One-to-One)

Every input maps to a unique output. No collisions.

Definition: f(x) = f(y) ⇒ x = y Test: Horizontal Line Test (graph hits at most once). CS Context: Hash functions (ideally), unique IDs.

2. Surjective (Onto)

Every element in the Codomain is covered. Logic: The Range equals the Codomain.

Definition: ∀y ∈ B, ∃x ∈ A such that f(x) = y. CS Context: A compression algorithm that can generate any possible file is surjective? No, usually not.

3. Bijective (One-to-One AND Onto)

Perfect pairing. Invertible.

Theorem: A function is invertible if and only if it is Bijective.
Section 3

Composition & Inverse

Composition

$(f \circ g)(x) = f(g(x))$. Output of g becomes input of f.

Order Matters: $f \circ g \neq g \circ f$ usually.

Inverse Functions

If $f: A \to B$ is bijective, $f^{-1}: B \to A$ reverses the mapping.

f(f^{-1}(y)) = y f^{-1}(f(x)) = x
Section 4

Relations

Start with set $A$. A connection between elements is a statement $xRy$ (x is related to y). Formally, a relation $R$ is a subset of $A \times A$.

Properties

Reflexive $\forall x, xRx$ Everyone relates to themselves (e.g. $\leq$, $=$)
Symmetric $xRy \implies yRx$ Two-way street (e.g. "is sibling of")
Transitive $xRy \land yRz \implies xRz$ Chain rule (e.g. $<$, $\leq$, "ancestor of" )
Section 5

Equivalence Relations

A relation that is Reflexive, Symmetric, and Transitive is an Equivalence Relation. It behaves like "equality".

Modular Arithmetic

The relation $a \equiv b \pmod n$ is an equivalence relation on $\mathbb{Z}$.

  • Reflexive: $a \equiv a \pmod n$ (Since $n | 0$).
  • Symmetric: If $a \equiv b$, then $b \equiv a$.
  • Transitive: If $a \equiv b$ and $b \equiv c$, then $a \equiv c$.

Partitions

An equivalence relation slices the set $A$ into disjoint Equivalence Classes. For modulo 3, the integers are sliced into:

[0] = {..., -3, 0, 3, 6, ...} [1] = {..., -2, 1, 4, 7, ...} [2] = {..., -1, 2, 5, 8, ...}
