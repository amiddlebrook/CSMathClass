Section 1

Function Anatomy

A function $f: A \to B$ assigns exactly one element of $B$ to each element of $A$.

Think of a function like a machine: you put something in, and you get exactly one thing out. It is the mathematical version of a programming function that has no side effects.

The Three Parts

Analogy: A vending machine. The Domain is the coins/codes you enter. The Codomain is the category "Snacks". The Range is the specific chips and sodas actually available inside.
  • Domain ($A$): The set of all valid inputs.
  • Codomain ($B$): The set of potential output types.
  • Range: The set of actual outputs produced ($f(A)$).

Section 2

Types of Functions

Functions can behave in specific ways depending on how they map inputs to outputs.

1. Injective (Unique mapping)

No two different inputs map to the same output. No collisions.

Analogy: A fingerprint scanner. Every unique finger maps to a unique ID.
f(a) = f(b) \implies a = b

2. Surjective (onto)

Every element in the Codomain is "hit" by at least one input. The Range is the entire Codomain.

Analogy: A bus where every single seat is taken. (The seats are the codomain).
\forall y \in B, \exists x \in A \text{ s.t. } f(x) = y

3. Bijective (Perfect pairs)

Both Injective and Surjective. Every input has exactly one partner, and vice versa. These functions are invertible.

Section 3

Composition & Inverse

Functions can be chained together or reversed if they are bijections.

Composition

$(f \circ g)(x) = f(g(x))$. The output of the inner function becomes the input of the outer function.

Order Matters: Doing your socks then your shoes is not the same as shoes then socks! $f(g(x)) \neq g(f(x))$.
Section 4

Relations

A Relation is a general way to link two things together (e.g. "is friend of", "is taller than", or "is divisible by").

The Three Pillar Properties

For a relation to behave like "Equality", it needs three specific traits:

Reflexive $\forall x, xRx$ You relate to yourself. (\(x = x\))
Symmetric $xRy \implies yRx$ A two-way street. (If I'm your sibling, you're mine).
Transitive $xRy \land yRz \implies xRz$ The chain rule. (If \(A < B\) and \(B < C\), then \(A < C\)).
Section 5

Equivalence Relations

If a relation is Reflexive, Symmetric, and Transitive, it is an Equivalence Relation. It slices the world into buckets where everything in a bucket is "the same" under that relation.

Equivalence Classes

In Modulo 3, every number has a "remainder bucket":

[0] = \{0, 3, 6, 9, ...\} \text{ (Divisible by 3)} [1] = \{1, 4, 7, 10, ...\} \text{ (Remainder 1)} [2] = \{2, 5, 8, 11, ...\} \text{ (Remainder 2)}
