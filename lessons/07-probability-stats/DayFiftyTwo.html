Section 0

Why Concentration Inequalities

Expected value tells you what happens "on average." But for algorithm analysis, we need stronger guarantees: what's the probability that our algorithm is far from its expected behavior?

Key Question: What is P(X deviates far from E[X])? Concentration inequalities give upper bounds on this probability.

The concentration hierarchy

WEAKEST → STRONGEST BOUNDS: Markov: P(X ≥ a) ≤ E[X]/a Only needs X ≥ 0 Chebyshev: P(|X - μ| ≥ t) ≤ σ²/t² Needs variance Chernoff: P(|X - μ| ≥ t) ≤ 2·exp(-t²/2n) Needs independence + boundedness EXPONENTIALLY small!
Section 1

Markov's Inequality

Statement

Markov's Inequality: For non-negative X and a > 0:
P(X ≥ a) ≤ E[X]/a
Example: E[X] = 10 P(X ≥ 50) ≤ 10/50 = 0.2 = 20% P(X ≥ 100) ≤ 10/100 = 0.1 = 10% Very weak but very general! Only requires X ≥ 0.

Proof sketch

E[X] = ∫ x·f(x)dx ≥ ∫_{x≥a} x·f(x)dx ≥ a·∫_{x≥a}f(x)dx = a·P(X≥a) Therefore: P(X ≥ a) ≤ E[X]/a
Markov is often loose: It can't be tight in general because it uses no information about the distribution beyond the mean. Use it when you have nothing else, or to prove other bounds.
Section 2

Chebyshev's Inequality

Statement

Chebyshev's Inequality: For any X with mean μ and variance σ²:
P(|X - μ| ≥ kσ) ≤ 1/k²
Or equivalently: P(|X - μ| ≥ t) ≤ σ²/t² Standard deviations from mean: k = 2: P(|X - μ| ≥ 2σ) ≤ 1/4 = 25% k = 3: P(|X - μ| ≥ 3σ) ≤ 1/9 ≈ 11% k = 4: P(|X - μ| ≥ 4σ) ≤ 1/16 ≈ 6% k = 10: P(|X - μ| ≥ 10σ) ≤ 1/100 = 1%

Proof (from Markov)

Let Y = (X - μ)² Then Y ≥ 0, so apply Markov: P(|X - μ| ≥ t) = P((X - μ)² ≥ t²) = P(Y ≥ t²) ≤ E[Y]/t² = Var(X)/t² = σ²/t² ✓
Python Comparing bounds 
import numpy as np

# Compare Chebyshev bound to actual probability
n = 100
samples = np.random.binomial(n, 0.5, 100000)
mu, sigma = n*0.5, np.sqrt(n*0.25)

k = 3  # 3 standard deviations
chebyshev_bound = 1/k**2
actual_prob = np.mean(np.abs(samples - mu) >= k*sigma)

print(f"Chebyshev bound: {chebyshev_bound:.4f}")  # 0.1111
print(f"Actual probability: {actual_prob:.4f}")  # ~0.003
# Chebyshev is very conservative for bell-shaped distributions!
Section 3

Chernoff Bounds

Statement

Chernoff Bound: For X = X₁ + ... + Xₙ where Xᵢ are independent [0,1]-valued RVs with μ = E[X]:
MULTIPLICATIVE FORM: P(X ≥ (1+δ)μ) ≤ exp(-δ²μ/(2+δ)) for δ > 0 P(X ≤ (1-δ)μ) ≤ exp(-δ²μ/2) for 0 < δ < 1 ADDITIVE FORM (Hoeffding): P(|X - μ| ≥ t) ≤ 2·exp(-2t²/n) Exponentially small deviation!

Why Chernoff is powerful

Chebyshev: P(|X-μ| ≥ t) ≤ σ²/t² (polynomial decay) Chernoff: P(|X-μ| ≥ t) ≤ 2·exp(-2t²/n) (exponential decay) Example: n = 1000 coin flips, μ = 500, want P(X ≥ 600) Chebyshev: P(X ≥ 600) ≤ σ²/(100)² = 250/10000 = 2.5% Chernoff: P(X ≥ 600) ≤ exp(-2·100²/1000) ≈ 2×10⁻⁹ Chernoff is exponentially tighter!
Python Chernoff bound application 
import numpy as np

def chernoff_bound(n, p, delta):
    """Multiplicative Chernoff bound for Binomial(n,p)"""
    mu = n * p
    return np.exp(-delta**2 * mu / (2 + delta))

def hoeffding_bound(n, t):
    """Additive bound for sum of n [0,1] RVs"""
    return 2 * np.exp(-2 * t**2 / n)

# Example: 1000 coin flips, probability of ≥600 heads
n, p = 1000, 0.5
delta = (600 - 500) / 500  # = 0.2

print(f"Chernoff: {chernoff_bound(n, p, delta):.2e}")  # ~1.7e-9
print(f"Hoeffding: {hoeffding_bound(n, 100):.2e}")    # ~4e-9

# Algorithm analysis: random sampling
# Want estimate within ε of true mean with probability 1-δ
# Need n ≥ log(2/δ)/(2ε²) samples
epsilon, delta = 0.05, 0.01
n_samples = int(np.log(2/delta) / (2*epsilon**2))
print(f"Need {n_samples} samples for 99% accuracy within 5%")
Related bounds: Hoeffding (bounded RVs), McDiarmid (bounded differences), Azuma (martingales). All give exponential concentration.
