Why Probability Matters in CS

Probability is the language of uncertainty. In computer science, we use it everywhere: analyzing randomized algorithms, training machine learning models, modeling networks, and making decisions under incomplete information.

Key Insight: Probability lets us reason precisely about random events and make optimal decisions even when the outcome is uncertain.

Where you'll use this

  • Algorithms: Randomized quicksort, hash table analysis, primality testing
  • Machine Learning: Bayesian inference, probabilistic models, sampling
  • Systems: Network reliability, queuing theory, load balancing
  • Security: Cryptographic protocols, randomness generation
Section 1

Sample Spaces & Events

Core definitions

SAMPLE SPACE Ω: Set of all possible outcomes EVENT A: Subset of Ω (collection of outcomes) Example (rolling a die): Ω = {1, 2, 3, 4, 5, 6} A = "even number" = {2, 4, 6} P(A) = |A|/|Ω| = 3/6 = 1/2

Event operations

A ∪ B: A OR B (union) A ∩ B: A AND B (intersection) Aᶜ: NOT A (complement) A \ B: A but not B
A ⊆ B: A implies B A ∩ B = ∅: A, B mutually exclusive Ω: certain event ∅: impossible event
Python Sample spaces as sets 
# Sample space for two coin flips
omega = {('H', 'H'), ('H', 'T'), ('T', 'H'), ('T', 'T')}

# Event: at least one head
A = {x for x in omega if 'H' in x}
# {('H', 'H'), ('H', 'T'), ('T', 'H')}

# Probability (uniform distribution)
prob_A = len(A) / len(omega)  # 3/4

# Event operations
B = {x for x in omega if x[0] == 'H'}  # First is heads
A_and_B = A & B         # Intersection
A_or_B = A | B          # Union
not_A = omega - A       # Complement
Section 2

Probability Axioms

Kolmogorov Axioms

The three axioms: All of probability theory derives from these.
1. NON-NEGATIVITY: P(A) ≥ 0 for all events A 2. NORMALIZATION: P(Ω) = 1 3. ADDITIVITY: If A ∩ B = ∅ (mutually exclusive), then P(A ∪ B) = P(A) + P(B)

Derived rules

From the axioms, we can prove: • P(∅) = 0 • P(Aᶜ) = 1 - P(A) • P(A) ≤ 1 • P(A ∪ B) = P(A) + P(B) - P(A ∩ B) (inclusion-exclusion) • If A ⊆ B, then P(A) ≤ P(B)
Common mistake: P(A ∪ B) = P(A) + P(B) is ONLY true when A and B are mutually exclusive. For general events, you must subtract P(A ∩ B) to avoid double-counting.
Section 3

Counting Principles

For uniform distributions, P(A) = |A|/|Ω|. The hard part is counting |A| and |Ω|.

Permutations (order matters)

n! = n × (n-1) × ... × 1 P(n,k) = n!/(n-k)! = ways to arrange k items from n Example: 3 medals from 8 runners P(8,3) = 8!/5! = 8×7×6 = 336

Combinations (order doesn't)

C(n,k) = n!/(k!(n-k)!) = "n choose k" = (n k) Example: 3 winners from 8 people C(8,3) = 8!/(3!5!) = 56
Python Counting in Python 
from math import factorial, comb, perm
from itertools import permutations, combinations

# Factorial
factorial(5)  # 120

# Permutations: P(n,k)
perm(8, 3)  # 336
list(permutations('ABC', 2))  # [('A','B'), ('A','C'), ...]

# Combinations: C(n,k)
comb(8, 3)  # 56
list(combinations('ABC', 2))  # [('A','B'), ('A','C'), ('B','C')]

# Probability example: 5-card poker hand
total_hands = comb(52, 5)  # 2,598,960
flush_hands = 4 * comb(13, 5)  # 5,148 (any suit, choose 5)
prob_flush = flush_hands / total_hands  # ~0.198%
Section 4

Conditional Probability & Bayes

Conditional probability

P(A|B): Probability of A, given that B occurred.
P(A|B) = P(A ∩ B) / P(B) Intuition: Restrict sample space to B, then find proportion where A also occurs. Example: Roll two dice, sum is 8. What's P(first die = 3)? B = {sum is 8} = {(2,6), (3,5), (4,4), (5,3), (6,2)} A = {first = 3} A ∩ B = {(3,5)} P(A|B) = 1/5

Bayes' Theorem

P(A|B) = P(B|A) · P(A) / P(B) Alternative form (with partition): P(A|B) = P(B|A)P(A) / [P(B|A)P(A) + P(B|Aᶜ)P(Aᶜ)] TERMS: • P(A): prior (before seeing evidence) • P(A|B): posterior (after seeing evidence) • P(B|A): likelihood (evidence given hypothesis) • P(B): normalizing constant
Bayes' theorem is the foundation of Bayesian statistics and probabilistic machine learning. It tells us how to update beliefs with evidence.

Independence

A and B are INDEPENDENT if any of these hold: • P(A ∩ B) = P(A) · P(B) • P(A|B) = P(A) • P(B|A) = P(B) Independence ≠ mutually exclusive! Mutually exclusive: P(A ∩ B) = 0 Independent: P(A ∩ B) = P(A)P(B)
Python Bayes' theorem example: medical testing 
def bayes(p_disease, p_positive_given_disease, p_positive_given_healthy):
    """
    Given a positive test, what's the probability of disease?
    
    p_disease: prior probability of having disease (prevalence)
    p_positive_given_disease: sensitivity (true positive rate)
    p_positive_given_healthy: false positive rate
    """
    p_healthy = 1 - p_disease
    
    # Total probability of positive test
    p_positive = (p_positive_given_disease * p_disease + 
                  p_positive_given_healthy * p_healthy)
    
    # Bayes' theorem
    p_disease_given_positive = (p_positive_given_disease * p_disease) / p_positive
    return p_disease_given_positive

# Example: Rare disease (1% prevalence)
# Test: 99% sensitivity, 5% false positive
result = bayes(0.01, 0.99, 0.05)
print(f"P(disease | positive) = {result:.2%}")  # ~16.7%
# Surprisingly low! The false positives dominate.
