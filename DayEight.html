Section 0

The Goal: Justify Every Step

The test of understanding: Can you explain WHY each step follows from the previous one? If you can't justify a step, you don't fully understand it.

Why this matters

  • Debugging: Finding bugs = finding unjustified assumptions in your mental model
  • Code Review: Good reviewers ask "why does this work?" not just "what does it do?"
  • Learning: Explaining forces deeper understanding than just reading
  • Communication: Others can verify your reasoning only if you make it explicit

The annotation format

For each step, ask: "What rule, definition, or prior result justifies this?"

Common justifications: ‚Ä¢ "by definition of [term]" ‚Ä¢ "by assumption" ‚Ä¢ "by algebra" or "by arithmetic" ‚Ä¢ "by the inductive hypothesis" ‚Ä¢ "by [theorem/lemma name]" ‚Ä¢ "since [prior step]" ‚Ä¢ "this is the contrapositive of..."

Example 4

Algorithm Correctness

Claim: This function returns the maximum element in a non-empty array.

Python
def find_max(arr):
    result = arr[0]
    for i in range(1, len(arr)):
        if arr[i] > result:
            result = arr[i]
    return result
Step
Justification
1.Let invariant I = "result is max of arr[0..i-1]"
Choose invariant that implies correctness
2.Init: Before loop, i=1, result=arr[0]
Program state after line 2
3.I holds: arr[0] is max of arr[0..0] ‚úì
Trivially true: single element is its own max
4.Maintenance: Assume I at start of iteration.

5.If arr[i] > result: result becomes arr[i]
Program logic: the if-branch
6.New result = max(old result, arr[i])
By cases: either update or keep
7.= max(max(arr[0..i-1]), arr[i])
By I.H. from step 4
8.= max(arr[0..i])
Property of max: max(max(A), x) = max(A ‚à™ {x})
9.After i++, I holds for next iteration. ‚úì
Invariant maintained
10.Termination: Loop ends when i = len(arr)
Loop condition: i < len(arr) becomes false
11.By I: result = max(arr[0..len-1]) = max(arr)

12.Return value is correct. ‚àé
