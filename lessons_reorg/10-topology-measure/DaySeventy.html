Section 1

Definition

A Turing Machine (TM) is a finite state machine with an infinite tape. At each step, it can:

  1. Write to the current cell.
  2. Move Left or Right.
  3. Change state.
\delta: Q \times \Gamma \to Q \times \Gamma \times \{L, R\}
Section 2

Church-Turing Thesis

Thesis: Everything that is effectively computable can be computed by a Turing Machine.

There is no computer stronger than a TM (only faster). Your laptop, a supercomputer, and a TM are equivalent in power.

Section 3

Recognizable vs Decidable

A machine can halt (Accept/Reject) or Loop forever.

  • Decidable (Recursive): TM always halts. (Yes or No).
  • Recognizable (Recursively Enumerable): TM halts on Yes, but might loop on No.

Decidability is what we want (algorithms). Recognizability is what we often get.

Section 4

Universal Turing Machine

A TM can take the code of another TM as input and simulate it. This is the Universal Turing Machine.

This is the theoretical basis for General Purpose Computers (Stored-program architecture).

Section 5

The Halting Problem

Theorem: There is no algorithm to determine if a generic TM halts on a given input.

Proof by Diagonalization (sketch):

  • Assume `H` decides halting.
  • Construct `D` which calls `H(D)`. If `H` says `D` halts, `D` loops. If `H` says `D` loops, `D` halts.
  • Contradiction.

This means some problems are strictly Undecidable.

Section 6

Python: TM Simulator

Python Turing Machine 
class TuringMachine:
    def __init__(self, tape, transitions, start_state, accept_state, reject_state):
        self.tape = list(tape) + ['_'] * 100 # Infinite tape extension
        self.head = 0
        self.state = start_state
        self.transitions = transitions
        self.accept_state = accept_state
        self.reject_state = reject_state

    def step(self):
        symbol = self.tape[self.head]
        if (self.state, symbol) in self.transitions:
            new_state, write_sym, move = self.transitions[(self.state, symbol)]
            self.tape[self.head] = write_sym
            self.state = new_state
            self.head += 1 if move == 'R' else -1
            self.head = max(0, self.head) # Tape bounded left
            return True
        return False

    def run(self, max_steps=1000):
        for _ in range(max_steps):
            if self.state == self.accept_state:
                return "ACCEPT"
            if self.state == self.reject_state:
                return "REJECT"
            if not self.step():
                return "STUCK"
        return "LOOP (Timeout)"

# TM to increment binary number (Reverse order for simplicity)
# State q0: Add 1. If 0 -> 1 done. If 1 -> 0 and carry.
transitions = {
    ('q0', '0'): ('q_done', '1', 'R'),
    ('q0', '1'): ('q0', '0', 'R'), # Carry
    ('q0', '_'): ('q_done', '1', 'R'), # Overflow
}

tm = TuringMachine(tape="111", # Binary 7 (reversed: 111)
                   transitions=transitions, 
                   start_state='q0', 
                   accept_state='q_done', 
                   reject_state='q_rej')

print(f"Result: {tm.run()}")
print(f"Tape: {''.join(tm.tape[:5])}") # Should be 0001 (8)
