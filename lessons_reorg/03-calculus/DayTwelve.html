Why Recurrences Matter

The pattern: Recursive code leads to recurrence relations. To find the Big-O of recursive algorithms, you must solve these recurrences.
üå≥ Simple: A recurrence relation is like predicting how many branches a tree will have. If you know how many branches sprout from the trunk, and how many sprout from each branch, you can calculate the total size of the tree without counting every single leaf.

From code to recurrence

PythonMerge sort
def merge_sort(arr):
    if len(arr) <= 1:
        return arr           # O(1) base case
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])   # T(n/2)
    right = merge_sort(arr[mid:])  # T(n/2)
    return merge(left, right)      # O(n)

# Recurrence: T(n) = 2T(n/2) + O(n)

The goal

Convert T(n) = 2T(n/2) + n into T(n) = O(n log n). That's what this day teaches!

Section 1

Recurrence Basics

Anatomy of a recurrence

T(n) = a¬∑T(n/b) + f(n) Where: ‚Ä¢ T(n) = time to solve problem of size n ‚Ä¢ a = number of subproblems ‚Ä¢ n/b = size of each subproblem ‚Ä¢ f(n) = work done outside recursive calls (divide + combine)

Common patterns

Algorithm Recurrence Solution
Binary search T(n) = T(n/2) + O(1) O(log n)
Merge sort T(n) = 2T(n/2) + O(n) O(n log n)
Naive multiply T(n) = 4T(n/2) + O(n) O(n¬≤)
Karatsuba T(n) = 3T(n/2) + O(n) O(n^1.58)
Linear scan T(n) = T(n-1) + O(1) O(n)
Section 2

The Unrolling Method

The technique

Unrolling: Repeatedly substitute the recurrence into itself until you see a pattern, then sum up all the work.

Example: T(n) = 2T(n/2) + n

Unroll step by step: T(n) = 2T(n/2) + n = 2[2T(n/4) + n/2] + n = 4T(n/4) + n + n = 4T(n/4) + 2n = 4[2T(n/8) + n/4] + 2n = 8T(n/8) + n + 2n = 8T(n/8) + 3n Pattern: After k steps: = 2·µè T(n/2·µè) + kn Stop when n/2·µè = 1, so k = log‚ÇÇ n: = n¬∑T(1) + n log n = O(n log n) ‚úì

Example: T(n) = T(n/2) + 1 (Binary search)

T(n) = T(n/2) + 1 = T(n/4) + 1 + 1 = T(n/8) + 1 + 1 + 1 ... = T(n/2·µè) + k When n/2·µè = 1, k = log n: = T(1) + log n = O(log n) ‚úì
Section 3

Guess and Check (Substitution)

The technique

Guess and Check:
1. Guess the answer (based on intuition or unrolling)
2. Prove by induction that your guess works
3. Find the constant that makes it valid

Example: T(n) = 2T(n/2) + n

Guess: T(n) ‚â§ cn log n for some constant c. Prove by induction: Assume T(k) ‚â§ ck log k for all k < n. T(n)=2T(n/2) + n ‚â§ 2 ¬∑ c(n/2)log(n/2) + n [by I.H.]=cn log(n/2) + n=cn log n - cn log 2 + n=cn log n - cn + n=cn log n + n(1 - c) For this ‚â§ cn log n, we need: n(1 - c) ‚â§ 0 c ‚â• 1 So T(n) ‚â§ cn log n for c ‚â• 1. ‚úì Therefore T(n)=O(n log n).
Common mistake: Guessing T(n) = O(n) for T(n) = 2T(n/2) + n doesn't work because the substitution doesn't go through. Always verify with induction!
Section 4

The Master Theorem

The theorem

Master Theorem
T(n) = aT(n/b) + f(n)

Compare f(n) with n^(log_b a):

Case 1: f(n) = O(n^(log_b a - Œµ)) for some Œµ > 0 ‚Üí T(n) = Œò(n^(log_b a)) "Recursion dominates" Case 2: f(n) = Œò(n^(log_b a)) ‚Üí T(n) = Œò(n^(log_b a) ¬∑ log n) "Balanced work" Case 3: f(n) = Œ©(n^(log_b a + Œµ)) for some Œµ > 0 AND a¬∑f(n/b) ‚â§ c¬∑f(n) for c < 1 ‚Üí T(n)=Œò(f(n)) "Non-recursive work dominates"

Step-by-step Master Theorem

Given T(n) = 4T(n/2) + n: 1. Identify: a = 4, b = 2, f(n) = n 2. Compute n^(log_b a) = n^(log_2 4) = n^2 3. Compare f(n) = n vs n^2: n = O(n^(2-Œµ)) for Œµ = 1 ‚úì 4. Case 1 applies: T(n) = Œò(n¬≤)

Common Algorithm Examples

Binary Search: T(n) = T(n/2) + O(1)

a = 1, b = 2, f(n) = 1 n^(log_2 1) = n^0 = 1 f(n) = Œò(1) = Œò(n^0) ‚Üí Case 2 T(n) = Œò(1 ¬∑ log n) = Œò(log n)

Merge Sort: T(n) = 2T(n/2) + O(n)

a = 2, b = 2, f(n) = n n^(log_2 2) = n^1 = n f(n) = Œò(n) ‚Üí Case 2 T(n) = Œò(n log n)

Strassen's Matrix Multiply: T(n) = 7T(n/2) + O(n¬≤)

a = 7, b = 2, f(n) = n¬≤ n^(log_2 7) ‚âà n^2.81 f(n) = n¬≤ = O(n^(2.81-Œµ)) for Œµ ‚âà 0.81 ‚Üí Case 1 T(n) = Œò(n^2.81) ‚âà Œò(n^2.81)

Fibonacci (naive): T(n) = T(n-1) + T(n-2) + O(1)

Master Theorem doesn't apply! The problem size doesn't divide by a constant. Use unrolling or tree method:

T(n) ‚âà 2^n (exponential) because the recursion tree has 2^n nodes.
