Section 0

Why Expected Value Matters

Expected value = average performance. For randomized algorithms, we analyze expected running time. For systems, we measure expected latency.

Where you'll use this

  • QuickSort: Expected O(n log n) with random pivot
  • Hash tables: Expected O(1) lookup
  • Load balancing: Expected load per server
  • Machine learning: Expected loss minimization
  • Monte Carlo: Estimate complex quantities
Section 1

Expected Value Definition

Formula

Expected Value of X
E[X] = Σₓ x · P(X = x)

The expected value is the weighted average of all possible values, weighted by their probabilities.

Example: Fair die

X = value of fair die roll E[X] = 1·(1/6) + 2·(1/6) + 3·(1/6) + 4·(1/6) + 5·(1/6) + 6·(1/6) = (1 + 2 + 3 + 4 + 5 + 6) / 6 = 21/6 = 3.5 The expected value need not be a possible outcome!

Example: Unfair coin

Coin: Heads = 1 (prob 0.7), Tails = 0 (prob 0.3) E[X] = 1·(0.7) + 0·(0.3) = 0.7 More generally, for any 0/1 random variable: E[X] = P(X = 1)
Section 2

Linearity of Expectation

The magic property

Linearity of Expectation
E[X + Y] = E[X] + E[Y] — ALWAYS!
Key insight: This works even when X and Y are NOT independent! This is what makes linearity so powerful.

Extended form

For any random variables X₁, X₂, ..., Xₙ and constants a, b: E[aX + b] = a·E[X] + b E[X₁ + X₂ + ... + Xₙ] = E[X₁] + E[X₂] + ... + E[Xₙ]

Example: Sum of 10 dice

Roll 10 dice. What's E[total]? Without linearity: compute probability of every sum from 10 to 60! With linearity: E[D₁ + D₂ + ... + D₁₀] = E[D₁] + E[D₂] + ... + E[D₁₀] = 3.5 + 3.5 + ... + 3.5 = 10 × 3.5 = 35 So easy! Dice aren't independent? Doesn't matter!
Section 3

Indicator Random Variables

Definition

Indicator Random Variable
Iₐ = 1 if event A occurs, 0 otherwise
E[Iₐ] = 1·P(A) + 0·P(not A) = P(A) This lets us count using expectations!

Counting technique

The trick: To count something, express it as a sum of indicators, then use linearity.
Number of heads in n flips: X = I₁ + I₂ + ... + Iₙ (where Iᵢ = 1 if flip i is heads) E[X] = E[I₁] + E[I₂] + ... + E[Iₙ] = P(heads) + P(heads) + ... + P(heads) = n × 0.5 = n/2
Section 4

Classic Examples

Hat check problem

n people randomly pick hats. Expected # who get their own hat? Let Xᵢ = 1 if person i gets their hat. E[Xᵢ] = P(person i gets their hat) = 1/n X = X₁ + X₂ + ... + Xₙ (total correct) E[X] = E[X₁] + ... + E[Xₙ] = n × (1/n) = 1 On average, exactly 1 person gets their hat!

Coupon collector

n types of coupons, each equally likely. Expected draws to collect all? Phase i: already have i-1 types, waiting for a new one. P(new type) = (n-i+1)/n Expected draws in phase i = n/(n-i+1) Total = Σᵢ₌₁ⁿ n/(n-i+1) = n × (1/n + 1/(n-1) + ... + 1/1) = n × Hₙ ≈ n ln n To collect n coupons: ~n ln n draws on average.

Birthday problem (expected)

n people. Expected number of shared birthday pairs? C(n,2) = n(n-1)/2 pairs. Each pair shares birthday with prob 1/365. E[# shared pairs] = n(n-1)/2 × (1/365) For n = 23: 23×22/2 × (1/365) ≈ 0.69 pairs expected.

Algorithm Analysis

QuickSort expected comparisons

Let Xᵢⱼ = 1 if elements i and j are compared. E[comparisons] = Σᵢ<ⱼ E[Xᵢⱼ]=Σᵢ<ⱼ P(i,j compared) Two elements are compared only if one is chosen as pivot before any element between them. P(i,j compared)=2/(j-i+1) Summing: E[comparisons]=O(n log n)

Randomized algorithms

For randomized algorithms: • Expected running time = E[T(n)] — average over random choices • Worst-case expectation: max over inputs of E[T(n, input)] QuickSort: E[T(n)] = O(n log n) for any input (randomness is in the algorithm, not the input)
