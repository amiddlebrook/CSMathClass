Why Counting Matters

Counting = Understanding the search space. How many possible passwords? How many ways to arrange data? How many states in your algorithm?

Where you'll use this

  • Security: Password strength = log₂(number of possibilities)
  • Algorithms: Brute force time = number of configurations to check
  • Probability: P(event) = favorable outcomes / total outcomes
  • Data Structures: Hash functions, bloom filters, random sampling
  • Testing: How many test cases cover all combinations?
Section 1

The Basic Counting Rules

Rule of Sum (OR)

Rule of Sum
If A has m outcomes and B has n outcomes (disjoint), then A OR B has m + n outcomes
Example: A restaurant offers 5 appetizers OR 8 entrees. If you can only pick one dish: 5 + 8 = 13 choices. Key: Use when choices are mutually exclusive (can't pick both).

Rule of Product (AND)

Rule of Product
If A has m outcomes and B has n outcomes (independent), then A AND B has m × n outcomes
Example: A restaurant offers 5 appetizers AND 8 entrees. If you pick one of each: 5 × 8 = 40 meal combinations. Key: Use when making a sequence of independent choices.

Extended product rule

For a sequence of k independent choices with n₁, n₂, ..., nₖ options: Total = n₁ × n₂ × ... × nₖ Example: License plate with 3 letters + 4 digits = 26 × 26 × 26 × 10 × 10 × 10 × 10 = 26³ × 10⁴ = 175,760,000
Section 2

Factorials

Definition

Factorial
n! = n × (n-1) × (n-2) × ... × 2 × 1
Examples: 0! = 1 (by definition) 1! = 1 2! = 2 3! = 6 4! = 24 5! = 120 10! = 3,628,800 20! ≈ 2.4 × 10¹⁸

Why factorial?

n! counts the number of ways to arrange n distinct objects. Arranging 3 books on a shelf: Position 1: 3 choices Position 2: 2 choices (one book used) Position 3: 1 choice Total: 3 × 2 × 1 = 3! = 6

In code

Python
import math

math.factorial(5)   # 120
math.factorial(10)  # 3628800

# Recursive definition
def factorial(n):
    if n <= 1:
        return 1
    return n * factorial(n - 1)
Section 3

Permutations P(n,k)

Definition

Permutations: Order Matters
P(n,k) = n! / (n-k)! = n × (n-1) × ... × (n-k+1)

P(n,k) = number of ways to arrange k items from n distinct items, where order matters.

Examples

How many 3-letter "words" from {A,B,C,D,E}? (no repeats) P(5,3) = 5 × 4 × 3 = 60 Gold/Silver/Bronze medals for 8 athletes? P(8,3) = 8 × 7 × 6 = 336 Arrange all 4 books? P(4,4) = 4! = 24

With repetition

If repetition is allowed: n choices for each of k positions = nᵏ PIN code (4 digits, can repeat): 10⁴ = 10,000 Password (8 lowercase letters): 26⁸ ≈ 209 billion

In code

Python
import math
from itertools import permutations

# P(n,k) using math
def P(n, k):
    return math.factorial(n) // math.factorial(n - k)

P(5, 3)  # 60

# Or use math.perm (Python 3.8+)
math.perm(5, 3)  # 60

# Generate all permutations
list(permutations('ABC', 2))
# [('A','B'), ('A','C'), ('B','A'), ('B','C'), ('C','A'), ('C','B')]
Section 4

Combinations C(n,k)

Definition

Combinations: Order Doesn't Matter
C(n,k) = n! / (k! × (n-k)!) = P(n,k) / k!

C(n,k), also written (n choose k) or ⁿCₖ, counts ways to choose k items from n where order doesn't matter.

Why divide by k!?

Permutations count different orderings as different. Combinations don't care about order. {A,B,C} has 3! = 6 orderings: ABC, ACB, BAC, BCA, CAB, CBA But as a set, they're all the same selection. So: C(n,k) = P(n,k) / k!

Examples

Choose 3 students from 10 for a committee? C(10,3) = 10!/(3! × 7!) = (10×9×8)/(3×2×1) = 120 5-card hand from 52-card deck? C(52,5) = 2,598,960 Choose 0 items from n? C(n,0) = 1 (the empty set)

Key properties

C(n,k) = C(n, n-k) (symmetry) C(n,0) = C(n,n) = 1 C(n,1) = n Σₖ C(n,k) = 2ⁿ (all subsets)

In code

Python
import math
from itertools import combinations

# C(n,k) using math
def C(n, k):
    return math.factorial(n) // (math.factorial(k) * math.factorial(n-k))

C(10, 3)  # 120

# Or use math.comb (Python 3.8+)
math.comb(10, 3)  # 120

# Generate all combinations
list(combinations('ABCD', 2))
# [('A','B'), ('A','C'), ('A','D'), ('B','C'), ('B','D'), ('C','D')]
Section 5

CS Applications

Password strength

8-char password (lowercase + uppercase + digits): 62 choices per position Total: 62⁸ ≈ 2.18 × 10¹⁴ Bits of entropy: log₂(62⁸) ≈ 48 bits To crack at 10 billion/sec: ~6 hours

Subsets and power sets

Python
# Number of subsets of a set of size n = 2ⁿ
# Each element: include or exclude (2 choices)

n = 20
subsets = 2 ** n  # 1,048,576

# Generate all subsets using bit manipulation
def all_subsets(items):
    n = len(items)
    for mask in range(2**n):
        yield [items[i] for i in range(n) if mask & (1 << i)]

Binomial coefficients in algorithms

Pascal's Triangle: C(n,k) = C(n-1,k-1) + C(n-1,k) Used in: • Dynamic programming • Polynomial expansion: (a+b)ⁿ • Probability distributions • Paths in a grid: C(m+n, m) paths from (0,0) to (m,n)
