Why Probability Matters

CS is full of uncertainty. Networks fail, users behave randomly, algorithms use randomness. Probability lets you reason about uncertain systems.

Where you'll use this

  • Randomized Algorithms: QuickSort's expected O(n log n)
  • Machine Learning: Every model is probabilistic
  • Systems: Failure probability, load balancing
  • Cryptography: Security margins
  • A/B Testing: Statistical significance
  • Bloom Filters: False positive probability
Section 1

Probability Basics

Definition

Probability of Event A
P(A) = |favorable outcomes| / |total outcomes|
Requirements: 0 ≤ P(A) ≤ 1 for all events A P(Ω) = 1 where Ω is the sample space (all outcomes) P(∅) = 0 where ∅ is the empty event

Complement

P(not A) = P(Ā) = 1 - P(A) Example: P(at least one head in 3 flips) = 1 - P(all tails) = 1 - (1/2)³ = 1 - 1/8 = 7/8

Examples

Fair die: P(roll 6) = 1/6 Fair coin: P(heads) = 1/2 Deck of cards: P(ace) = 4/52 = 1/13 Random bit: P(1) = 1/2
Section 2

Addition and Multiplication Rules

Addition rule (OR)

General Addition Rule
P(A or B) = P(A) + P(B) - P(A and B)
If A and B are mutually exclusive (can't both happen): P(A or B) = P(A) + P(B) Example: P(roll 1 or 6) = 1/6 + 1/6 = 2/6 = 1/3

Multiplication rule (AND)

General Multiplication Rule
P(A and B) = P(A) × P(B|A)
If A and B are independent: P(A and B) = P(A) × P(B) Example: P(two heads) = 1/2 × 1/2 = 1/4
Section 3

Conditional Probability

Definition

Conditional Probability
P(A|B) = P(A and B) / P(B)

Read as: "probability of A given B" — the probability of A occurring, knowing that B has occurred.

Example: Drawing cards

Draw 2 cards without replacement. P(second is ace | first was ace)? After removing one ace: 3 aces remain in 51 cards. P(second ace | first ace) = 3/51 = 1/17 Compare to with replacement: P(second ace | first ace) = 4/52 = 1/13

Chain rule

P(A and B and C) = P(A) × P(B|A) × P(C|A and B) Example: Draw 3 aces in a row without replacement = (4/52) × (3/51) × (2/50) = 24 / 132,600 ≈ 0.00018
Section 4

Independence

Definition

Independent Events
P(A and B) = P(A) × P(B)

Equivalently: P(A|B) = P(A) — knowing B gives no information about A.

Independent vs mutually exclusive

Don't confuse these!

Independent: P(A and B) = P(A) × P(B) — occurrence of one doesn't affect the other
Mutually exclusive: P(A and B) = 0 — can't both happen

If P(A) > 0 and P(B) > 0, mutually exclusive events are NOT independent!

CS example: System reliability

Two independent servers, each fails with probability 0.1. P(both fail) = 0.1 × 0.1 = 0.01 P(at least one works) = 1 - 0.01 = 0.99 With 3 servers: P(all fail) = 0.1³ = 0.001 P(at least one works) = 0.999

Birthday paradox intuition

How many people until P(shared birthday) > 50%? Day 12 · Calculus Sequencent) = (365/365) × (364/365) × ... × ((365-n+1)/365) With 23 people: P(all different) ≈ 0.49 So P(at least one shared) ≈ 0.51 Surprisingly small! Independence is tricky.
Section 5

Bayes' Theorem

The formula

Bayes' Theorem
P(A|B) = P(B|A) × P(A) / P(B)
Also written as: P(hypothesis|evidence) = P(evidence|hypothesis) × P(hypothesis) / P(evidence) • P(A) = prior (belief before evidence) • P(A|B) = posterior (belief after evidence) • P(B|A) = likelihood (how likely is evidence if A is true) • P(B) = marginal (total probability of evidence)

Classic example: Disease testing

A disease affects 1% of population. Test is 99% accurate (for both positive and negative). You test positive. What's P(you have disease)? P(disease) = 0.01 P(positive | disease) = 0.99 P(positive | no disease) = 0.01 P(positive) = P(pos|D)×P(D) + P(pos|¬D)×P(¬D) = 0.99×0.01 + 0.01×0.99 = 0.0099 + 0.0099 = 0.0198 P(disease | positive) = (0.99 × 0.01) / 0.0198 = 0.0099 / 0.0198 = 0.5 Only 50%! Base rate matters enormously.

CS applications

  • Spam filtering: P(spam | contains "FREE")
  • Classification: Naive Bayes classifier
  • A/B testing: Bayesian inference
  • Debugging: P(bug in module A | test failed)
