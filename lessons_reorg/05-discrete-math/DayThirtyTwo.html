Section 0

Why Data Structures Matter

A data structure is a way of organizing data so that it can be used efficiently. The choice of data structure often determines the efficiency of an algorithm.

Key Insight: The right data structure can transform an O(n²) algorithm into O(n log n) or even O(n). Always choose based on your access patterns.

Common Operations to Consider

  • Access: How fast can we get element i?
  • Search: How fast can we find element x?
  • Insert: How fast can we add a new element?
  • Delete: How fast can we remove an element?
Python Choosing the right structure matters 
# Bad: O(n) lookup in list
def find_in_list(items, target):
    for item in items:  # O(n)
        if item == target:
            return True
    return False

# Good: O(1) lookup in set
def find_in_set(items_set, target):
    return target in items_set  # O(1) average!
Section 1

Arrays

Fixed-size Arrays

Contiguous memory block. O(1) access by index, but O(n) insert/delete.

Array[i] address = base + (i × element_size)

Dynamic Arrays (Python list, Java ArrayList)

Automatically resize when full. Amortized O(1) append, O(n) insert at arbitrary position.

Gotcha: Inserting at the front of a dynamic array is O(n) — every element must shift. Use a deque if you need fast front insertion.
Python Array operations and their costs 
arr = [1, 2, 3, 4, 5]

# O(1) - Direct access
x = arr[2]        # 3

# O(1) amortized - Append
arr.append(6)    # [1, 2, 3, 4, 5, 6]

# O(n) - Insert at position
arr.insert(0, 0) # [0, 1, 2, 3, 4, 5, 6] - shifts all!

# O(n) - Search
idx = arr.index(3) # Linear scan to find
Section 2

Singly Linked List

Node: Contains data and a pointer to the next node.
HEAD → [data|next] → [data|next] → [data|next] → NULL
Python Singly linked list node 
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

# Create: 1 → 2 → 3
head = ListNode(1)
head.next = ListNode(2)
head.next.next = ListNode(3)
Section 3

Doubly Linked List

NULL ← [prev|data|next] ↔ [prev|data|next] ↔ [prev|data|next] → NULL

Each node has both prev and next pointers. Enables O(1) delete if you have the node.

Advantages

  • O(1) delete with node reference
  • Traverse both directions
  • Easy insertBefore

Disadvantages

  • Extra memory per node
  • More complex code
  • Cache less friendly

Operations

OPERATION | SINGLY | DOUBLY ------------------|---------|-------- Access by index | O(n) | O(n) Search | O(n) | O(n) Insert at head | O(1) | O(1) Insert at tail* | O(n)/O(1)| O(1) Delete head | O(1) | O(1) Delete with node | O(n) | O(1) * O(1) if you maintain a tail pointer
Python Insert at head 
def insert_at_head(head, val):
    new_node = ListNode(val)
    new_node.next = head  # Point to old head
    return new_node       # Return new head
Section 5

Common Patterns

Two-pointer (fast/slow)

Detect cycles, find middle, find nth from end.

Python Find middle with slow/fast pointers 
def find_middle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow  # slow is at middle
Dummy head trick: Use a dummy node before head to simplify edge cases when head might change.
Section 6

Stacks & Queues

Stack (LIFO)

push(x): Add to top pop(): Remove from top peek(): View top All O(1)

Use cases: function calls, undo, parsing parentheses

Queue (FIFO)

enqueue(x): Add to back dequeue(): Remove from front peek(): View front All O(1)*

Use cases: BFS, task scheduling, buffers

*Note: Queue with array needs circular buffer for O(1) dequeue, or use linked list. Python's collections.deque handles this.
Python Stack and Queue implementations 
from collections import deque

# Stack (use list)
stack = []
stack.append(1)      # push
stack.append(2)
top = stack.pop()    # 2

# Queue (use deque)
queue = deque()
queue.append(1)      # enqueue
queue.append(2)
front = queue.popleft()  # 1 - O(1)!
Section 7

Hash Tables

Hash Table: Maps keys to values using a hash function. Average O(1) for insert, delete, and lookup!

How it works

1. hash(key) → integer 2. index = hash(key) % table_size 3. Store (key, value) at that index 4. Handle collisions (chaining or probing)
Worst case is O(n) if many collisions. Good hash functions and proper load factor (typically < 0.75) keep performance at O(1).
Python Python dict is a hash table 
# Python dict = hash table
counts = {}
counts["apple"] = 3    # O(1) insert
counts["banana"] = 5

# O(1) lookup
print(counts["apple"])  # 3

# O(1) membership test
if "apple" in counts:
    print("Found!")

# Set = hash table with only keys
seen = {1, 2, 3}
print(2 in seen)  # True, O(1)
