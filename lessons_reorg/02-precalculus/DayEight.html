Section 1

Translations (Shifting)

Shifting a graph represents a change in the "starting point" (input) or the "baseline" (output).

Vertical Shift: $f(x) + k$

Adding $k$ outside the function moves the entire graph up (if $k > 0$) or down (if $k < 0$).

Example: $f(x) = x^2 + 5$ is a parabola shifted 5 units up.

Horizontal Shift: $f(x - h)$

Changing the input inside the parentheses shifts the graph left or right. This is often counter-intuitive:

  • $f(x - 3)$ shifts the graph Right by 3 (delaying the input).
  • $f(x + 3)$ shifts the graph Left by 3 (advancing the input).
PythonGUI Analogy
# Moving a button in a GUI frame
def draw_button(x, y):
    # Logic to draw at (x, y)
    pass

# Shifting the baseline (Vertical Translation)
y_offset = 50
draw_button(x, y + y_offset)

# Shifting the input (Horizontal Translation)
x_offset = 100
draw_button(x - x_offset, y)
Section 2

Reflections & Scaling

Reflections

  • $y = -f(x)$: Reflects across the $x$-axis (flips upside down).
  • $y = f(-x)$: Reflects across the $y$-axis (flips left-to-right).

Vertical Scaling: $a \cdot f(x)$

Multiplying the output by $a$ stretches or compresses the graph vertically.

  • If $|a| > 1$: Vertical Stretch (graph gets taller/steeper).
  • If $0 < |a| < 1$: Vertical Compression (graph gets shorter/flatter).

Horizontal Scaling: $f(c \cdot x)$

Multiplying the input by $c$ compresses or stretches the graph horizontally.

  • If $|c| > 1$: Horizontal Compression (graph gets narrower).
  • If $0 < |c| < 1$: Horizontal Stretch (graph gets wider).
Section 3

Asymptotes (Invisible Fences)

Asymptotes represent the values a function approaches but can never (usually) touch. In CS, these represent the constraints or limits of a system.

Vertical Asymptotes (VA)

Occur where the denominator of a rational function is zero (and doesn't cancel out). The function "explodes" to $\infty$ or $-\infty$.

f(x) = \frac{1}{x - 2} \implies \text{VA at } x = 2

Horizontal Asymptotes (HA)

Describe the "end behavior" of a function as $x \to \infty$.

The "Degree Duel": Compare high powers of $D(x)$ and $N(x)$:
  • Bottom Heavy: $y = 0$ (e.g., $1/x$).
  • Balanced: $y = \text{Ratio of coefficients}$ (e.g., $3x/x \to y = 3$).
  • Top Heavy: No HA (goes to infinity).
Section 4

Even & Odd Symmetry

Symmetry allows us to avoid redundant work. If we know half the graph, we know the rest.

Even Functions

f(-x) = f(x)

Symmetric across the y-axis. Think of $x^2$.

Odd Functions

f(-x) = -f(x)

Symmetric about the origin (180Â° rotation). Think of $x^3$.

Section 5

Growth Rates (The CS Link)

End behavior is the foundation of Big-O Notation. We care about which part of a function "wins" as $x$ gets huge.

Hierarchy of Growth:

$\text{Constant} < \log(n) < \text{Linear} < n \log n < \text{Polynomial} < \text{Exponential} < \text{Factorial}$

In a rational function like $f(x) = \frac{x^3 + 10x^2}{x^2}$, for very large $x$, the $10x^2$ is irrelevant. The $x^3$ dominates the top, and the $x^2$ dominates the bottom. Thus, the function behaves like $x^3/x^2 = x$.
