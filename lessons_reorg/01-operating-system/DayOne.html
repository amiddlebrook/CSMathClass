Why Arithmetic Fluency Matters in CS

Every algorithm you write manipulates numbers. Graphics shaders blend colors using fractions. Cryptographic systems rely on modular exponentiation. Machine learning is just matrix multiplication at scale. If you hesitate on basic operations, you'll struggle with the real stuff.

The goal: These operations should be reflexive. When you see 3/4 + 5/6, your brain should pattern-match to "common denominator" without conscious thought.

Where you'll use this

  • Graphics/Games: Color blending, interpolation, physics simulation
  • Cryptography: RSA uses exponents mod n, key sizes are powers of 2
  • ML/Data Science: Normalization, gradients, floating-point precision
  • Systems: Memory alignment, bit manipulation, overflow detection
Python Color blending uses fractions constantly 
# Blend two colors: result = color1 * (1-t) + color2 * t
def lerp_color(c1, c2, t):
    # t=0 ‚Üí c1, t=1 ‚Üí c2, t=0.5 ‚Üí halfway
    return tuple(
        int(c1[i] * (1 - t) + c2[i] * t)
        for i in range(3)
    )

# What's 3/4 of the way from red to blue?
red = (255, 0, 0)
blue = (0, 0, 255)
result = lerp_color(red, blue, 0.75)  # (63, 0, 191)

Fractions

Core definition

a/b means a √∑ b   (where b ‚â† 0)

A fraction is just delayed division. This matters in programming because integer division truncates: 7 // 2 = 3 in Python, not 3.5.

üçï Simple: Think of a fraction as pizza slices. If you have 3/4, you have 3 slices out of a pizza cut into 4 pieces. The bottom number (denominator) is how many slices the whole pizza was cut into. The top number (numerator) is how many slices you have.

Equivalent fractions

a/b = (a¬∑k) / (b¬∑k)   for any k ‚â† 0

Multiplying top and bottom by the same number doesn't change the value. This is how we find common denominators.

Simplifying (reducing)

Divide numerator and denominator by their GCD (greatest common divisor).
24/36 = (24 √∑ 12) / (36 √∑ 12) = 2/3
Python GCD is the foundation of fraction simplification 
import math

def simplify(num, den):
    """Reduce a fraction to lowest terms."""
    g = math.gcd(num, den)
    return num // g, den // g

simplify(24, 36)  # ‚Üí (2, 3)
simplify(100, 75) # ‚Üí (4, 3)

Adding & subtracting fractions

You must have the same denominator. Find the LCD (least common denominator), convert both fractions, then add/subtract the numerators.
a/b + c/d = (a¬∑d + c¬∑b) / (b¬∑d)

Example: 2/3 + 1/6 = 4/6 + 1/6 = 5/6

Python Fraction addition in code 
from fractions import Fraction

# Python's Fraction class handles this automatically
a = Fraction(2, 3)
b = Fraction(1, 6)
print(a + b)  # ‚Üí 5/6 (auto-simplified)

# Manual implementation
def add_fractions(n1, d1, n2, d2):
    num = n1 * d2 + n2 * d1
    den = d1 * d2
    return simplify(num, den)

add_fractions(2, 3, 1, 6)  # ‚Üí (5, 6)

Multiplying fractions

(a/b) ¬∑ (c/d) = (a¬∑c) / (b¬∑d)

Just multiply straight across. Pro tip: cancel common factors before multiplying to keep numbers small.

Dividing fractions

Dividing by a fraction = multiplying by its reciprocal (flip it).
(a/b) √∑ (c/d) = (a/b) ¬∑ (d/c) = (a¬∑d) / (b¬∑c)
üçï Simple Mode: "How many 1/4 pizzas fit in 2 pizzas?" You're asking 2 √∑ (1/4). Flip and multiply: 2 √ó 4 = 8. Eight quarter-pizzas fit in 2 whole pizzas!

Mixed numbers

m p/q = (m¬∑q + p) / q

Example: 2 3/5 = (2¬∑5 + 3) / 5 = 13/5

Convert mixed numbers to improper fractions before doing any operations. Convert back at the end if needed.

Negatives (Signed Numbers)

Subtraction is addition of the opposite

a ‚àí b = a + (‚àíb)

This reframing eliminates a whole class of errors. There's no "subtraction rule" anymore‚Äîjust addition.

üí∞ Simple: Think of negative numbers as debt. If you have $5 and owe $3, you have 5 + (-3) = $2. "Subtracting a negative" is like removing a debt‚Äîif someone cancels your $3 debt, you're $3 richer! 5 - (-3) = 5 + 3 = 8.

Adding signed numbers

  • Same signs: Add magnitudes, keep the sign ‚Üí (‚àí7) + (‚àí5) = ‚àí12
  • Different signs: Subtract magnitudes, keep sign of larger magnitude ‚Üí (‚àí9) + 4 = ‚àí5

Subtracting a negative

Subtracting a negative flips to addition: a ‚àí (‚àíb) = a + b

Example: 5 ‚àí (‚àí3) = 5 + 3 = 8

Multiplying & dividing signs

(+) √ó (+) = (+)      (‚àí) √ó (‚àí) = (+) (+) √ó (‚àí) = (‚àí)      (‚àí) √ó (+) = (‚àí)
Same signs ‚Üí positive. Different signs ‚Üí negative.

This applies to division too.

JavaScript Sign handling in coordinate systems 
// In screen coordinates, Y increases downward
// In math coordinates, Y increases upward
// Conversion requires sign flipping

  // Flip Y axis: subtract from height

// Velocity with direction

The classic trap: negatives and exponents

Watch out: ‚àí3¬≤ = ‚àí(3¬≤) = ‚àí9 but (‚àí3)¬≤ = 9

Without parentheses, the exponent binds tighter than the negative sign.
Python Python follows standard math precedence

Exponents & Roots

Core definition

a‚Åø = a √ó a √ó ... √ó a   (n times)
üìö Simple: Exponents are just a shorthand for repeated multiplication. Instead of writing 2√ó2√ó2√ó2√ó2, we write 2‚Åµ. Think of it like stacking multiplications. Each exponent "layer" multiplies by that base one more time.

The seven exponent laws (memorize these)

a·µê ¬∑ a‚Åø = a·µê‚Å∫‚Åø // same base ‚Üí add exponents
a·µê / a‚Åø = a·µê‚Åª‚Åø // same base ‚Üí subtract exponents
(a·µê)‚Åø = a·µê‚Åø // power of power ‚Üí multiply
(ab)‚Åø = a‚Åøb‚Åø // distribute over multiplication
(a/b)‚Åø = a‚Åø / b‚Åø // distribute over division
a‚Å∞ = 1   (a ‚â† 0) // any nonzero number to 0 is 1
a‚Åª‚Åø = 1 / a‚Åø // negative exponent ‚Üí reciprocal
Python Exponent laws in action: big-O simplification 
# When analyzing algorithms, we use these constantly

# O(n¬≤) ¬∑ O(n) = O(n¬≥)  ‚Üí exponents add
# O(n¬≥) / O(n) = O(n¬≤)  ‚Üí exponents subtract

# Example: nested loops
def example(n):
    count = 0
    for i in range(n):       # O(n)
        for j in range(n):   # √ó O(n)
            for k in range(n): # √ó O(n)
                count += 1
    return count  # Total: O(n¬≥) = O(n¬π‚Å∫¬π‚Å∫¬π)

Fractional exponents (roots)

a^(1/n) = ‚Åø‚àöa      (nth root of a) a^(m/n) = ‚Åø‚àö(a·µê) = (‚Åø‚àöa)·µê

A fractional exponent combines a power and a root. The denominator is the root, the numerator is the power. Order doesn't matter mathematically, but computationally take the root first to keep numbers smaller.

Python Roots as fractional exponents 
# Square root = power of 1/2
print(16 ** 0.5)    # ‚Üí 4.0
print(16 ** (1/2))  # ‚Üí 4.0

# Cube root = power of 1/3
print(27 ** (1/3))  # ‚Üí 3.0

# 16^(3/4) = fourth root of 16, then cubed
print(16 ** (3/4))  # ‚Üí 8.0 (because ‚Å¥‚àö16 = 2, and 2¬≥ = 8)

# Useful in graphics: gamma correction
def gamma_correct(color, gamma=2.2):
    """Convert linear color to sRGB (gamma = 1/2.2 ‚âà 0.45)"""
    return color ** (1 / gamma)

Even vs odd roots of negatives

  • Even roots of negatives are not real: ‚àö(‚àí9) is imaginary
  • Odd roots of negatives are real: ‚àõ(‚àí8) = ‚àí2
Python Handling negative roots

# Even root of negative ‚Üí error (or nan)
# math.sqrt(-9)  # ValueError!

# Use complex math for imaginary results

# Cube root of negative is real, but Python's ** is weird:

# For real cube roots of negatives, preserve the sign:
def cbrt(x):
    """Real cube root that handles negatives correctly."""
    if x >= 0:
        return x ** (1/3)
    else:
        return -(abs(x) ** (1/3))

print(cbrt(-8))  # ‚Üí -2.0 ‚úì

Scientific Notation

Standard form

a √ó 10‚Åø   where 1 ‚â§ |a| < 10 and n is an integer

Scientific notation is how computers represent floating-point numbers internally. Understanding it helps you reason about precision and overflow.

üî≠ Simple: Scientific notation is just counting zeros. 5,000,000 has 6 zeros after the 5, so it's 5 √ó 10‚Å∂. For tiny numbers like 0.00003, you move the decimal 5 places to get 3, so it's 3 √ó 10‚Åª‚Åµ. The exponent = how many places you moved the decimal.

Converting to/from

  • Move decimal left ‚Üí positive exponent: 12,300 = 1.23 √ó 10‚Å¥
  • Move decimal right ‚Üí negative exponent: 0.0051 = 5.1 √ó 10‚Åª¬≥

Count the places you move. The exponent tells you "how many times to multiply/divide by 10".

Python Scientific notation in code

# Formatting numbers in scientific notation

print(f"{big:.2e}")  # ‚Üí 1.23e+08

small = 0.00000456

# Understanding floating point limits
import sys

print(sys.float_info.min)  # ~2.2e-308 (smallest positive)

Multiplying & dividing

(a √ó 10·µê) √ó (b √ó 10‚Åø) = (a √ó b) √ó 10·µê‚Å∫‚Åø (a √ó 10·µê) √∑ (b √ó 10‚Åø) = (a √∑ b) √ó 10·µê‚Åª‚Åø

Multiply/divide the coefficients, add/subtract the exponents. This is exactly how floating-point multiplication works in hardware.

Adding & subtracting

You must align the exponents first. Convert to the same power of 10, then add/subtract the coefficients.
3.2 √ó 10‚Åµ + 4.5 √ó 10‚Å¥ = 3.2 √ó 10‚Åµ + 0.45 √ó 10‚Åµ = 3.65 √ó 10‚Åµ
Python Why floating-point addition can lose precision 
# Adding numbers with very different magnitudes
big = 1e16
small = 1.0

print(big + small == big)  # ‚Üí True! The 1.0 was lost

# Why? In binary scientific notation, 1e16 looks like:
# 1.0 √ó 2^53 (roughly)
# Adding 1.0 means aligning to same exponent:
# 1.0 √ó 2^53 + 0.0000000000000001 √ó 2^53
# The small coefficient rounds to 0!

# This is called "catastrophic cancellation"
# Solution: use libraries like Decimal for precision
from decimal import Decimal
big_d = Decimal('1e16')
small_d = Decimal('1')
print(big_d + small_d)  # ‚Üí 10000000000000001 ‚úì

Ratios

What is a ratio?

A ratio compares two quantities. Written as a : b or a/b.

ratio of a to b = a : b = a/b

Simplifying ratios

Divide both parts by their GCD, just like fractions.
12 : 18 = (12√∑6) : (18√∑6) = 2 : 3
PythonAspect ratio calculation
import math

def aspect_ratio(width, height):
    g = math.gcd(width, height)
    return (width // g, height // g)

print(aspect_ratio(1920, 1080))  # (16, 9)
print(aspect_ratio(2560, 1440))  # (16, 9)
print(aspect_ratio(1024, 768))   # (4, 3)

Part-to-part vs part-to-whole

  • Part-to-part: wins:losses = 3:2
  • Part-to-whole: wins:total = 3:5 (this is win rate!)
PythonCache hit rate
cache_hits = 850
cache_misses = 150
total = cache_hits + cache_misses  # 1000

# Part-to-part: hit:miss ratio
hit_miss = cache_hits / cache_misses  # 5.67

# Part-to-whole: hit rate (what you usually want)
hit_rate = cache_hits / total  # 0.85 = 85%

Rates

What is a rate?

A rate is a ratio with different units. It describes how one quantity changes with respect to another.

rate = quantity / unit Examples: 60 miles/hour (speed) 1000 requests/second (throughput) 8 bits/byte (conversion factor)
üö® Simple: A rate is "how much per something." Think of a water faucet: if it fills 2 gallons per minute, that's the rate. To find how much water in 5 minutes, multiply: 2 √ó 5 = 10 gallons. Rates let you convert between "how fast" and "how much."

The fundamental formula

amount = rate √ó time rate = amount / time time = amount / rate
PythonDownload time estimation
def download_time(file_mb, speed_mbps):
    """
    Note: ISPs use Mbps (megabits), files use MB (megabytes)
    1 byte = 8 bits
    """
    speed_mb_per_sec = speed_mbps / 8  # Mbps ‚Üí MB/s
    return file_mb / speed_mb_per_sec

# 4GB file on 100 Mbps connection
time_sec = download_time(4000, 100)  # 320 seconds

Combining rates

PythonParallel vs sequential
# Parallel workers: rates ADD
workers = [100, 150, 120]  # ops/sec each

# Pipeline: use SLOWEST rate (bottleneck)
stages = [200, 80, 150]  # ops/sec each stage
pipeline_rate = min(stages)  # 80 ops/sec

Percentages

What is a percentage?

p% = p/100 = p per 100 25% = 25/100 = 0.25 = 1/4

The three percentage problems

1. Find the part: What is 20% of 150? ‚Üí 0.20 √ó 150 = 30
2. Find the percent: 30 is what % of 150? ‚Üí 30/150 = 20%
3. Find the whole: 30 is 20% of what? ‚Üí 30/0.20 = 150

Percent change

percent change = (new - old) / old √ó 100%
Percentage points vs percent:
If conversion goes from 2% to 3%:
‚Ä¢ 1 percentage point increase (3% - 2%)
‚Ä¢ 50 percent increase ((3-2)/2 √ó 100)
These are very different!

Unit Conversions

The core method

Multiply by a conversion factor that equals 1. The old unit cancels.
5 km √ó (1000 m / 1 km) = 5000 m

CS-essential conversions

Category Conversion Factor
Data bytes ‚Üî bits 1 byte = 8 bits
Data KB ‚Üî bytes 1 KB = 1024 bytes (binary)
Data MB ‚Üî KB 1 MB = 1024 KB
Time ms ‚Üî s 1 s = 1000 ms
Time Œºs ‚Üî ms 1 ms = 1000 Œºs
Network Mbps ‚Üî MB/s Mbps √∑ 8 = MB/s
Binary vs SI: A "500 GB" drive has ~465 GiB usable.
SI (1000): kB, MB, GB ¬∑ Binary (1024): KiB, MiB, GiB
PythonByte formatting
def bytes_to_human(num_bytes, binary=True):
    base = 1024 if binary else 1000
    suffix = 'i' if binary else ''
    for unit in ['B', f'K{suffix}B', f'M{suffix}B', f'G{suffix}B']:
        if abs(num_bytes) < base:
            return f"{num_bytes:.2f} {unit}"
        num_bytes /= base
    return f"{num_bytes:.2f} TiB"

print(bytes_to_human(1073741824))  # 1.00 GiB

Common Gotchas

These are the mistakes that trip up even experienced developers. Burn these into memory.

1. Exponent + negative sign
-3¬≤ = -9 but (-3)¬≤ = 9
The exponent binds before the negation.
2. Subtracting a negative
a - (-b) = a + b
Two negatives make a positive.
3. Adding fractions wrong
1/2 + 1/3 ‚â† 2/5 (wrong!)
1/2 + 1/3 = 3/6 + 2/6 = 5/6 ‚úì
You cannot add numerators if denominators differ.
4. Negative exponents
2‚Åª¬≥ = 1/2¬≥ = 1/8, not -8
A negative exponent means "reciprocal", not "negative result".
5. Scientific notation addition
You must match exponents first. Don't just add coefficients.
Python All gotchas in one place 
# Gotcha 1: Negation vs exponentiation
assert -3**2 == -9
assert (-3)**2 == 9

# Gotcha 2: Double negative
assert 5 - (-3) == 8

# Gotcha 3: Fraction addition (use Fraction class!)
from fractions import Fraction
assert Fraction(1,2) + Fraction(1,3) == Fraction(5,6)

# Gotcha 4: Negative exponent = reciprocal
assert 2**-3 == 0.125  # = 1/8
assert 2**-3 == 1 / 2**3

# Gotcha 5: Scientific notation precision
# 3.2e5 + 4.5e4 ‚Üí align first!
assert abs((3.2e5 + 4.5e4) - 3.65e5) < 1e-10
6. Bits vs Bytes
100 Mbps = 12.5 MB/s, not 100 MB/s!
7. Percentage points vs percent change
2% ‚Üí 4% is +2pp but +100% increase. Clarify which you mean.
8. Inverse rates
100 ops/sec = 10 ms/op (not 100 ms/op).
ms/op = 1000 / (ops/sec)
9. Binary vs decimal prefixes
1 TB = 10¬π¬≤ bytes, but 1 TiB = 2‚Å¥‚Å∞ bytes. ~9% difference!
10. Compound growth
10% for 5 years ‚â† 50%. It's 1.10‚Åµ = 61%.
