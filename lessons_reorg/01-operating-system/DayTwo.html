Why Systems of Equations Matter in CS

Every time you see an intersection point, a balance, or an optimization‚Äîthere's a system of equations underneath. Graphics, games, ML, networking, economics simulations‚Äîthey all reduce to solving systems.

The core insight: A system of equations asks "where do these constraints all hold simultaneously?" Finding that point (or points) is what solving means.
üó∫Ô∏è Simple: Imagine a treasure hunt where you have two clues: "The treasure is 3 steps north of the big rock" and "The treasure is 5 steps east of the old tree." A system of equations is like having multiple clues that must ALL be true at once. The solution is the ONLY spot where all clues line up!

Where you'll use this

  • Graphics: Ray-object intersections, collision detection, inverse kinematics
  • ML/Optimization: Gradient descent finds where ‚àáf = 0 (system of equations)
  • Physics: Force equilibrium, circuit analysis (Kirchhoff's laws)
  • Economics: Supply/demand equilibrium, game theory Nash equilibria
  • Networking: Traffic flow optimization, load balancing
PythonLine intersection = solving a 2√ó2 system
import numpy as np

def line_intersection(p1, d1, p2, d2):
    """
    Find intersection of two lines in 2D.
    Line 1: p1 + t*d1
    Line 2: p2 + s*d2
    
    Setting equal: p1 + t*d1 = p2 + s*d2
    Rearranging:   t*d1 - s*d2 = p2 - p1
    
    This is a 2√ó2 linear system: [d1 | -d2] * [t, s]·µÄ = p2 - p1
    """
    A = np.column_stack([d1, -d2])
    b = p2 - p1
    
    try:
        params = np.linalg.solve(A, b)  # Solve the system!
        t = params[0]
        intersection = p1 + t * d1
        return intersection
    except np.linalg.LinAlgError:
        return None  # Lines are parallel

# Example: Where do y = 2x + 1 and y = -x + 4 intersect?
# Line 1: (0,1) + t*(1,2)  [point + direction]
# Line 2: (0,4) + s*(1,-1)
pt = line_intersection(
    np.array([0, 1]), np.array([1, 2]),
    np.array([0, 4]), np.array([1, -1])
)
print(pt)  # [1. 3.] ‚Üí intersection at (1, 3)

Single Linear Equations

The basic form

ax + b = c Solve for x: ax = c - b x = (c - b) / a (provided a ‚â† 0)

This is the foundation. Every technique we use on systems is just this idea applied cleverly.

Strategy: isolate the variable

The goal: Get the variable alone on one side. Use inverse operations:
‚Ä¢ Addition ‚Üî Subtraction
‚Ä¢ Multiplication ‚Üî Division
Example: 3x + 7 = 22 3x = 22 - 7 (subtract 7) 3x = 15 x = 15 / 3 (divide by 3) x = 5

Variables on both sides

5x - 3 = 2x + 9 5x - 2x = 9 + 3 (collect terms) 3x = 12 x = 4

Section 2

Systems of Linear Equations

What is a system?

Multiple equations that must all be true simultaneously. The solution is the point(s) where all constraints intersect.

System: 2x + y = 7 x - y = 2 We need (x, y) that satisfies BOTH equations.

Method 1: Substitution

Strategy: Solve one equation for one variable, then substitute into the other.
2x + y = 7 ... (1) x - y = 2 ... (2) From (2): x = y + 2 Substitute into (1): 2(y + 2) + y = 7 2y + 4 + y = 7 3y = 3 y = 1 Back-substitute: x = 1 + 2 = 3 Solution: (3, 1)

Method 2: Elimination

Strategy: Add/subtract equations to eliminate one variable.
2x + y = 7 ... (1) x - y = 2 ... (2) Add (1) + (2): 3x = 9 x = 3 Substitute back: 3 - y = 2, so y = 1 Solution: (3, 1)
When to use which:
‚Ä¢ Substitution: When one equation easily solves for a variable
‚Ä¢ Elimination: When coefficients are already set up to cancel

Method 3: Matrix form (the CS way)

Day 2 ¬∑ Operating System can be written as Ax = b and solved with linear algebra.
2x + y = 7 [2 1] [x] [7] x - y = 2 ‚Üí [1 -1] [y] = [2] A ¬∑ x = b Solution: x = A‚Åª¬πb (if A is invertible)
PythonSolving systems with NumPy
import numpy as np

# System: 2x + y = 7, x - y = 2
A = np.array([[2, 1],
              [1, -1]])
b = np.array([7, 2])

# Method 1: np.linalg.solve (preferred - numerically stable)

print(solution)  # [3. 1.]

# Method 2: Matrix inverse (less stable, but instructive)

print(solution2)  # [3. 1.]

# Verify: A @ solution should equal b
print(np.allclose(A @ solution, b))  # True

Three types of outcomes

One solution Lines intersect at one point. det(A) ‚â† 0 Example: x + y = 3 x - y = 1 ‚Üí (2, 1)
No solution Lines are parallel. Inconsistent system. Example: x + y = 3 x + y = 5 ‚Üí ‚àÖ (impossible)
Infinite solutions Lines are the same (dependent equations). Example: x + y = 3 2x + 2y = 6 (same line!) ‚Üí All points on the line
PythonDetecting solution types
import numpy as np

def analyze_system(A, b):
    """Determine if system has one, none, or infinite solutions."""
    det = np.linalg.det(A)
    
    if abs(det) > 1e-10:
        solution = np.linalg.solve(A, b)
        return "unique", solution
    else:
        # Singular matrix - check if consistent
        # Augmented matrix rank vs coefficient matrix rank
        augmented = np.column_stack([A, b])
        rank_A = np.linalg.matrix_rank(A)
        rank_aug = np.linalg.matrix_rank(augmented)
        
        if rank_A == rank_aug:
            return "infinite", None
        else:
            return "none", None

# Test cases
print(analyze_system(
    np.array([[1, 1], [1, -1]]), 
    np.array([3, 1])
))  # ('unique', array([2., 1.]))

print(analyze_system(
    np.array([[1, 1], [2, 2]]), 
    np.array([3, 6])
))  # ('infinite', None)

print(analyze_system(
    np.array([[1, 1], [1, 1]]), 
    np.array([3, 5])
))  # ('none', None)

Larger systems (3+ variables)

The same methods scale up. In practice, use np.linalg.solve() for systems of any size‚Äîit uses optimized algorithms (LU decomposition) that are both fast and stable.

Quadratic Equations

Standard form

ax¬≤ + bx + c = 0 (a ‚â† 0) Quadratics have at most 2 solutions (roots).
„Ä∞Ô∏è Simple: A quadratic equation makes a U-shaped curve (parabola). The "solutions" are where this U crosses the x-axis. It can cross twice (2 solutions), touch once (1 solution), or never touch (0 real solutions, like a U floating above the axis). The quadratic formula finds these crossing points.

Method 1: Factoring

If you can factor, it's the fastest method. Look for two numbers that multiply to ac and add to b.
x¬≤ + 5x + 6 = 0 (x + 2)(x + 3) = 0 x = -2 or x = -3
x¬≤ - 9 = 0 (difference of squares) (x + 3)(x - 3) = 0 x = 3 or x = -3

Method 2: Quadratic formula

Always works. Memorize this.
x = (-b ¬± ‚àö(b¬≤ - 4ac)) / (2a) The discriminant Œî = b¬≤ - 4ac tells you the number of solutions: Œî > 0 ‚Üí Two distinct real roots Œî = 0 ‚Üí One repeated root Œî < 0 ‚Üí No real roots (complex roots)
PythonQuadratic formula implementation
import math

def solve_quadratic(a, b, c):
    """
    Solve ax¬≤ + bx + c = 0.
    Returns tuple of real roots, or None for complex.
    """
    discriminant = b**2 - 4*a*c
    
    if discriminant > 0:
        sqrt_d = math.sqrt(discriminant)
        x1 = (-b + sqrt_d) / (2*a)
        x2 = (-b - sqrt_d) / (2*a)
        return (x1, x2)
    elif discriminant == 0:
        x = -b / (2*a)
        return (x,)
    else:
        # Complex roots
        import cmath
        sqrt_d = cmath.sqrt(discriminant)
        x1 = (-b + sqrt_d) / (2*a)
        x2 = (-b - sqrt_d) / (2*a)
        return (x1, x2)

print(solve_quadratic(1, 5, 6))   # (-2.0, -3.0)
print(solve_quadratic(1, 4, 4))   # (-2.0,) - one root
print(solve_quadratic(1, 1, 1))   # complex roots

Method 3: Completing the square

Useful for deriving the quadratic formula and for rewriting quadratics in vertex form.

x¬≤ + 6x + 5 = 0 Step 1: Move constant x¬≤ + 6x = -5 Step 2: Add (b/2)¬≤ = 9 to both sides x¬≤ + 6x + 9 = -5 + 9 (x + 3)¬≤ = 4 Step 3: Take square root x + 3 = ¬±2 x = -3 ¬± 2 x = -1 or x = -5

PythonRay tracing fundamentals
import numpy as np

def ray_sphere_intersection(ray_origin, ray_dir, sphere_center, sphere_radius):
    """
    Find where a ray intersects a sphere.
    
    Ray: P(t) = origin + t * direction
    Sphere: |P - center|¬≤ = radius¬≤
    
    Substituting gives a quadratic in t:
    |origin + t*dir - center|¬≤ = r¬≤
    
    Let oc = origin - center
    |oc + t*dir|¬≤ = r¬≤
    (oc + t*dir)¬∑(oc + t*dir) = r¬≤
    t¬≤(dir¬∑dir) + 2t(oc¬∑dir) + (oc¬∑oc) - r¬≤ = 0
    
    This is at¬≤ + bt + c = 0 where:
    a = dir¬∑dir, b = 2(oc¬∑dir), c = oc¬∑oc - r¬≤
    """
    oc = ray_origin - sphere_center
    
    a = np.dot(ray_dir, ray_dir)
    b = 2 * np.dot(oc, ray_dir)
    c = np.dot(oc, oc) - sphere_radius**2
    
    discriminant = b**2 - 4*a*c
    
    if discriminant < 0:
        return None  # No intersection
    
    # Return the closer intersection (smaller positive t)
    sqrt_d = np.sqrt(discriminant)
    t1 = (-b - sqrt_d) / (2*a)
    t2 = (-b + sqrt_d) / (2*a)
    
    if t1 > 0:
        return ray_origin + t1 * ray_dir
    elif t2 > 0:
        return ray_origin + t2 * ray_dir
    return None

# Test: ray from (0,0,5) pointing toward origin, sphere at origin with r=1
hit = ray_sphere_intersection(
    np.array([0, 0, 5]),
    np.array([0, 0, -1]),
    np.array([0, 0, 0]),
    1.0
)
print(hit)  # [0. 0. 1.] - hits front of sphere

Expanding Expressions

The distributive property

Core rule: a(b + c) = ab + ac
3(x + 4) = 3x + 12 -2(a - 5) = -2a + 10 x(x + 3) = x¬≤ + 3x

FOIL for binomials

When multiplying two binomials, use FOIL: First, Outer, Inner, Last.

(x + 2)(x + 3) First: x ¬∑ x = x¬≤ Outer: x ¬∑ 3 = 3x Inner: 2 ¬∑ x = 2x Last: 2 ¬∑ 3 = 6 Result: x¬≤ + 3x + 2x + 6 = x¬≤ + 5x + 6

Special Forms (Memorize These)

Difference of squares

a¬≤ - b¬≤ = (a + b)(a - b) Examples: x¬≤ - 9 = (x + 3)(x - 3) 4x¬≤ - 25 = (2x + 5)(2x - 5)

Perfect square trinomials

(a + b)¬≤ = a¬≤ + 2ab + b¬≤ (a - b)¬≤ = a¬≤ - 2ab + b¬≤ Examples: x¬≤ + 6x + 9 = (x + 3)¬≤ x¬≤ - 10x + 25 = (x - 5)¬≤

Sum/difference of cubes

a¬≥ + b¬≥ = (a + b)(a¬≤ - ab + b¬≤) a¬≥ - b¬≥ = (a - b)(a¬≤ + ab + b¬≤)
Mnemonic: SOAP ‚Äî Same, Opposite, Always Positive.

Boolean algebra parallel

Common Gotchas

1. Division by zero
Before dividing by a variable, make sure it's not zero. x/x = 1 only if x ‚â† 0.
2. Losing solutions when squaring
Squaring both sides can introduce extraneous solutions. Always check answers in the original equation.
3. Sign errors in the quadratic formula
It's -b ¬± ‚àö..., not b ¬± ‚àö.... The minus sign is easy to forget.
4. Forgetting both roots
x¬≤ = 9 means x = ¬±3, not just x = 3.
5. Numerical instability
When b¬≤ ‚âà 4ac, the quadratic formula can have precision issues. Use np.roots() or SymPy for reliable numerical solutions.
6. Sign errors when distributing negatives
-(x - 3) = -x + 3, NOT -x - 3
7. Forgetting the middle term in squares
(x + 3)¬≤ = x¬≤ + 6x + 9, NOT x¬≤ + 9
8. Canceling across addition
(x + 2)/2 ‚â† x + 1. You can only cancel factors, not terms!
9. Sum of squares doesn't factor
a¬≤ + b¬≤ has no real factorization. Only difference of squares factors.
10. Losing domain restrictions
(x¬≤ - 4)/(x - 2) = x + 2, but only for x ‚â† 2.
PythonChecking for extraneous solutions
import math

# Solve ‚àö(x+3) = x - 3
# Squaring: x + 3 = (x-3)¬≤ = x¬≤ - 6x + 9
# Rearranging: x¬≤ - 7x + 6 = 0
# Factoring: (x-6)(x-1) = 0
# Candidates: x = 6, x = 1

def check_solution(x):
    lhs = math.sqrt(x + 3)
    rhs = x - 3
    return abs(lhs - rhs) < 1e-10

print(check_solution(6))  # True: ‚àö9 = 3, 6-3 = 3 ‚úì
print(check_solution(1))  # False: ‚àö4 = 2, 1-3 = -2 ‚úó

# x = 1 is EXTRANEOUS - introduced by squaring!
