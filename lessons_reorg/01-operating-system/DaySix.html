Section 0

The Goal: Justify Every Step

The "Receipt" Rule: Just like a store receipt shows exactly what you bought for every cent spent, a proof shows exactly why every line is true. If you can't justify a step, you don't fully understand it yet!

Why this matters

Debugging is Proof-Reading: Finding a bug is often just finding an "unjustified assumption" in your code. Good engineers ask "Why does this work?" instead of just saying "It seems to work."
üèÅ Simple: This day is a pit stop. We aren't learning new racing techniques today; we are checking the engine. We're going line-by-line to make sure nothing is loose.

The annotation format

For each step, ask: "What rule, definition, or prior result justifies this?"

Common justifications: ‚Ä¢ "by definition of [term]" ‚Ä¢ "by assumption" ‚Ä¢ "by algebra" ‚Ä¢ "by inductive hypothesis" ‚Ä¢ "by [theorem name]"
Example 1

Sum of Two Even Numbers

Claim: The sum of two even integers is even.

The Idea: Every even number is just a collection of couples/pairs. If you combine two collections of pairs, you still have only pairs! No singles left over.
Step
Justification
1.Let $a$ and $b$ be even integers.
Setup: introduce variables
2.$a = 2m$, $b = 2n$ for $m,n \in \mathbb{Z}$.
By definition of even
3.$a + b = 2m + 2n$
Substitution
4.$a + b = 2(m + n)$
Algebra (distributive)
5.$a + b = 2k$ (where $k = m+n$)
Closure (integer + integer = integer)
6.Therefore $a+b$ is even. ‚àé
By definition of even (form $2k$)
Example 2

Contrapositive Proof

Claim: If $n^2$ is even, then $n$ is even.

The Strategy: "If it's raining, the grass is wet" is the same as saying "If the grass is dry, it's not raining." This is the contrapositive. To prove this claim, we instead prove "If $n$ is odd, then $n^2$ is odd."
1.Assume $n$ is odd.
Setup contrapositive
2.$n = 2k + 1$
Definition of odd
3.$n^2 = (2k + 1)^2 = 4k^2 + 4k + 1$
Algebra
4.$n^2 = 2(2k^2 + 2k) + 1$
Algebra (factor out 2)
5.$n^2 = 2j + 1$
Substitution ($j = 2k^2+2k$)
6.$n^2$ is odd. ‚àé
Definition of odd
Example 3

Induction Proof

Claim: $1 + 2 + \dots + n = \frac{n(n+1)}{2}$.

The Domino Effect: You only need to prove two things:
  1. The first domino falls (Base Case).
  2. If any domino falls, the next one MUST fall (Inductive Step).
1.Base case: $n=1$. $1 = \frac{1(2)}{2}$. ‚úì
Arithmetic
2.Assume $P(k)$: $1+\dots+k = \frac{k(k+1)}{2}$
Inductive Hypothesis
3.Add next term: $(1+\dots+k) + (k+1)$
Goal step
4.$= \frac{k(k+1)}{2} + (k+1)$
By I.H. (Substituting step 2)
5.$= (k+1)(\frac{k}{2} + 1) = \frac{(k+1)(k+2)}{2}$
Algebra (Factoring out $k+1$)
6.Induction complete. ‚àé
Principle of Induction
Example 4

Algorithm Correctness

Claim: find_max returns the maximum element.

The "Checklist" Rule: For algorithms, we use a checklist to make sure we're never wrong. If you skip any of these, your code might fail on special cases (like empty lists).
Python
def find_max(arr):
    result = arr[0]
    for x in arr[1:]:
        if x > result: result = x
    return result
1.Invariant $I$: result is max observed so far.
Strategy
2.Init: result = arr[0]
Initialization (First item is max of first item)
3.Maintenance: If x > result, update.
Loop logic (Keeps invariant true)
4.Termination: Final result is real max.
Verification (After seeing all items)

Proof: 1. Let a and b be odd integers. 2. Then a = 2m + 1 and b = 2n + 1 for integers m, n. 3. ab = (2m + 1)(2n + 1) 4. ab = 4mn + 2m + 2n + 1 5. ab = 2(2mn + m + n) + 1 6. Let k = 2mn + m + n. This is an integer. 7. So ab = 2k + 1. 8. Therefore ab is odd. ‚àé
