Section 1

The Domino Effect

To prove a statement $P(n)$ is true for all integers $n \geq k$:
  1. Base Case: Prove $P(k)$ is true.
  2. Inductive Step: Prove that if $P(n)$ is true (the hypothesis), then $P(n+1)$ is true.

This is conceptually identical to knocking down an infinite line of dominoes. You push the first one, and you ensure every domino knocks down the next.

Section 2

Standard Induction

Example: Sum of Integers

Prove $\sum_{i=1}^n i = \frac{n(n+1)}{2}$.

1. Base Case (n=1): LHS = 1. RHS = 1(2)/2 = 1. Checked. 2. Inductive Step: Assume 1 + ... + k = k(k+1)/2. Goal: Show 1 + ... + k + (k+1) = (k+1)(k+2)/2. LHS = [1 + ... + k] + (k+1) = k(k+1)/2 + (k+1) (by Hypothesis) = (k+1) [ k/2 + 1 ] = (k+1) [ (k+2)/2 ] = (k+1)(k+2)/2. QED.
Section 3

Strong Induction

Sometimes knowing $P(k)$ isn't enough to prove $P(k+1)$, but knowing $P(1), P(2), ..., P(k)$ all are true helps.

Strong Induction: Assume $P(i)$ is true for ALL $1 \leq i \leq k$. Prove $P(k+1)$.

Example: Fundamental Theorem of Arithmetic

Prove every integer $n > 1$ can be written as a product of primes.
If $n+1$ is prime, done. If composite, $n+1 = a \cdot b$. By Strong Induction, $a$ and $b$ (being smaller than $n+1$) have prime factorizations. Thus their product $a \cdot b$ is a product of primes.

Section 4

Structural Induction

CS often deals with recursively defined structures (Trees, Lists, Formulas). Structural Induction is induction on the structure rather than integers.

For a Binary Tree T: 1. Base Case: T is empty (or a single leaf). Prove P(T). 2. Inductive Step: Assume P(L) and P(R) for subtrees. Prove P(Tree(L, R)).
Section 5

Recursion is Induction

Every recursive function relies on inductive logic for correctness. You handle the base case (stopping condition) and the recursive step (reducing the problem).

Python Towers of Hanoi 
def hanoi(n, source, target, auxiliary):
    # Base Case
    if n == 1:
        print(f"Move disk 1 from {source} to {target}")
        return
    
    # Inductive Step (Recursive Case)
    # Move n-1 disks from Source to Aux (Inductive Hypothesis)
    hanoi(n-1, source, auxiliary, target)
    
    # Move nth disk from Source to Target
    print(f"Move disk {n} from {source} to {target}")
    
    # Move n-1 disks from Aux to Target (Inductive Hypothesis)
    hanoi(n-1, auxiliary, target, source)

print("Solving for 3 disks:")
hanoi(3, 'A', 'C', 'B')
Section 6

The Pigeonhole Principle

If $n$ pigeons fly into $k$ holes, and $n > k$, then at least one hole contains at least two pigeons.

Applications

  • Hashing: If you hash $N$ keys into a table of size $M < N$, collisions are inevitable.
  • Compression: Lossless compression cannot shrink every file.
  • NYC Hair: Two people in NYC have the same number of hairs (Pop ~8M, hairs ~150k).
Generalized: If $N$ objects are placed into $k$ boxes, at least one box has $\lceil N/k \rceil$ objects.
Section 7

Inclusion-Exclusion (PIE)

How to count the size of a union of sets?

Two Sets: $|A \cup B| = |A| + |B| - |A \cap B|$
Three Sets: |A ∪ B ∪ C| = |A| + |B| + |C| - (|A∩B| + |A∩C| + |B∩C|) + |A∩B∩C|

Pattern: Add singles, subtract pairs, add triples, subtract quads...

Section 8

Birthday Paradox

With just 23 people, there's a >50% chance of a shared birthday. This is a probabilistic version of Pigeonhole.

Python Birthday Simulation 
import random

def has_collision(num_people):
    birthdays = []
    for _ in range(num_people):
        day = random.randint(1, 365)
        if day in birthdays:
            return True
        birthdays.append(day)
    return False

# Run 10000 trials for 23 people
collisions = sum(has_collision(23) for _ in range(10000))
print(f"Probability: {collisions/10000:.1%}")  # ~50%
