Section 1

The Language of CS

Logic is the formal study of valid reasoning. It is the operating system of mathematics.

In Computer Science, logic appears everywhere:

  • Hardware: Logic gates (AND, OR, NOT) build CPUs.
  • Software: `if (p && q)` conditional statements.
  • databases: SQL queries are predicates filtering data.
  • AI: Knowledge graphs and inference engines.
Section 2

Propositional Logic

Propositions

A proposition is a declarative sentence that is either True or False.

✅ "The Earth is round." (True)
✅ "2 + 2 = 5." (False)
❌ "What time is it?" (Not a proposition - question)
❌ "x + 1 = 2." (Not a proposition - depends on x. This is a predicate.)

Connectives & Truth Tables

p | q | ¬p | p ∧ q | p ∨ q | p ⊕ q | p → q | p ↔ q --+---+----+-------+-------+-------+-------+------- T | T | F | T | T | F | T | T T | F | F | F | T | T | F | F F | T | T | F | T | T | T | F F | F | T | F | F | F | T | T
Note on Implication ($p \to q$): "If p, then q" is only False when p is True and q is False. If p is False, the statement is vacuously true.
"If pigs can fly, then I am the Pope" is a TRUE statement.
Section 3

First-Order Logic (Predicate Logic)

Propositional logic can't talk about "variables." First-Order Logic introduces Predicates $P(x)$ and Quantifiers.

Quantifiers

Universal ($\forall$)

"For all x..."

∀x ∈ ℝ, x² ≥ 0 "Every real number squared is non-negative."

Existential ($\exists$)

"There exists x..."

∃x ∈ ℤ, x + 1 = 0 "There is an integer x such that x+1 is 0."

Negating Quantifiers (De Morgan for Logic)

To move a negation inside a quantifier, flip the quantifier and negate the predicate.

$\neg (\forall x, P(x)) \iff \exists x, \neg P(x)$
$\neg (\exists x, P(x)) \iff \forall x, \neg P(x)$
Example: Statement: "Everyone loves ice cream." (∀x, Loves(x, IceCream)) Negation: "It is not the case that everyone loves ice cream." Equivalent: "Someone does NOT love ice cream." (∃x, ¬Loves(x, IceCream))
Section 4

Rules of Inference

Valid Arguments

An argument is valid if the truth of the premises guarantees the truth of the conclusion.

Modus Ponens
$p \to q$
$p$
$\therefore q$
Modus Tollens
$p \to q$
$\neg q$
$\therefore \neg p$
Hypothetical Syllogism
$p \to q$
$q \to r$
$\therefore p \to r$
Resolution
$p \lor q$
$\neg p \lor r$
$\therefore q \lor r$
Section 5

Logic in Code

We can write a simple Python script to generate truth tables for any logical expression.

Python Truth Table Generator 
import itertools

def truth_table(variables, func):
    # Header
    header = " | ".join(variables) + " | Result"
    print(header)
    print("-" * len(header))

    # Iterate all 2^n combinations
    # [False, True] product n times
    for values in itertools.product([False, True], repeat=len(variables)):
        # Map variable names to values
        assignment = dict(zip(variables, values))
        
        # Evaluate function
        result = func(**assignment)
        
        # Format output (T/F)
        row = ["T" if v else "F" for v in values]
        res_str = "T" if result else "F"
        print(" | ".join(row) + f" | {res_str}")

# Example: Implication (p -> q) equivalent to (not p or q)
def implication(p, q):
    return (not p) or q

# Example: XOR (p ^ q)
def xor_gate(p, q):
    return p ^ q

print("Truth Table for p -> q:")
truth_table(['p', 'q'], implication)

print("\nTruth Table for p XOR q:")
truth_table(['p', 'q'], xor_gate)
